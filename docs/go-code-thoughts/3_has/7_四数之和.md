# 四数之和：排序、固定与双指针

四数之和可以看成三数之和的扩展。
核心仍然是排序后的双指针，只是需要先固定前两个数，
再在剩余区间中寻找后两个数。

## 18 四数之和

[题目链接](https://leetcode.cn/problems/4sum/)

### 18 题意

给定一个由 `n` 个整数组成的数组 `nums` 和一个目标值 `target`。

找出并返回所有满足条件且不重复的四元组：

```text
nums[a] + nums[b] + nums[c] + nums[d] == target
```

### 18 核心策略

这道题的整体策略是排序、嵌套循环和双指针。

1. 先对数组排序。
2. 用第一层循环固定第一个数 `nums[i]`。
3. 用第二层循环固定第二个数 `nums[j]`。
4. 在 `j` 之后的区间中使用双指针。
5. 通过 `left` 和 `right` 寻找剩下两个数。

固定 `nums[i]` 和 `nums[j]` 后，问题会变成：

```text
nums[left] + nums[right] == target - nums[i] - nums[j]
```

这就转化成了有序数组中的两数之和问题。

### 18 去重逻辑

四数之和最容易出错的地方是去重。

需要处理三层去重：

1. 对 `i` 去重：
   如果 `nums[i] == nums[i-1]`，跳过当前 `i`。
2. 对 `j` 去重：
   如果 `j > i+1` 且 `nums[j] == nums[j-1]`，跳过当前 `j`。
3. 对 `left` 和 `right` 去重：
   找到答案后，分别跳过和当前左右指针值相同的元素。

其中 `j > i+1` 很关键。
它表示只在当前 `i` 的内部循环中对 `j` 去重，
避免错误跳过新一轮 `i` 对应的第一个 `j`。

### 18 代码

```go
import "sort"

func fourSum(nums []int, target int) [][]int {
    ans := [][]int{}
    sort.Ints(nums)

    for i := 0; i < len(nums); i++ {
        if i > 0 && nums[i] == nums[i-1] {
            continue
        }

        for j := i + 1; j < len(nums); j++ {
            if j > i+1 && nums[j] == nums[j-1] {
                continue
            }

            left, right := j+1, len(nums)-1
            for left < right {
                sum := int64(nums[i]) +
                    int64(nums[j]) +
                    int64(nums[left]) +
                    int64(nums[right])

                if sum == int64(target) {
                    ans = append(
                        ans,
                        []int{nums[i], nums[j], nums[left], nums[right]},
                    )

                    for left < right && nums[left] == nums[left+1] {
                        left++
                    }
                    for left < right && nums[right] == nums[right-1] {
                        right--
                    }

                    left++
                    right--
                } else if sum < int64(target) {
                    left++
                } else {
                    right--
                }
            }
        }
    }

    return ans
}
```

### 18 溢出处理

原始代码中使用 `int64` 计算四数之和，这是一个重要细节。

如果直接用 `int` 相加，在数据范围较大时可能发生溢出。
把每个数转成 `int64` 后再相加，可以让比较更安全。

### 18 复杂度

- 时间复杂度：`O(n^3)`
- 空间复杂度：`O(1)`，不计返回结果

排序后，前两层循环固定两个数，内部双指针线性移动，
因此整体时间复杂度是 `O(n^3)`。

## 本题总结

四数之和的关键是把问题降维。

固定前两个数后，剩下的问题就变成了排序数组中的两数之和。
真正需要小心的是每一层的去重：

- 外层 `i` 去重。
- 内层 `j` 去重。
- 找到答案后，对 `left` 和 `right` 去重。

这类 N 数之和问题通常可以沿着同一条思路扩展：
先排序，再固定若干个数，最后用双指针处理剩余两个数。

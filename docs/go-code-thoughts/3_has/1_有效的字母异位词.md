# 字母异位词专题：用计数表处理字符串

这一组题都围绕字母异位词展开。把它们放在一起看，
最重要的收获是：很多字符串问题并不需要比较排列顺序，
而是要比较字符的组成情况。

这类问题的核心工具是计数表。题目只包含小写英文字母时，
可以用长度为 26 的数组记录每个字符出现的次数；如果字符范围
扩展到 Unicode，就应改用 `map[rune]int`。

## 核心概念

两段字符串互为字母异位词，表示它们包含的每种字符数量完全相同，
只是排列顺序不同。

例如 `anagram` 和 `nagaram` 的字符组成一致：

| 字符 | 次数 |
| --- | ---: |
| `a` | 3 |
| `g` | 1 |
| `m` | 1 |
| `n` | 1 |
| `r` | 1 |

所以，判断异位词的本质就是判断两份字符成分表是否一致。

## 计数数组

如果字符串只包含小写英文字母，可以使用 `[26]int` 或 `[]int`
作为计数表。

```go
index := ch - 'a'
```

这个表达式可以把字符映射到数组下标：

| 字符 | 下标 |
| --- | ---: |
| `a` | `0` |
| `b` | `1` |
| `z` | `25` |

下面几道题都是这个思想的不同变体。

## 242 有效的字母异位词

[题目链接](https://leetcode.cn/problems/valid-anagram/)

### 242 题意

给定两个字符串 `s` 和 `t`，判断 `t` 是否是 `s` 的字母异位词。

### 242 思路

这是计数数组最直接的应用。

1. 如果两个字符串长度不同，一定不是异位词。
2. 遍历 `s`，将对应字符计数加一。
3. 遍历 `t`，将对应字符计数减一。
4. 最后检查计数数组。
   如果所有位置都为 `0`，说明两边字符组成完全一致。

这个过程保留了先加后减、最后归零的判断方式。
它比排序更贴合这道题，因为题目只关心字符数量，不关心顺序。

### 242 代码

```go
func isAnagram(s string, t string) bool {
    if len(s) != len(t) {
        return false
    }

    count := make([]int, 26)

    for i := 0; i < len(s); i++ {
        count[s[i]-'a']++
    }

    for i := 0; i < len(t); i++ {
        count[t[i]-'a']--
    }

    for _, value := range count {
        if value != 0 {
            return false
        }
    }

    return true
}
```

### 242 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`，因为计数数组长度固定为 26

### 242 进阶

如果输入可能包含中文、emoji 或其他 Unicode 字符，`[26]int`
就不够用了。这时可以使用 `map[rune]int`。整体思路仍然是计数，
只是计数容器从固定数组变成了哈希表。

## 383 赎金信

[题目链接](https://leetcode.cn/problems/ransom-note/)

### 383 题意

给定两个字符串 `ransomNote` 和 `magazine`，判断 `ransomNote`
能不能由 `magazine` 中的字符构成。`magazine` 中的每个字符
只能使用一次。

### 383 思路

这道题和 242 有效的字母异位词相似，但要求不是两边完全相等，
而是 `magazine` 的字符数量能够覆盖 `ransomNote` 的需求。

可以把它理解成需求和库存：

1. `ransomNote` 是需求清单。
2. `magazine` 是字符库存。
3. 先遍历 `ransomNote`，把需要的字符在计数表中减一。
4. 再遍历 `magazine`，用已有字符把计数加回来。
5. 如果最后仍有某个位置小于 `0`，说明对应字符库存不足。

这个写法保留了原文中先记需求，再用材料偿还的思路。

### 383 代码

```go
func canConstruct(ransomNote string, magazine string) bool {
    if len(ransomNote) > len(magazine) {
        return false
    }

    count := make([]int, 26)

    for i := 0; i < len(ransomNote); i++ {
        index := ransomNote[i] - 'a'
        count[index]--
    }

    for i := 0; i < len(magazine); i++ {
        index := magazine[i] - 'a'
        count[index]++
    }

    for _, value := range count {
        if value < 0 {
            return false
        }
    }

    return true
}
```

### 383 复杂度

- 时间复杂度：`O(m + n)`
- 空间复杂度：`O(1)`

## 49 字母异位词分组

[题目链接](https://leetcode.cn/problems/group-anagrams/)

### 49 题意

给定一个字符串数组，将所有互为字母异位词的字符串分到同一组。

### 49 思路

这一题的关键是把字符成分表作为哈希表的 key。

1. 遍历每个字符串。
2. 为当前字符串生成一个 `[26]int` 计数数组。
3. 互为异位词的字符串会得到完全相同的计数数组。
4. 在 Go 中，数组可以作为 map 的 key。
5. 因此可以直接使用 `map[[26]int][]string` 分组。
6. 最后把 map 中所有分组取出。

这里保留了原来的重要发现：不是把字符串本身作为 key，
而是把它的字符组成作为 key。

### 49 代码

```go
func groupAnagrams(strs []string) [][]string {
    groups := make(map[[26]int][]string)

    for _, str := range strs {
        key := [26]int{}
        for _, ch := range str {
            key[ch-'a']++
        }
        groups[key] = append(groups[key], str)
    }

    ans := make([][]string, 0, len(groups))
    for _, group := range groups {
        ans = append(ans, group)
    }

    return ans
}
```

### 49 复杂度

设字符串数量为 `n`，单个字符串平均长度为 `k`。

- 时间复杂度：`O(n * k)`
- 空间复杂度：`O(n * k)`，主要用于保存分组结果

## 438 找到字符串中所有字母异位词

[题目链接](https://leetcode.cn/problems/find-all-anagrams-in-a-string/)

### 438 题意

给定两个字符串 `s` 和 `p`，找到 `s` 中所有 `p` 的异位词子串，
返回这些子串的起始索引。

### 438 思路

这一题是计数表加滑动窗口。

不需要枚举 `s` 的所有子串再逐个统计。因为目标子串的长度一定
等于 `len(p)`，所以可以在 `s` 上维护一个长度固定的窗口。

具体步骤：

1. 统计 `p` 的字符组成，得到 `target`。
2. 初始化 `s` 的第一个窗口，得到 `window`。
3. 如果 `window == target`，记录起点 `0`。
4. 之后窗口每次右移一格：
   - 新进入窗口的字符计数加一。
   - 滑出窗口的字符计数减一。
   - 比较当前窗口和目标计数表是否相等。

这个思路保留了原文中最重要的一点：滑动时不重新计算整个窗口，
只维护进入和离开的两个字符，因此更新代价很低。

### 438 代码

```go
func findAnagrams(s string, p string) []int {
    if len(s) < len(p) {
        return []int{}
    }

    ans := []int{}
    var target, window [26]int

    for i := 0; i < len(p); i++ {
        target[p[i]-'a']++
        window[s[i]-'a']++
    }

    if window == target {
        ans = append(ans, 0)
    }

    for i := len(p); i < len(s); i++ {
        window[s[i]-'a']++
        window[s[i-len(p)]-'a']--

        if window == target {
            ans = append(ans, i-len(p)+1)
        }
    }

    return ans
}
```

### 438 复杂度

- 时间复杂度：`O(n)`，其中 `n` 是 `s` 的长度
- 空间复杂度：`O(1)`

每次比较两个 `[26]int` 的成本是常数。

## 专题总结

这四道题看起来形式不同，但底层思想一致：用计数表表示字符串
的字符组成。

- 242 有效的字母异位词：
  计数加减后检查是否归零。
- 383 赎金信：
  需求减一，库存加一，检查是否有负数。
- 49 字母异位词分组：
  使用 `[26]int` 作为 map key。
- 438 找到字符串中所有字母异位词：
  固定长度滑动窗口加计数表比较。

这组题的学习价值在于：当字符串问题只关心字符数量时，
可以优先考虑计数数组或哈希表；当目标出现在连续子串中时，
再结合滑动窗口维护局部计数。

# 三数之和：排序后的双指针核心片段

这篇原始记录只保留了三数之和的核心双指针片段，
重点在于固定一个数之后，如何用左右指针寻找另外两个数。

## 15 三数之和

[题目链接](https://leetcode.cn/problems/3sum/)

### 15 题意

给定整数数组 `nums`，找出所有满足和为 `0` 的三元组。
答案中不能包含重复的三元组。

### 15 原始记录重点

原始记录保留的是以下逻辑：

1. 固定下标 `i`。
2. 使用 `left := i + 1` 和 `right := len(nums) - 1`。
3. 在 `left < right` 时，根据两数之和和目标值的关系移动指针。
4. 找到答案后，对 `left` 和 `right` 都进行去重。
5. 去重后继续收缩窗口。

这里默认数组已经排序，并且外层已经完成了对 `i` 的遍历和去重。

### 15 核心代码片段

```go
left, right := i+1, len(nums)-1
for left < right {
    if nums[left]+nums[right] == target {
        ans = append(ans, []int{nums[left], nums[right], nums[i]})
        for left < right && nums[left] == nums[left+1] {
            left++
        }
        for left < right && nums[right-1] == nums[right] {
            right--
        }
        left++
        right--
    } else if nums[left]+nums[right] < target {
        left++
    } else {
        right--
    }
}
```

### 15 片段解析

这段代码的核心是固定 `nums[i]` 后，在右侧区间寻找两个数。

如果 `nums[left] + nums[right] == target`，说明找到了一组答案。
随后必须移动指针跳过重复值，否则结果中可能出现重复三元组。

如果两数之和小于 `target`，说明当前和偏小。
因为数组已经排序，所以应该移动 `left`，尝试增大和。

如果两数之和大于 `target`，说明当前和偏大。
此时应该移动 `right`，尝试减小和。

### 15 注意点

- 三数之和通常先排序。
- 外层固定 `i` 时也需要去重。
- 找到一组答案后，`left` 和 `right` 都要跳过重复值。
- 跳过重复值之后，还要继续执行 `left++` 和 `right--`。

## 本题总结

三数之和的关键不是哈希表，而是排序后的双指针。

固定一个数后，问题会转化成有序数组中的两数之和。
双指针能够根据当前和的大小决定移动方向，从而避免暴力枚举。

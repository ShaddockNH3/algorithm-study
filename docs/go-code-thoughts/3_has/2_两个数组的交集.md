# 数组交集专题：从存在判断到频率计数

这两道题都在求数组交集，但要求并不一样。

- 349 两个数组的交集：
  只关心元素是否同时出现，结果必须唯一。
- 350 两个数组的交集 II：
  需要考虑元素出现次数，结果中元素数量取两个数组中的较小值。

因此，349 更接近集合问题，350 更接近频率计数问题。

## 349 两个数组的交集

[题目链接](https://leetcode.cn/problems/intersection-of-two-arrays/)

### 349 题意

给定两个数组 `nums1` 和 `nums2`，返回它们的交集。
输出结果中的每个元素必须唯一，可以不考虑输出顺序。

### 349 核心要求

找出两个数组中共同出现过的元素，并且结果不能重复。

### 349 思路

这道题适合用集合。

1. 先遍历 `nums1`，用 `map[int]struct{}` 建立集合。
2. 再遍历 `nums2`，检查当前元素是否存在于集合中。
3. 如果存在，说明这个元素属于交集。
4. 为了防止 `nums2` 中的重复元素导致答案重复，
   再用一个 `resultSet` 保存交集元素。
5. 最后把 `resultSet` 中的 key 转成切片返回。

这里的关键点是结果去重。即使 `nums2` 中有多个相同元素，
答案中也只能出现一次。

### 349 代码

```go
func intersection(nums1 []int, nums2 []int) []int {
    set := make(map[int]struct{})
    for _, num := range nums1 {
        set[num] = struct{}{}
    }

    resultSet := make(map[int]struct{})
    for _, num := range nums2 {
        if _, ok := set[num]; ok {
            resultSet[num] = struct{}{}
        }
    }

    ans := make([]int, 0, len(resultSet))
    for num := range resultSet {
        ans = append(ans, num)
    }

    return ans
}
```

### 349 复杂度

- 时间复杂度：`O(m + n)`
- 空间复杂度：`O(m + k)`

其中 `m` 是 `nums1` 的长度，`n` 是 `nums2` 的长度，
`k` 是交集去重后的元素数量。

## 350 两个数组的交集 II

[题目链接](https://leetcode.cn/problems/intersection-of-two-arrays-ii/)

### 350 题意

输出结果中每个元素出现的次数，应与元素在两个数组中都出现的次数
一致。如果出现次数不一致，则取较小值。

### 350 核心要求

找出可重复的交集元素。重复次数不能只看是否存在，
而要看两个数组中对应元素的最小出现次数。

### 350 思路

这道题需要计数，因此工具从集合变成 `map[int]int`。

1. 遍历 `nums1`，统计每个数字出现的次数。
2. 遍历 `nums2`，把每个数字拿去计数表里匹配。
3. 如果这个数字还存在，并且剩余次数大于 `0`，
   就把它加入答案。
4. 每匹配一次，就将计数表中的数量减一。
5. 遍历完成后，答案中保留的就是数量匹配后的交集。

这份思路的核心是库存消耗。`nums1` 提供可用次数，
`nums2` 每命中一次就消耗一次。

### 350 代码

```go
func intersect(nums1 []int, nums2 []int) []int {
    count := make(map[int]int)

    for _, num := range nums1 {
        count[num]++
    }

    ans := []int{}

    for _, num := range nums2 {
        if value, ok := count[num]; ok && value > 0 {
            count[num]--
            ans = append(ans, num)
        }
    }

    return ans
}
```

### 350 复杂度

- 时间复杂度：`O(m + n)`
- 空间复杂度：`O(m)`

其中 `m` 是 `nums1` 的长度，`n` 是 `nums2` 的长度。

## 进阶思考

### 数组已经有序

如果两个数组都已经排序，可以使用双指针。

两个指针分别指向两个数组开头：

- 如果两个元素相等，记录答案，并同时右移。
- 如果 `nums1[i] < nums2[j]`，移动 `i`。
- 如果 `nums1[i] > nums2[j]`，移动 `j`。

这种做法不需要额外哈希表，适合已经排序的输入。

### 一个数组明显更小

如果 `nums1` 比 `nums2` 小，更适合用较小的数组构建哈希表。
这样可以减少额外空间。

一般原则是：谁小就用谁建表，谁大就用来查询或消耗计数。

### 一个数组在磁盘上

如果 `nums2` 在磁盘上，不能一次性全部放入内存，可以分情况处理。

如果 `nums1` 能放入内存：

1. 用 `nums1` 构建哈希表。
2. 分块读取 `nums2`。
3. 对每一块数据执行查询或计数消耗。

如果 `nums1` 也很大：

1. 先对两个大文件做外部排序。
2. 再用双指针思想分块读取比较。

## 专题总结

- 349 两个数组的交集：
  结果必须唯一，所以使用集合。
- 350 两个数组的交集 II：
  结果允许重复，所以使用频率计数。

判断这类题时，先看题目要的是存在关系还是数量关系。
只问有没有，用集合；要问有多少，用计数表。

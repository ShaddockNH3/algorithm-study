# 四数相加 II：用哈希表把四层枚举拆成两组

四数相加 II 是哈希表降维的典型题。
它的关键不是直接枚举四个数组，而是把四个数组拆成两组，
先保存前两组的和，再用后两组去查询相反数。

## 454 四数相加 II

[题目链接](https://leetcode.cn/problems/4sum-ii/)

### 454 题意

给定四个整数数组 `nums1`、`nums2`、`nums3` 和 `nums4`，
数组长度都是 `n`。

计算有多少个元组 `(i, j, k, l)` 满足：

```text
nums1[i] + nums2[j] + nums3[k] + nums4[l] == 0
```

### 454 暴力解法的问题

如果直接使用四层循环枚举所有组合，时间复杂度是 `O(n^4)`。

当 `n = 200` 时，枚举量会非常大，基本无法通过。
因此需要用哈希表存储中间结果，把查询成本降下来。

### 454 分组思路

目标式可以写成：

```text
(nums1[i] + nums2[j]) + (nums3[k] + nums4[l]) == 0
```

因此可以把四个数组分成两组：

- 第一组：`nums1` 和 `nums2`
- 第二组：`nums3` 和 `nums4`

处理步骤：

1. 遍历 `nums1` 和 `nums2`，计算所有 `num1 + num2`。
2. 用 `map[int]int` 记录每个两数之和出现的次数。
3. 遍历 `nums3` 和 `nums4`，计算 `num3 + num4`。
4. 为了让四数之和为 `0`，需要查找：
   `target := -(num3 + num4)`。
5. 如果 `target` 在 map 中出现过，就把出现次数累加到答案中。

这个做法保留了原文中的核心思想：分组求和，哈希暂存。
哈希表里不只记录某个和是否存在，还要记录这个和出现了多少次。

### 454 代码

```go
func fourSumCount(
    nums1 []int,
    nums2 []int,
    nums3 []int,
    nums4 []int,
) int {
    count := make(map[int]int)

    for _, num1 := range nums1 {
        for _, num2 := range nums2 {
            sum := num1 + num2
            count[sum]++
        }
    }

    ans := 0

    for _, num3 := range nums3 {
        for _, num4 := range nums4 {
            target := -(num3 + num4)
            if value, ok := count[target]; ok {
                ans += value
            }
        }
    }

    return ans
}
```

### 454 复杂度

- 时间复杂度：`O(n^2)`
- 空间复杂度：`O(n^2)`

前两组数组会产生 `n^2` 个和，后两组数组也会枚举 `n^2` 次。
整体从四层循环降成两个两层循环。

## 本题总结

四数相加 II 的关键是拆分。

如果直接枚举四个数组，会得到 `O(n^4)` 的复杂度。
如果拆成两组，并把第一组的和存入哈希表，就可以把问题转成：

```text
当前后半部分的和，需要多少个前半部分的相反数来配对
```

这类题的经验是：当多个变量共同满足一个等式时，
可以尝试把变量分组，先缓存一部分组合结果，再用另一部分查询。

# 两数之和：哈希表中的空间换时间

两数之和是哈希表最经典的入门题之一。
它的关键不是枚举所有组合，而是在遍历过程中快速查询目标值的另一半。

## 1 两数之和

[题目链接](https://leetcode.cn/problems/two-sum/)

### 1 题意

给定一个整数数组 `nums` 和一个整数目标值 `target`，
在数组中找出和为 `target` 的两个整数，并返回它们的数组下标。

题目保证每组输入只对应一个答案，且同一个元素不能重复使用。

### 1 暴力解法的问题

直接做法是双层循环：

1. 枚举第一个数。
2. 再枚举第二个数。
3. 判断两数之和是否等于 `target`。

这种做法时间复杂度是 `O(n^2)`。题目的进阶提示要求寻找
小于 `O(n^2)` 的算法，因此需要用哈希表降低查询成本。

### 1 哈希表思路

哈希表保存已经遍历过的数字及其下标。

遍历到当前数字 `currentNum` 时，需要寻找的另一个数字是：

```go
complement := target - currentNum
```

具体步骤：

1. 创建 `map[int]int`，key 存数字，value 存下标。
2. 遍历数组。
3. 对当前数字 `currentNum`，计算 `complement`。
4. 先在哈希表中查找 `complement` 是否存在。
5. 如果存在，说明已经找到一组答案，直接返回两个下标。
6. 如果不存在，把当前数字和下标写入哈希表，供后续数字查询。

这里最重要的是先查后存。

如果先把当前数字存进去，再查询 `complement`，在某些情况下可能会
错误地把同一个元素使用两次。先查后存可以保证当前数字只会和之前
出现过的数字配对。

### 1 代码

```go
func twoSum(nums []int, target int) []int {
    hasmap := make(map[int]int)

    for i, currentNum := range nums {
        complement := target - currentNum

        if complementIndex, ok := hasmap[complement]; ok {
            return []int{complementIndex, i}
        }

        hasmap[currentNum] = i
    }

    return nil
}
```

### 1 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

哈希表查询平均时间复杂度是 `O(1)`。
整道题只需要遍历一次数组，因此总时间复杂度降为 `O(n)`。

## 本题总结

两数之和体现了空间换时间的典型思路。

暴力枚举慢，是因为每次都要重新寻找另一个数；哈希表快，
是因为它把已经见过的数字登记下来，让后续查询可以在平均 `O(1)`
时间内完成。

遇到这类需要查找某个目标值是否出现过的问题时，可以优先考虑哈希表。

# 快乐数：用集合检测循环

快乐数是一道检测循环的题。它表面上是在做数字变换，
本质上是在判断一个序列最终会到达 `1`，还是会进入循环。

## 202 快乐数

[题目链接](https://leetcode.cn/problems/happy-number/)

### 202 题意

编写一个算法，判断一个数 `n` 是不是快乐数。

快乐数的定义如下：

1. 对于一个正整数，每一次将该数替换为它每个位置上的数字平方和。
2. 重复这个过程。
3. 如果最终结果变为 `1`，那么这个数就是快乐数。
4. 如果过程进入无限循环，并且始终无法变成 `1`，
   那么它就不是快乐数。

## 哈希集合解法

### 集合解法思路

这道题的核心是判断数字变换过程会不会陷入循环。

只要某个数字在变换过程中重复出现，那么从这个数字开始，
后续序列也会完全重复。此时整个过程进入了环，永远无法到达 `1`。

因此，核心任务就是检测重复值。

具体步骤：

1. 使用 `map[int]struct{}` 记录已经出现过的数字。
2. 当 `n != 1` 时持续进行变换。
3. 每次变换前，先检查当前 `n` 是否已经出现过。
4. 如果出现过，说明进入循环，返回 `false`。
5. 如果没有出现过，就把当前 `n` 记录下来。
6. 计算各位数字平方和，得到下一个 `n`。
7. 如果循环结束时 `n == 1`，说明它是快乐数。

这个方法直观、稳定，也符合哈希表记录历史状态的常见用法。

### 集合解法代码

```go
func getNext(n int) int {
    sum := 0
    for n > 0 {
        digit := n % 10
        sum += digit * digit
        n = n / 10
    }
    return sum
}

func isHappy(n int) bool {
    set := make(map[int]struct{})

    for n != 1 {
        if _, ok := set[n]; ok {
            return false
        }

        set[n] = struct{}{}
        n = getNext(n)
    }

    return true
}
```

### 集合解法复杂度

- 时间复杂度：`O(k)`
- 空间复杂度：`O(k)`

其中 `k` 是进入循环或到达 `1` 之前经历的数字个数。

## 快慢指针解法

### 快慢指针思路

这道题也可以使用快慢指针。

数字每次通过 `getNext` 变成下一个数字，可以看成一个隐式链表：

- 如果最终到达 `1`，说明链表通向终点。
- 如果无法到达 `1`，说明链表中存在环。

快慢指针的做法和链表判环类似：

1. `slow` 每次走一步：`slow = getNext(slow)`。
2. `fast` 每次走两步：`fast = getNext(getNext(fast))`。
3. 如果存在循环，`fast` 最终会追上 `slow`。
4. 当两者相遇时，判断相遇点是否为 `1`。

相比哈希集合法，快慢指针法的空间复杂度更低，但理解成本稍高。

### 快慢指针代码

```go
func isHappy(n int) bool {
    slow := n
    fast := getNext(n)

    for fast != 1 && slow != fast {
        slow = getNext(slow)
        fast = getNext(getNext(fast))
    }

    return fast == 1
}
```

### 快慢指针复杂度

- 时间复杂度：`O(k)`
- 空间复杂度：`O(1)`

## 本题总结

快乐数的关键不是数字本身，而是变换序列是否重复。

- 想要直观解决，可以用哈希集合记录历史值。
- 想要优化空间，可以把数字变换过程看成隐式链表，
  再使用快慢指针判环。

这类题的启发是：当过程会不断生成下一个状态时，
要优先考虑是否存在状态重复，以及重复后是否意味着循环。

# 平衡二叉树：自底向上判断高度

平衡二叉树要求每个节点的左右子树高度差都不超过 1。

这道题的重点是：不能只检查根节点。
必须保证整棵树中所有节点都满足平衡条件。

## 110 平衡二叉树

[题目链接](https://leetcode.cn/problems/balanced-binary-tree/)

### 110 题意

给定一个二叉树，判断它是否是平衡二叉树。

平衡二叉树定义为：

```text
每个节点的左右两个子树高度差的绝对值不超过 1
```

## 自顶向下思路

最直观的做法是按照定义写递归。

对每个节点：

1. 计算左子树高度。
2. 计算右子树高度。
3. 判断高度差是否超过 1。
4. 再递归判断左子树是否平衡。
5. 再递归判断右子树是否平衡。

这个思路正确，但会重复计算高度。

例如计算根节点高度时已经遍历过左子树，
之后判断左子树是否平衡时，又会重新遍历左子树。

因此需要更高效的自底向上思路。

## 自底向上思路

自底向上可以用后序遍历。

父节点先等待左右子树返回信息：

- 子树的深度。
- 子树是否已经不平衡。

为了把这两个信息合并，可以让辅助函数 `getDepth` 返回：

- 非负数：表示当前子树平衡，并返回真实深度。
- `-1`：表示当前子树已经不平衡。

一旦某个子树返回 `-1`，就直接继续向上传递 `-1`。

## 110 代码

```go
func isBalanced(root *TreeNode) bool {
    return getDepth(root) != -1
}

func getDepth(node *TreeNode) int {
    if node == nil {
        return 0
    }

    leftDepth := getDepth(node.Left)
    if leftDepth == -1 {
        return -1
    }

    rightDepth := getDepth(node.Right)
    if rightDepth == -1 {
        return -1
    }

    if abs(leftDepth-rightDepth) > 1 {
        return -1
    }

    return 1 + max(leftDepth, rightDepth)
}

func abs(x int) int {
    if x < 0 {
        return -x
    }
    return x
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

## 后序遍历含义

`getDepth` 的处理顺序是：

```text
左 -> 右 -> 中
```

也就是：

1. 先判断左子树。
2. 再判断右子树。
3. 最后根据左右子树深度判断当前节点。

这正好符合平衡二叉树的定义。

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(h)`

其中 `h` 是树高。

每个节点只会被访问一次。
递归栈空间最坏为 `O(n)`，平衡树中为 `O(log n)`。

## 本题总结

这道题的关键是避免重复计算高度。

自顶向下方法直观，但会反复计算子树深度。

自底向上方法把求高度和判断平衡合并到一次后序遍历中。
使用 `-1` 作为不平衡信号，可以让错误状态快速向上传递。

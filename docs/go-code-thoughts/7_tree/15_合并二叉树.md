# 合并二叉树：递归返回合并后的子树

合并二叉树是典型的递归构造问题。

递归函数不仅要向下处理左右子树，
还要向上返回合并后的子树根节点。

## 617 合并二叉树

[题目链接](https://leetcode.cn/problems/merge-two-binary-trees/)

### 617 题意

给定两棵二叉树 `root1` 和 `root2`。

将两棵树合并成一棵新二叉树：

- 如果两个节点重叠，新节点值为两个节点值之和。
- 如果只有一个节点存在，不为空的节点直接作为结果节点。
- 合并过程必须从根节点开始。

## 合并规则

对于任意两个节点 `node1` 和 `node2`，有四种情况：

1. 两个节点都为空，返回 `nil`。
2. 只有 `node1` 存在，返回 `node1`。
3. 只有 `node2` 存在，返回 `node2`。
4. 两个节点都存在，值相加，并递归合并左右子树。

## 原始探索中的错误

原始版本已经正确理解了递归返回值，
但在合并右子树时有一个笔误：

```go
root.Right = buildTree(root2.Right, root2.Right)
```

这里把 `root2.Right` 传了两次。

正确写法应该是：

```go
root.Right = buildTree(root1.Right, root2.Right)
```

这个错误的本质是：左右子树递归时，必须同时传入两棵树的对应位置。

## 最终思路

可以选择新建节点，也可以直接在 `root1` 上修改。

直接在 `root1` 上修改的写法更简洁：

1. 如果 `root1 == nil`，返回 `root2`。
2. 如果 `root2 == nil`，返回 `root1`。
3. 两者都不为空时，把 `root2.Val` 加到 `root1.Val`。
4. 递归合并左子树。
5. 递归合并右子树。
6. 返回 `root1`。

## 617 代码

```go
func mergeTrees(root1 *TreeNode, root2 *TreeNode) *TreeNode {
    if root1 == nil {
        return root2
    }
    if root2 == nil {
        return root1
    }

    root1.Val += root2.Val

    root1.Left = mergeTrees(root1.Left, root2.Left)
    root1.Right = mergeTrees(root1.Right, root2.Right)

    return root1
}
```

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(h)`

其中 `n` 是两棵树重叠和非重叠节点的总访问量，
`h` 是递归栈高度。

## 本题总结

合并二叉树的关键是理解递归返回值。

每一层递归都返回当前两棵子树合并后的根节点。
父节点接收这个返回值，并把它连接到自己的左孩子或右孩子位置。

如果选择在 `root1` 上原地修改，可以减少新建节点的空间开销。

# 二叉树的最近公共祖先：递归返回发现结果

最近公共祖先这道题的关键是理解递归函数的返回值。

`lowestCommonAncestor(root, p, q)` 不只是直接寻找 LCA。
它更像是在回答：

```text
在以 root 为根的子树中，能不能找到 p 或 q
```

## 236 二叉树的最近公共祖先

[题目链接](https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/)

### 236 题意

给定一棵二叉树，找到树中两个指定节点 `p` 和 `q`
的最近公共祖先。

最近公共祖先是一个节点 `x`：

- `x` 是 `p` 和 `q` 的祖先。
- `x` 的深度尽可能大。
- 一个节点可以是它自己的祖先。

## 递归返回值含义

对于当前子树，递归函数可能返回三类结果：

1. 如果没有找到 `p` 或 `q`，返回 `nil`。
2. 如果只找到了 `p` 或 `q` 中的一个，返回找到的那个节点。
3. 如果当前节点正好是最近公共祖先，返回当前节点。

这让父节点可以根据左右子树的返回结果做判断。

## 递归逻辑

### 终止条件

如果当前节点为空：

```go
return nil
```

如果当前节点就是 `p` 或 `q`：

```go
return root
```

这表示当前子树已经找到了一个目标节点。

### 汇总左右子树结果

递归搜索左右子树：

```go
leftResult := lowestCommonAncestor(root.Left, p, q)
rightResult := lowestCommonAncestor(root.Right, p, q)
```

如果左右子树都有返回值，说明 `p` 和 `q` 分别位于当前节点两侧。

此时当前节点就是最近公共祖先：

```go
return root
```

如果只有一侧有返回值，就把这一侧的结果向上返回。

如果两侧都没有返回值，返回 `nil`。

## 236 代码

```go
func lowestCommonAncestor(root, p, q *TreeNode) *TreeNode {
    if root == nil {
        return nil
    }

    if root == p || root == q {
        return root
    }

    leftResult := lowestCommonAncestor(root.Left, p, q)
    rightResult := lowestCommonAncestor(root.Right, p, q)

    if leftResult != nil && rightResult != nil {
        return root
    }

    if leftResult != nil {
        return leftResult
    }

    return rightResult
}
```

## 为什么这样能找到最近公共祖先

递归是自底向上返回结果。

当某个节点第一次同时从左子树和右子树收到非空结果时，
说明：

- 一个目标节点在它的左子树中。
- 另一个目标节点在它的右子树中。

这个节点就是两条路径第一次汇合的位置。
因此它是最近公共祖先。

如果两个目标都在同一侧，
当前节点只会收到一侧的非空返回值。
此时当前节点不是 LCA，只需要把结果继续向上传递。

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(h)`

其中 `h` 是树高。

## 本题总结

这道题的核心是递归返回值设计。

递归函数返回的不是简单的布尔值，
而是在当前子树中找到的重要节点。

最近公共祖先会自然地出现在第一个同时收到左右子树返回值的节点上。

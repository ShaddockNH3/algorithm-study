# 验证二叉搜索树：从局部比较到全局上下界

验证二叉搜索树时，不能只比较父节点和直接子节点。

BST 的要求是全局约束：

- 左子树所有节点都必须小于当前节点。
- 右子树所有节点都必须大于当前节点。

因此，每个节点都需要满足由祖先节点传下来的取值范围。

## 98 验证二叉搜索树

[题目链接](https://leetcode.cn/problems/validate-binary-search-tree/)

### 98 题意

给定二叉树根节点 `root`，判断它是否是有效的二叉搜索树。

有效 BST 定义：

- 节点左子树只包含严格小于当前节点的数。
- 节点右子树只包含严格大于当前节点的数。
- 所有左子树和右子树自身也必须是二叉搜索树。

## 局部比较的陷阱

最初容易想到只检查：

```text
root.Left.Val < root.Val < root.Right.Val
```

但这只保证局部正确，不保证全局正确。

例如某个节点 `6` 可能位于 `5` 的右子树中，
但又出现在右子树里某个节点的左侧。

它和直接父节点可能满足大小关系，
但它仍然必须满足祖先节点 `5` 的约束：

```text
所有位于 5 右子树中的节点都必须大于 5
```

所以必须把上下界一起传入递归。

## 方法一：上下界递归

### 上下界思路

定义辅助函数：

```go
isValid(node, lower, upper)
```

含义是：判断 `node` 这棵子树是否满足所有节点值都在开区间
`(lower, upper)` 内。

对当前节点：

1. 如果为空，返回 `true`。
2. 如果 `node.Val <= lower` 或 `node.Val >= upper`，返回 `false`。
3. 左子树范围更新为 `(lower, node.Val)`。
4. 右子树范围更新为 `(node.Val, upper)`。

注意：传给子树的新边界来自当前父节点 `node.Val`，
不是子节点自己的值。

### 上下界代码

```go
import "math"

func isValidBST(root *TreeNode) bool {
    return isValid(root, math.MinInt64, math.MaxInt64)
}

func isValid(node *TreeNode, lower int64, upper int64) bool {
    if node == nil {
        return true
    }

    value := int64(node.Val)
    if value <= lower || value >= upper {
        return false
    }

    return isValid(node.Left, lower, value) &&
        isValid(node.Right, value, upper)
}
```

### 边界说明

使用 `int64` 作为上下界更稳妥。

如果节点值刚好等于 `math.MinInt` 或 `math.MaxInt`，
用普通 `int` 边界并做严格比较时容易踩边界问题。

## 方法二：中序遍历

### 中序思路

BST 的中序遍历结果一定是严格递增序列。

因此可以在中序遍历过程中维护上一个访问节点值 `prev`。

如果当前值小于等于 `prev`，说明不是有效 BST。

### 中序代码

```go
import "math"

func isValidBSTInorder(root *TreeNode) bool {
    prev := int64(math.MinInt64)
    var inorder func(*TreeNode) bool

    inorder = func(node *TreeNode) bool {
        if node == nil {
            return true
        }

        if !inorder(node.Left) {
            return false
        }

        value := int64(node.Val)
        if value <= prev {
            return false
        }
        prev = value

        return inorder(node.Right)
    }

    return inorder(root)
}
```

## 复杂度

两种方法都是：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(h)`

其中 `h` 是树高。

## 本题总结

验证 BST 的关键是全局约束。

只比较当前节点和直接孩子是不够的。

常见正确做法有两种：

- 递归时携带上下界。
- 利用 BST 中序遍历严格递增的性质。

上下界法更直接表达了 BST 的定义，
中序法则利用了 BST 的遍历性质。

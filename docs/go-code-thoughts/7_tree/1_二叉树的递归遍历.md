# 二叉树的递归遍历：递归三要素

二叉树的前序、中序、后序遍历都可以用同一套递归方法理解。

写递归时先明确三件事：

1. 递归函数的参数和返回值。
2. 递归终止条件。
3. 单层递归逻辑。

这三点确定后，前中后序遍历只是在单层逻辑中调整访问顺序。

## 144 二叉树的前序遍历

[题目链接](https://leetcode.cn/problems/binary-tree-preorder-traversal/)

### 144 递归分析

前序遍历顺序是：

```text
中 -> 左 -> 右
```

递归函数需要当前节点 `node *TreeNode`。
为了收集结果，可以传入 `result *[]int`。

终止条件是：

```go
if node == nil {
    return
}
```

单层逻辑：

1. 先记录当前节点值。
2. 递归遍历左子树。
3. 递归遍历右子树。

### 144 代码

```go
func preorderTraversal(root *TreeNode) []int {
    result := []int{}
    traversal(root, &result)
    return result
}

func traversal(node *TreeNode, result *[]int) {
    if node == nil {
        return
    }

    *result = append(*result, node.Val)
    traversal(node.Left, result)
    traversal(node.Right, result)
}
```

## 94 二叉树的中序遍历

[题目链接](https://leetcode.cn/problems/binary-tree-inorder-traversal/)

### 94 递归分析

中序遍历顺序是：

```text
左 -> 中 -> 右
```

参数、返回值和终止条件都与前序遍历一致。
区别只在单层逻辑的顺序。

单层逻辑：

1. 递归遍历左子树。
2. 记录当前节点值。
3. 递归遍历右子树。

### 94 代码

```go
func inorderTraversal(root *TreeNode) []int {
    result := []int{}
    traversal(root, &result)
    return result
}

func traversal(node *TreeNode, result *[]int) {
    if node == nil {
        return
    }

    traversal(node.Left, result)
    *result = append(*result, node.Val)
    traversal(node.Right, result)
}
```

## 145 二叉树的后序遍历

[题目链接](https://leetcode.cn/problems/binary-tree-postorder-traversal/)

### 145 递归分析

后序遍历顺序是：

```text
左 -> 右 -> 中
```

参数、返回值和终止条件仍然不变。
只需要调整当前节点的访问时机。

单层逻辑：

1. 递归遍历左子树。
2. 递归遍历右子树。
3. 记录当前节点值。

### 145 代码

```go
func postorderTraversal(root *TreeNode) []int {
    result := []int{}
    traversal(root, &result)
    return result
}

func traversal(node *TreeNode, result *[]int) {
    if node == nil {
        return
    }

    traversal(node.Left, result)
    traversal(node.Right, result)
    *result = append(*result, node.Val)
}
```

## 本篇总结

二叉树递归遍历的核心不是背代码，而是先想清楚递归三要素。

三种遍历的差异只在访问当前节点的位置：

- 前序：先访问当前节点。
- 中序：左子树之后访问当前节点。
- 后序：左右子树都处理完后访问当前节点。

掌握这一点后，前中后序遍历就变成了同一套递归框架的三种排列。

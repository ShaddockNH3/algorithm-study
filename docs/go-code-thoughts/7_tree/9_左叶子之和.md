# 左叶子之和：站在父节点视角判断

左叶子之和的关键是先理解什么是左叶子。

一个节点要成为左叶子，必须同时满足：

1. 它是叶子节点。
2. 它是父节点的左孩子。

因此，一个节点自己只能判断自己是不是叶子，
但不能判断自己是不是左叶子。

左叶子必须由父节点来判断。

## 404 左叶子之和

[题目链接](https://leetcode.cn/problems/sum-of-left-leaves/)

### 404 题意

给定二叉树的根节点 `root`，
返回所有左叶子的值之和。

## 递归思路

递归函数 `sumOfLeftLeaves(root)` 的核心任务是：

```text
站在 root 节点上，判断 root 的孩子
```

对于当前节点 `root`，需要判断：

```go
root.Left != nil &&
    root.Left.Left == nil &&
    root.Left.Right == nil
```

这个条件表示：

1. 当前节点有左孩子。
2. 左孩子没有左孩子。
3. 左孩子没有右孩子。

也就是说，`root.Left` 是一个左叶子。

## 两种情况

### 左孩子是叶子节点

如果 `root.Left` 是左叶子，就把它的值加入结果。

左孩子本身已经是叶子，不需要继续向左递归。
但右子树中还可能存在其他左叶子，所以还要递归右子树。

```go
return root.Left.Val + sumOfLeftLeaves(root.Right)
```

### 左孩子不是叶子节点

如果当前节点没有直接找到左叶子，
就把任务继续交给左右子树。

```go
return sumOfLeftLeaves(root.Left) + sumOfLeftLeaves(root.Right)
```

## 404 代码

```go
func sumOfLeftLeaves(root *TreeNode) int {
    if root == nil {
        return 0
    }

    if root.Left != nil &&
        root.Left.Left == nil &&
        root.Left.Right == nil {
        return root.Left.Val + sumOfLeftLeaves(root.Right)
    }

    return sumOfLeftLeaves(root.Left) + sumOfLeftLeaves(root.Right)
}
```

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(h)`

其中 `h` 是树高，空间来自递归调用栈。

## 本题总结

这道题的核心不是判断当前节点，而是判断当前节点的左孩子。

很多树题都需要明确判断视角：

- 当前节点能判断什么。
- 父节点能判断什么。
- 子节点需要把什么信息返回给父节点。

左叶子必须站在父节点视角判断。

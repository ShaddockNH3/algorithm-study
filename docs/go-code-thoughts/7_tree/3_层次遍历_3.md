# 层序遍历专题三：最大深度与最小深度

最大深度和最小深度都可以用层序遍历解决。

区别在于：

- 最大深度需要遍历完整棵树。
- 最小深度遇到第一个叶子节点就可以返回。

## 104 二叉树的最大深度

[题目链接](https://leetcode.cn/problems/maximum-depth-of-binary-tree/)

### 104 题意

给定一个二叉树 `root`，返回它的最大深度。

二叉树的最大深度是从根节点到最远叶子节点的最长路径上的节点数。

### 104 思路

层序遍历会一层一层处理节点。

每处理完一层，深度加一。
当所有节点都处理完后，累计层数就是最大深度。

### 104 代码

```go
func maxDepth(root *TreeNode) int {
    if root == nil {
        return 0
    }

    depth := 0
    queue := []*TreeNode{root}

    for len(queue) > 0 {
        levelSize := len(queue)

        for i := 0; i < levelSize; i++ {
            node := queue[0]
            queue = queue[1:]

            if node.Left != nil {
                queue = append(queue, node.Left)
            }
            if node.Right != nil {
                queue = append(queue, node.Right)
            }
        }

        depth++
    }

    return depth
}
```

## 111 二叉树的最小深度

[题目链接](https://leetcode.cn/problems/minimum-depth-of-binary-tree/)

### 111 题意

给定一个二叉树，找出它的最小深度。

最小深度是从根节点到最近叶子节点的最短路径上的节点数量。

叶子节点是没有子节点的节点。

### 111 思路

层序遍历会按层从近到远处理节点。

因此，第一个遇到的叶子节点，一定在离根节点最近的那一层。

处理步骤：

1. 每进入一层，`depth++`。
2. 遍历当前层所有节点。
3. 如果发现某个节点没有左右孩子，说明它是叶子节点。
4. 立即返回当前 `depth`。

### 111 代码

```go
func minDepth(root *TreeNode) int {
    if root == nil {
        return 0
    }

    depth := 0
    queue := []*TreeNode{root}

    for len(queue) > 0 {
        levelSize := len(queue)
        depth++

        for i := 0; i < levelSize; i++ {
            node := queue[0]
            queue = queue[1:]

            if node.Left == nil && node.Right == nil {
                return depth
            }

            if node.Left != nil {
                queue = append(queue, node.Left)
            }
            if node.Right != nil {
                queue = append(queue, node.Right)
            }
        }
    }

    return depth
}
```

## 对比总结

最大深度：

- 目标是最远叶子节点。
- 必须遍历完整棵树。
- 每处理完一层，深度加一。

最小深度：

- 目标是最近叶子节点。
- 层序遍历中遇到第一个叶子即可返回。
- 可以提前结束。

## 本篇总结

层序遍历处理深度问题很直观。

它的优势是天然按层推进。

如果题目要求最大深度，就统计总层数。
如果题目要求最小深度，就在首次遇到叶子节点时返回当前层数。

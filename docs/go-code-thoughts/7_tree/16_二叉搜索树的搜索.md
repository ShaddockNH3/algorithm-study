# 二叉搜索树中的搜索：利用有序性导航

二叉搜索树和普通二叉树的区别在于有序性。

对于任意节点：

- 左子树所有节点值小于当前节点值。
- 右子树所有节点值大于当前节点值。

搜索时可以利用这个性质，每一步只进入一边子树。

## 700 二叉搜索树中的搜索

[题目链接](https://leetcode.cn/problems/search-in-a-binary-search-tree/)

### 700 题意

给定二叉搜索树的根节点 `root` 和整数 `val`。

在 BST 中找到节点值等于 `val` 的节点，
并返回以该节点为根的子树。

如果节点不存在，则返回 `nil`。

## 递归思路

递归搜索时有三种情况：

1. 当前节点为空，说明没有找到，返回 `nil`。
2. 当前节点值等于 `val`，返回当前节点。
3. 当前节点值不等于 `val`，根据 BST 性质选择方向。

如果 `val < root.Val`，只需要搜索左子树。

如果 `val > root.Val`，只需要搜索右子树。

## 递归代码

```go
func searchBST(root *TreeNode, val int) *TreeNode {
    return searchVal(root, val)
}

func searchVal(root *TreeNode, val int) *TreeNode {
    if root == nil {
        return nil
    }

    if root.Val == val {
        return root
    }

    if val < root.Val {
        return searchVal(root.Left, val)
    }

    return searchVal(root.Right, val)
}
```

## 迭代思路

迭代法用一个指针沿着树向下走。

每一步都根据当前节点值决定方向。

如果走到 `nil`，说明目标不存在。

## 迭代代码

```go
func searchBSTIterative(root *TreeNode, val int) *TreeNode {
    for root != nil {
        if root.Val == val {
            return root
        } else if val < root.Val {
            root = root.Left
        } else {
            root = root.Right
        }
    }

    return nil
}
```

## 复杂度

- 时间复杂度：平均 `O(log n)`，最坏 `O(n)`
- 空间复杂度：
  - 递归法：`O(h)`
  - 迭代法：`O(1)`

其中 `h` 是树高。

如果 BST 接近平衡，查找效率接近 `O(log n)`。
如果 BST 退化成链表，最坏会变成 `O(n)`。

## 本题总结

BST 搜索的核心是利用有序性。

普通二叉树搜索可能需要遍历整棵树。
BST 每次比较后都能排除一边子树，
因此查找路径只会沿着一个方向向下延伸。

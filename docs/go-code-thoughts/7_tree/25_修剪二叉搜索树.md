# 669 修剪二叉搜索树

[题目链接](https://leetcode.cn/problems/trim-a-binary-search-tree/)

## 题目描述

给定一个二叉搜索树，同时给定最小边界 `low` 和最大边界 `high`。
通过修剪二叉搜索树，使得所有节点的值都在 `[low, high]` 中。

修剪过程可能会改变树的根节点，
所以结果应当返回修剪好的二叉搜索树的新根节点。

## 递归函数的约定

这道题也需要先明确递归函数的含义：

`trimBST(root, low, high)` 会把以 `root` 为根的树修剪好，
使其中所有节点值都落在 `[low, high]` 范围内，
并返回修剪之后这棵子树的新根节点。

理解这个返回值非常重要。
一旦当前节点不符合范围，当前子树的新根就可能不再是 `root`。

## 当前节点太小

如果 `root.Val < low`，当前节点必须被删掉。

因为这是 BST，当前节点的整个左子树都小于当前节点，
所以左子树也一定全部小于 `low`，可以整体舍弃。

此时答案只可能来自右子树。
所以直接返回右子树修剪后的新根节点：

```go
return trimBST(root.Right, low, high)
```

这里不需要自己去右子树里找新根。
递归函数的约定已经保证它会返回修剪后的新根。

## 当前节点太大

如果 `root.Val > high`，当前节点必须被删掉。

由于 BST 的右子树都大于当前节点，
整个右子树也一定全部大于 `high`，可以整体舍弃。

此时答案只可能来自左子树：

```go
return trimBST(root.Left, low, high)
```

## 当前节点合格

如果 `root.Val` 在 `[low, high]` 范围内，
当前节点可以继续作为这棵子树的根。

不过它的左右子树仍然可能包含不合格的节点，
所以要继续修剪左右子树，并把返回结果重新接回当前节点：

```go
root.Left = trimBST(root.Left, low, high)
root.Right = trimBST(root.Right, low, high)
return root
```

## 最终代码

```go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func trimBST(root *TreeNode, low int, high int) *TreeNode {
    if root == nil {
        return nil
    }

    if root.Val < low {
        return trimBST(root.Right, low, high)
    }

    if root.Val > high {
        return trimBST(root.Left, low, high)
    }

    root.Left = trimBST(root.Left, low, high)
    root.Right = trimBST(root.Right, low, high)

    return root
}
```

## 本篇总结

这道题最容易卡住的地方是递归的返回值。

每次递归返回的不是简单的处理结果，
而是当前子树修剪之后的新根节点。

只要相信并使用这个约定，
当前节点过小、过大、合格三种情况就都能自然写出来。

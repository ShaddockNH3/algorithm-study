# 树的相同与子树判断

这一篇把两道树比较问题放在一起：

- 100 相同的树。
- 572 另一棵树的子树。

它们的核心都是比较两棵树的结构和节点值。

## 100 相同的树

[题目链接](https://leetcode.cn/problems/same-tree/)

### 100 题意

给定两棵二叉树的根节点 `p` 和 `q`，
判断这两棵树是否相同。

两棵树相同，要求结构相同，并且对应节点的值相同。

### 100 迭代思路

这道题和对称二叉树类似，都是成对比较节点。

区别在于：

- 对称二叉树要交叉比较。
- 相同的树要直接比较。

因此入队时比较：

```text
p.Left  和 q.Left
p.Right 和 q.Right
```

### 100 迭代代码

```go
func isSameTree(p *TreeNode, q *TreeNode) bool {
    if p == nil && q == nil {
        return true
    }
    if p == nil || q == nil {
        return false
    }

    queue := []*TreeNode{p, q}

    for len(queue) > 0 {
        node1 := queue[0]
        node2 := queue[1]
        queue = queue[2:]

        if node1 == nil && node2 == nil {
            continue
        }
        if node1 == nil || node2 == nil || node1.Val != node2.Val {
            return false
        }

        queue = append(queue, node1.Left)
        queue = append(queue, node2.Left)
        queue = append(queue, node1.Right)
        queue = append(queue, node2.Right)
    }

    return true
}
```

### 100 递归思路

两棵树相同，当且仅当：

1. 根节点值相同。
2. 左子树相同。
3. 右子树相同。

### 100 递归代码

```go
func isSameTreeRecursive(p *TreeNode, q *TreeNode) bool {
    if p == nil && q == nil {
        return true
    }

    if p == nil || q == nil || p.Val != q.Val {
        return false
    }

    return isSameTreeRecursive(p.Left, q.Left) &&
        isSameTreeRecursive(p.Right, q.Right)
}
```

## 572 另一棵树的子树

[题目链接](https://leetcode.cn/problems/subtree-of-another-tree/)

### 572 题意

给定两棵二叉树 `root` 和 `subRoot`。

判断 `root` 中是否包含一棵与 `subRoot` 结构相同、
节点值相同的子树。

## 方法一：遍历 root 并调用 isSameTree

### 朴素思路

遍历大树 `root` 的每一个节点。

把每个节点都当作一个潜在的子树根节点，
然后调用 `isSameTree(node, subRoot)` 判断是否完全相同。

只要有一个节点匹配成功，就返回 `true`。

### 朴素代码

```go
func isSubtree(root *TreeNode, subRoot *TreeNode) bool {
    if root == nil {
        return subRoot == nil
    }

    queue := []*TreeNode{root}

    for len(queue) > 0 {
        node := queue[0]
        queue = queue[1:]

        if isSameTree(node, subRoot) {
            return true
        }

        if node.Left != nil {
            queue = append(queue, node.Left)
        }
        if node.Right != nil {
            queue = append(queue, node.Right)
        }
    }

    return false
}
```

这个方法直观，但最坏时间复杂度约为 `O(m * n)`。

## 方法二：序列化与字符串匹配

### 序列化思路

也可以把树比较问题转成字符串匹配问题。

做法：

1. 用带空节点标记的前序遍历序列化 `root`。
2. 用同样方式序列化 `subRoot`。
3. 判断 `subRoot` 的序列化结果是否是 `root` 的子串。

必须保留空节点标记和分隔符。
否则不同结构的树可能被序列化成相同字符串。

### 序列化代码

```go
import (
    "strconv"
    "strings"
)

func getString(root *TreeNode) string {
    if root == nil {
        return ",#,"
    }

    var builder strings.Builder
    stack := []*TreeNode{root}

    for len(stack) > 0 {
        node := stack[len(stack)-1]
        stack = stack[:len(stack)-1]

        if node == nil {
            builder.WriteString(",#,")
            continue
        }

        builder.WriteString("," + strconv.Itoa(node.Val) + ",")
        stack = append(stack, node.Right)
        stack = append(stack, node.Left)
    }

    return builder.String()
}

func isSubtree(root *TreeNode, subRoot *TreeNode) bool {
    if subRoot == nil {
        return true
    }
    if root == nil {
        return false
    }

    rootStr := getString(root)
    subRootStr := getString(subRoot)

    return strings.Contains(rootStr, subRootStr)
}
```

字符串匹配可以使用 Go 内置的 `strings.Contains`。
如果要进一步分析理论复杂度，也可以联想到 KMP。

## 方法三：递归查找

### 递归思路

`subRoot` 是 `root` 的子树，当且仅当以下任意一种情况成立：

1. `root` 和 `subRoot` 本身是相同的树。
2. `subRoot` 是 `root.Left` 的子树。
3. `subRoot` 是 `root.Right` 的子树。

### 递归代码

```go
func isSubtreeRecursive(root *TreeNode, subRoot *TreeNode) bool {
    if root == nil {
        return subRoot == nil
    }

    if isSameTree(root, subRoot) {
        return true
    }

    return isSubtreeRecursive(root.Left, subRoot) ||
        isSubtreeRecursive(root.Right, subRoot)
}
```

## 本篇总结

相同的树是子树判断的基础。

先掌握 `isSameTree`，再解决 `isSubtree` 会自然很多。

`isSubtree` 有三种常见思路：

- 遍历每个节点，再调用 `isSameTree`。
- 序列化两棵树，再做字符串匹配。
- 用递归表达当前相同、左子树包含、右子树包含三种情况。

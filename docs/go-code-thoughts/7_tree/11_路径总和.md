# 路径总和：回溯判断根到叶路径

路径总和要求判断是否存在一条从根节点到叶子节点的路径，
使路径上所有节点值之和等于目标值。

这道题适合用递归回溯。

## 112 路径总和

[题目链接](https://leetcode.cn/problems/path-sum/)

### 112 题意

给定二叉树的根节点 `root` 和整数 `targetSum`。

判断树中是否存在一条从根节点到叶子节点的路径，
这条路径上所有节点值之和等于 `targetSum`。

如果存在，返回 `true`；否则返回 `false`。

叶子节点是没有子节点的节点。

## 回溯思路

递归过程中维护当前路径和 `currentSum`。

每进入一个节点：

1. 把当前节点值加入 `currentSum`。
2. 如果当前节点是叶子节点，就判断路径和是否等于目标值。
3. 继续递归探索左右子树。
4. 离开当前节点前，把当前节点值从 `currentSum` 中减掉。

最后一步就是回溯。
它保证返回父节点后，状态恢复到进入当前节点之前。

## Go 递归闭包写法

如果递归函数定义为闭包，需要先声明，再赋值。

```go
var backtracking func(*TreeNode)

backtracking = func(node *TreeNode) {
    // 函数体中才能递归调用 backtracking
}
```

这样可以避免闭包内部引用尚未定义的函数变量。

## 112 代码

```go
func hasPathSum(root *TreeNode, targetSum int) bool {
    res := false
    currentSum := 0

    var backtracking func(*TreeNode)

    backtracking = func(node *TreeNode) {
        if node == nil || res {
            return
        }

        currentSum += node.Val

        if node.Left == nil &&
            node.Right == nil &&
            currentSum == targetSum {
            res = true
        }

        backtracking(node.Left)
        backtracking(node.Right)

        currentSum -= node.Val
    }

    backtracking(root)
    return res
}
```

## 关于 path 变量

原始思路中也曾记录具体路径。

但这道题只需要返回 `true` 或 `false`，
不需要输出路径本身。

因此最终代码只保留 `currentSum`，
让逻辑更聚焦。

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(h)`

其中 `h` 是树高，空间来自递归调用栈。

## 本题总结

路径总和的判断必须发生在叶子节点。

中间节点即使当前路径和等于 `targetSum`，
也不能直接返回成功，因为题目要求路径必须到叶子节点。

回溯的关键是状态恢复：

```go
currentSum += node.Val
// 递归探索
currentSum -= node.Val
```

这样才能保证探索兄弟分支时，路径和是干净的。

# 路径总和 II：树形回溯与路径复制

路径总和 II 要返回所有符合条件的路径。

相比 112 路径总和，这道题不只判断是否存在，
还要保存每一条从根节点到叶子节点的路径。

## 113 路径总和 II

[题目链接](https://leetcode.cn/problems/path-sum-ii/)

### 113 题意

给定二叉树的根节点 `root` 和整数 `targetSum`，
找出所有从根节点到叶子节点路径总和等于 `targetSum` 的路径。

叶子节点是没有子节点的节点。

## 回溯结构辨析

原始记录中有一个重要总结：

```text
也就是说，是有没有 for 的问题
```

这句话对应的是两类回溯结构的差异。

### 组合类回溯

组合、排列、子集问题通常有显式的选择列表。

它们常见结构是：

```text
if 满足收集条件 {
    收集结果
    return
}

for 每个选择 {
    做选择
    递归
    撤销选择
}
```

这里的回溯操作在 `for` 循环内部。

终止条件里的 `return` 是安全的，
因为撤销选择由调用者所在的那一层 `for` 负责。

### 树路径回溯

树路径问题通常没有显式的 `for` 循环。

它的结构更像：

```text
做选择

if 是叶子且满足条件 {
    收集结果
}

递归左子树
递归右子树

撤销选择
```

这里的做选择和撤销选择包住了整个函数主体。

如果在收集结果后直接 `return`，
函数末尾的撤销选择就会被跳过，路径状态会出错。

因此，树形回溯中要特别注意：

```text
有借有还
```

只要 `append` 了当前节点，函数结束前就必须把它弹出。

## 113 代码

```go
func pathSum(root *TreeNode, targetSum int) [][]int {
    res := [][]int{}
    path := []int{}

    var backtracking func(*TreeNode, int)

    backtracking = func(node *TreeNode, currentSum int) {
        if node == nil {
            return
        }

        path = append(path, node.Val)
        currentSum += node.Val

        if node.Left == nil &&
            node.Right == nil &&
            currentSum == targetSum {
            temp := make([]int, len(path))
            copy(temp, path)
            res = append(res, temp)
        }

        backtracking(node.Left, currentSum)
        backtracking(node.Right, currentSum)

        path = path[:len(path)-1]
    }

    backtracking(root, 0)
    return res
}
```

## 为什么要复制 path

保存答案时，不能直接写：

```go
res = append(res, path)
```

因为 `path` 后续还会继续被修改。
如果直接保存切片，结果中的多条路径可能共享同一个底层数组，
最终内容会被后续回溯过程影响。

正确做法是创建副本：

```go
temp := make([]int, len(path))
copy(temp, path)
res = append(res, temp)
```

## currentSum 作为参数

`currentSum` 作为参数传递，而不是外部变量。

这样每次递归调用拿到的是自己的路径和。
函数返回后，上一层的 `currentSum` 自然保持原值，
不需要手动执行减法回溯。

但 `path` 是外部共享切片，
所以必须手动弹出最后一个节点。

## 复杂度

- 时间复杂度：`O(n * h)`
- 空间复杂度：`O(h)`

其中 `n` 是节点数，`h` 是树高。

保存结果时需要复制路径，因此时间复杂度和路径长度有关。

## 本题总结

路径总和 II 的关键有三点：

1. 只有根到叶路径才能作为答案。
2. 保存路径时必须复制 `path`。
3. 树形回溯中，收集结果后不能跳过末尾的撤销选择。

组合类回溯常有 `for` 循环包住选择和撤销。
树路径回溯则通常由函数本身完成选择和撤销。

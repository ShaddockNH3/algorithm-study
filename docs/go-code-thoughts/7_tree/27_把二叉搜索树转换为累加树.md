# 538 把二叉搜索树转换为累加树

[题目链接](https://leetcode.cn/problems/convert-bst-to-greater-tree/)

## 题目描述

给出二叉搜索树的根节点，
请将其转换为累加树。

转换后，每个节点 `node` 的新值等于原树中
大于或等于 `node.Val` 的所有节点值之和。

## 核心思路

题眼是大于或等于当前节点的值。

BST 的中序遍历顺序是：

```text
左 -> 根 -> 右
```

这个顺序可以得到升序序列。

如果想从大到小访问节点，就把中序遍历反过来：

```text
右 -> 根 -> 左
```

在反向中序遍历过程中，维护一个累加值 `sum`。
每访问一个节点，就先把当前节点的原值加入 `sum`，
再把当前节点值更新成 `sum`。

因为访问顺序是从大到小，
所以 `sum` 正好表示所有大于或等于当前节点的值之和。

## 值传递带来的问题

一开始容易尝试把 `sum` 作为参数传给递归函数：

```go
func NInorder(root *TreeNode, sum int) {
    if root == nil {
        return
    }

    NInorder(root.Right, sum)
    sum += root.Val
    root.Val = sum
    NInorder(root.Left, sum)
}
```

这个写法不对。

Go 里 `int` 是值类型，作为参数传递时传的是一份拷贝。
右子树递归里对 `sum` 的修改，
不会自动影响外层递归中的 `sum`。

这样累加状态就不能在整棵树的遍历过程中共享。

## 使用闭包共享状态

解决方式是把 `sum` 定义在外层函数 `convertBST` 中，
再把辅助递归函数也定义在 `convertBST` 中。

这样递归函数形成闭包，可以直接访问并修改外层的同一个 `sum`。
所有递归调用共享这一份累加状态，
反向中序遍历才能连续累加。

## 最终代码

```go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func convertBST(root *TreeNode) *TreeNode {
    sum := 0

    var NInorder func(*TreeNode)
    NInorder = func(node *TreeNode) {
        if node == nil {
            return
        }

        NInorder(node.Right)

        sum += node.Val
        node.Val = sum

        NInorder(node.Left)
    }

    NInorder(root)
    return root
}
```

## 本篇总结

累加树可以看作反向中序遍历中的前缀和问题。

BST 的中序遍历是升序，
反向中序遍历就是降序。

按降序访问节点时维护共享的 `sum`，
就能在访问当前节点时得到所有大于或等于当前节点的值之和。

在 Go 里，如果想让递归过程共享一个累加状态，
闭包是一种很自然的写法。

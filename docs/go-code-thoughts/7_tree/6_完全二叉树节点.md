# 完全二叉树的节点个数：利用满二叉树剪枝

完全二叉树的节点计数，不能只把它当普通二叉树遍历。

题目给出的完全二叉树条件，可以帮助我们用满二叉树公式剪枝。

## 222 完全二叉树的节点个数

[题目链接](https://leetcode.cn/problems/count-complete-tree-nodes/)

### 222 题意

给定一棵完全二叉树的根节点 `root`，
求出这棵树的节点个数。

## 阶段一：普通遍历

### 普通遍历思路

最直接的做法是把它当成普通二叉树。

使用 DFS 或 BFS 遍历所有节点，逐个计数。

### 普通遍历代码

```go
func countNodes(root *TreeNode) int {
    if root == nil {
        return 0
    }

    stack := []*TreeNode{root}
    sum := 0

    for len(stack) != 0 {
        node := stack[len(stack)-1]
        stack = stack[:len(stack)-1]

        if node.Right != nil {
            stack = append(stack, node.Right)
        }
        if node.Left != nil {
            stack = append(stack, node.Left)
        }

        sum++
    }

    return sum
}
```

这个方法完全正确，但时间复杂度是 `O(n)`。
它没有利用完全二叉树这个条件。

## 阶段二：尝试使用满二叉树公式

完全二叉树中，很多子树可能是满二叉树。

如果一棵子树是满二叉树，并且高度为 `h`，
那么节点数可以直接计算：

```text
2^h - 1
```

在 Go 中可以写成：

```go
(1 << h) - 1
```

原始探索中已经意识到：

1. 可以先判断子树是否为满二叉树。
2. 如果是，就直接套公式。
3. 如果不是，再继续递归处理左右子树。

真正需要打磨的是：如何准确判断当前子树是否是满二叉树。

## 最终思路

对于以 `root` 为根的子树：

1. 从 `root` 一直向左走，计算 `leftHeight`。
2. 从 `root` 一直向右走，计算 `rightHeight`。
3. 如果两者相等，说明当前子树是满二叉树。
4. 直接返回 `(1 << leftHeight) - 1`。
5. 如果不相等，递归计算左右子树节点数。

注意：每一轮递归都只判断当前子树是否为满二叉树。

## 222 代码

```go
func countNodes(root *TreeNode) int {
    if root == nil {
        return 0
    }

    leftNode, rightNode := root, root
    leftHeight, rightHeight := 0, 0

    for leftNode != nil {
        leftHeight++
        leftNode = leftNode.Left
    }

    for rightNode != nil {
        rightHeight++
        rightNode = rightNode.Right
    }

    if leftHeight == rightHeight {
        return (1 << leftHeight) - 1
    }

    return 1 + countNodes(root.Left) + countNodes(root.Right)
}
```

## 复杂度

- 时间复杂度：`O((log n)^2)`
- 空间复杂度：`O(log n)`

每次计算左右高度需要 `O(log n)`。
递归过程中会不断剪掉满二叉树子树，
因此整体复杂度优于普通遍历。

## 本题总结

这道题的关键是利用完全二叉树的结构。

如果当前子树的最左深度和最右深度相同，
说明它是满二叉树，可以直接用公式计算节点数。

如果不是满二叉树，再递归处理左右子树。

这个过程体现了从暴力遍历到结构剪枝的优化思路。

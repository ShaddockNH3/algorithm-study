# 二叉搜索树中的众数：中序遍历中的连续计数

BST 的中序遍历结果是有序序列。

如果树中有重复值，相同的值会在中序序列中连续出现。

因此，求众数时不一定需要哈希表。
可以在中序遍历过程中统计连续相同值的出现次数。

## 501 二叉搜索树中的众数

[题目链接](https://leetcode.cn/problems/find-mode-in-binary-search-tree/)

### 501 题意

给定一个含重复值的二叉搜索树根节点 `root`，
找出并返回 BST 中的所有众数。

众数是出现频率最高的元素。

如果有多个众数，可以按任意顺序返回。

## 变量含义

中序遍历过程中需要维护四个状态：

- `prev *TreeNode`：
  上一个访问过的节点。
- `currentCount int`：
  当前值已经连续出现的次数。
- `maxCount int`：
  当前发现的最高频次。
- `result []int`：
  所有达到最高频次的节点值。

`prev` 初始化为 `nil`，
可以自然处理第一个访问节点。

## 结算逻辑

当遇到新值时，说明上一段连续值已经结束。

此时要先结算上一段连续值：

1. 如果 `currentCount > maxCount`，
   更新 `maxCount`，并重置结果数组。
2. 如果 `currentCount == maxCount`，
   把上一段的值加入结果数组。

遍历结束后，还要对最后一段连续值做一次最终结算。

## 501 代码

```go
func findMode(root *TreeNode) []int {
    var prev *TreeNode
    currentCount := 0
    maxCount := 0
    result := []int{}

    var updateResult func()
    updateResult = func() {
        if currentCount > maxCount {
            maxCount = currentCount
            result = []int{prev.Val}
        } else if currentCount == maxCount {
            result = append(result, prev.Val)
        }
    }

    var inorder func(*TreeNode)
    inorder = func(node *TreeNode) {
        if node == nil {
            return
        }

        inorder(node.Left)

        if prev != nil && node.Val == prev.Val {
            currentCount++
        } else {
            if prev != nil {
                updateResult()
            }
            currentCount = 1
        }

        prev = node

        inorder(node.Right)
    }

    inorder(root)
    updateResult()

    return result
}
```

## 执行追踪

以树 `[2, 1, 2]` 为例，中序序列是：

```text
1, 2, 2
```

初始状态：

- `prev = nil`
- `currentCount = 0`
- `maxCount = 0`
- `result = []`

访问 `1`：

- `prev == nil`，说明这是第一个值。
- 不结算。
- `currentCount = 1`。
- `prev` 更新为节点 `1`。

访问第一个 `2`：

- `2 != 1`，说明遇到新值。
- 先结算上一段值 `1`。
- `currentCount = 1 > maxCount = 0`。
- 更新 `maxCount = 1`，`result = [1]`。
- 新值 `2` 开始计数，`currentCount = 1`。
- `prev` 更新为当前节点 `2`。

访问第二个 `2`：

- 当前值和 `prev` 相同。
- `currentCount++`，变成 `2`。
- `prev` 更新为新的节点 `2`。

遍历结束：

- 最后一段值 `2` 还没有结算。
- 执行最终结算。
- `currentCount = 2 > maxCount = 1`。
- 更新 `maxCount = 2`，`result = [2]`。

最终返回 `[2]`。

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(h)`

其中 `h` 是树高。

## 本题总结

BST 众数问题的关键是利用中序遍历的有序性。

相同值会连续出现，
所以只需要比较当前节点和前一个节点。

最容易漏掉的是最后一段连续值。
遍历结束后必须再执行一次最终结算。

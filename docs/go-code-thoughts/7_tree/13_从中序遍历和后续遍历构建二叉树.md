# 根据遍历序列构造二叉树

根据两种遍历序列构造二叉树，核心是两份信息互补。

- 前序或后序遍历负责确定当前子树的根节点。
- 中序遍历负责根据根节点位置划分左子树和右子树。

整个过程可以概括为：

```text
找根 -> 在 inorder 中划分左右区间 -> 递归构造
```

## 核心思想

前序遍历结构：

```text
根 -> 左 -> 右
```

后序遍历结构：

```text
左 -> 右 -> 根
```

中序遍历结构：

```text
左 -> 根 -> 右
```

因此：

- 前序的当前第一个元素是根。
- 后序的当前最后一个元素是根。
- 中序中根节点左侧属于左子树，右侧属于右子树。

递归函数中的左右边界，表示当前子树在 `inorder` 中的有效范围。
每次找到根节点后，都会把这个范围继续切成左右两段。

## 106 中序与后序构造二叉树

[题目链接](https://leetcode.cn/problems/construct-binary-tree-from-inorder-and-postorder-traversal/)

### 106 思路

后序遍历是：

```text
左 -> 右 -> 根
```

如果从后往前读，就变成：

```text
根 -> 右 -> 左
```

因此构造时要先构造右子树，再构造左子树。

处理步骤：

1. 用哈希表记录 `inorder` 中每个值的位置。
2. 用指针 `k` 指向 `postorder` 末尾。
3. 每次取 `postorder[k]` 作为当前根节点。
4. 在 `inorder` 中找到根节点位置 `index`。
5. 先递归构造右子树。
6. 再递归构造左子树。

### 106 代码

```go
func buildTree(inorder []int, postorder []int) *TreeNode {
    mp := make(map[int]int)
    for index, num := range inorder {
        mp[num] = index
    }

    k := len(inorder) - 1

    var getTree func(int, int) *TreeNode
    getTree = func(inorderLeft int, inorderRight int) *TreeNode {
        if inorderLeft > inorderRight {
            return nil
        }

        val := postorder[k]
        k--

        root := &TreeNode{Val: val}
        index := mp[val]

        root.Right = getTree(index+1, inorderRight)
        root.Left = getTree(inorderLeft, index-1)

        return root
    }

    return getTree(0, len(inorder)-1)
}
```

### 106 关键点

`postorder` 从后往前读时，根后面紧接着的是右子树的根。

所以必须先构造右子树。

如果先构造左子树，`k` 指针会拿错节点。

## 105 前序与中序构造二叉树

[题目链接](https://leetcode.cn/problems/construct-binary-tree-from-preorder-and-inorder-traversal/)

### 105 思路

前序遍历是：

```text
根 -> 左 -> 右
```

从前往后读，顺序本来就是：

```text
根 -> 左 -> 右
```

因此构造时先构造左子树，再构造右子树。

处理步骤：

1. 用哈希表记录 `inorder` 中每个值的位置。
2. 用指针 `k` 从 `preorder` 开头向后移动。
3. 每次取 `preorder[k]` 作为当前根节点。
4. 在 `inorder` 中找到根节点位置。
5. 先递归构造左子树。
6. 再递归构造右子树。

### 105 代码

```go
func buildTree(preorder []int, inorder []int) *TreeNode {
    mp := make(map[int]int)
    for index, num := range inorder {
        mp[num] = index
    }

    k := 0

    var build func(int, int) *TreeNode
    build = func(inLeft int, inRight int) *TreeNode {
        if inLeft > inRight {
            return nil
        }

        val := preorder[k]
        k++

        root := &TreeNode{Val: val}
        index := mp[val]

        root.Left = build(inLeft, index-1)
        root.Right = build(index+1, inRight)

        return root
    }

    return build(0, len(inorder)-1)
}
```

## 两题对比

中序加后序：

- 根在 `postorder` 的末尾。
- 读取方向是从右向左，`k--`。
- 构造顺序是根、右、左。

中序加前序：

- 根在 `preorder` 的开头。
- 读取方向是从左向右，`k++`。
- 构造顺序是根、左、右。

共同点：

- 都用 `inorder` 划分左右子树范围。
- 都用哈希表加速根节点位置查询。
- 都通过递归边界控制当前子树范围。

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

哈希表存储 `inorder` 中每个值的位置。
递归调用栈最坏可能达到 `O(n)`。

## 本题总结

构造二叉树时，要先判断哪份遍历序列能提供根节点。

然后在中序遍历中找到根节点的位置，
把当前子树范围分成左子树和右子树。

前序和后序的差别，只是根节点读取方向不同，
从而导致左右子树构造顺序不同。

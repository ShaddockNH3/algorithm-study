# 二叉树层序遍历专题一：队列模板与二次加工

层序遍历的核心数据结构是队列。

队列先进先出的特性，可以保证节点按层从左到右被处理。

这一组题可以先用通用 `levelOrder` 得到按层分组的结果，
再根据题目要求进行二次加工。

更高效的写法，是在层序遍历过程中直接完成加工。

## 层序遍历模板

通用模板的关键是记录当前层的节点数量 `levelSize`。

只处理当前层的 `levelSize` 个节点，
就能把不同层的节点分开。

```go
func levelOrder(root *TreeNode) [][]int {
    if root == nil {
        return [][]int{}
    }

    ans := [][]int{}
    queue := []*TreeNode{root}

    for len(queue) > 0 {
        levelSize := len(queue)
        levelAns := []int{}

        for i := 0; i < levelSize; i++ {
            node := queue[0]
            queue = queue[1:]

            levelAns = append(levelAns, node.Val)

            if node.Left != nil {
                queue = append(queue, node.Left)
            }
            if node.Right != nil {
                queue = append(queue, node.Right)
            }
        }

        ans = append(ans, levelAns)
    }

    return ans
}
```

## 102 二叉树的层序遍历

每一层单独收集成一个 `[]int`，

## 107 二叉树的层序遍历 II

[题目链接](https://leetcode.cn/problems/binary-tree-level-order-traversal-ii/)

### 107 思路

先执行标准层序遍历，得到从上到下的结果。

然后将整个结果反转，得到自底向上的层序结果。

在 Go 中，向切片头部插入元素效率较低。
因此先正序遍历再整体反转，是更自然的写法。

### 107 代码

```go
import "slices"

func levelOrderBottom(root *TreeNode) [][]int {
    ans := levelOrder(root)
    slices.Reverse(ans)
    return ans
}
```

## 199 二叉树的右视图

[题目链接](https://leetcode.cn/problems/binary-tree-right-side-view/)

### 199 原始思路

先执行标准层序遍历。

然后取每一层最后一个元素。

```go
func rightSideView(root *TreeNode) []int {
    levels := levelOrder(root)
    ans := []int{}

    for _, level := range levels {
        ans = append(ans, level[len(level)-1])
    }

    return ans
}
```

### 199 遍历时加工

也可以在层序遍历过程中，直接记录每层最后一个节点。

```go
func rightSideView(root *TreeNode) []int {
    if root == nil {
        return []int{}
    }

    ans := []int{}
    queue := []*TreeNode{root}

    for len(queue) > 0 {
        levelSize := len(queue)

        for i := 0; i < levelSize; i++ {
            node := queue[0]
            queue = queue[1:]

            if i == levelSize-1 {
                ans = append(ans, node.Val)
            }

            if node.Left != nil {
                queue = append(queue, node.Left)
            }
            if node.Right != nil {
                queue = append(queue, node.Right)
            }
        }
    }

    return ans
}
```

## 637 二叉树的层平均值

[题目链接](https://leetcode.cn/problems/average-of-levels-in-binary-tree/)

### 637 原始思路

先层序遍历得到每一层的节点值。

然后对每一层单独计算平均值。

### 637 遍历时加工

也可以在遍历每一层时，直接维护当前层总和。

```go
func averageOfLevels(root *TreeNode) []float64 {
    if root == nil {
        return []float64{}
    }

    ans := []float64{}
    queue := []*TreeNode{root}

    for len(queue) > 0 {
        levelSize := len(queue)
        levelSum := 0.0

        for i := 0; i < levelSize; i++ {
            node := queue[0]
            queue = queue[1:]
            levelSum += float64(node.Val)

            if node.Left != nil {
                queue = append(queue, node.Left)
            }
            if node.Right != nil {
                queue = append(queue, node.Right)
            }
        }

        ans = append(ans, levelSum/float64(levelSize))
    }

    return ans
}
```

## 429 N 叉树的层序遍历

[题目链接](https://leetcode.cn/problems/n-ary-tree-level-order-traversal/)

### 429 思路

N 叉树的层序遍历和二叉树类似。

区别是：

- 二叉树只需要处理 `Left` 和 `Right`。
- N 叉树需要遍历 `Children` 中的所有孩子。

Go 中可以用 `...` 将所有孩子展开后追加到队列中。

### 429 代码

```go
func levelOrder(root *Node) [][]int {
    if root == nil {
        return [][]int{}
    }

    ans := [][]int{}
    queue := []*Node{root}

    for len(queue) != 0 {
        levelSize := len(queue)
        levelAns := []int{}

        for i := 0; i < levelSize; i++ {
            node := queue[0]
            queue = queue[1:]
            levelAns = append(levelAns, node.Val)
            queue = append(queue, node.Children...)
        }

        ans = append(ans, levelAns)
    }

    return ans
}
```

## 515 在每个树行中找最大值

[题目链接](https://leetcode.cn/problems/find-largest-value-in-each-tree-row/)

### 515 原始思路

先执行标准层序遍历。

然后对每一层的节点值计算最大值。

### 515 遍历时加工

也可以在遍历每一层时，维护一个 `levelMax`。

```go
import "math"

func largestValues(root *TreeNode) []int {
    if root == nil {
        return []int{}
    }

    ans := []int{}
    queue := []*TreeNode{root}

    for len(queue) > 0 {
        levelSize := len(queue)
        levelMax := math.MinInt64

        for i := 0; i < levelSize; i++ {
            node := queue[0]
            queue = queue[1:]

            if node.Val > levelMax {
                levelMax = node.Val
            }

            if node.Left != nil {
                queue = append(queue, node.Left)
            }
            if node.Right != nil {
                queue = append(queue, node.Right)
            }
        }

        ans = append(ans, levelMax)
    }

    return ans
}
```

## 本篇总结

层序遍历的通用模板非常重要。

可以先写一个通用 `levelOrder`，再做二次加工。
这种方式模块清晰，适合学习和复用。

如果追求更少的中间存储，可以在遍历每一层时顺手完成加工：

- 右视图：记录每层最后一个节点。
- 层平均值：维护每层总和。
- 每行最大值：维护每层最大值。

本质上，这些题都依赖同一个层序遍历框架。

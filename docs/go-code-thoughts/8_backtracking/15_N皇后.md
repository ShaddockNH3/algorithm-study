# 51 N 皇后

[题目链接](https://leetcode.cn/problems/n-queens/)

## 题目描述

按照国际象棋的规则，
皇后可以攻击与之处在同一行、同一列或同一斜线上的棋子。

`n` 皇后问题研究的是如何将 `n` 个皇后放置在 `n x n` 的棋盘上，
并且使皇后彼此之间不能相互攻击。

返回所有不同的解决方案。

## 错误复盘

### 最初的遍历思路

一开始容易把棋盘当成普通二维数组，
用嵌套 `for` 循环去尝试每一个格子。

这样递归状态会变得很复杂：
不知道如何有效传递状态，
也不清楚终止条件应该怎么定义。

后来发现这道题的关键是：

每一行只能放一个皇后。

因此问题可以简化为：

在第 `row` 行，为皇后找到一个安全的 `col` 列。

这样递归深度就是行号，
每一层只需要遍历列。

### 斜线判断

列冲突很好判断，
但斜线冲突需要利用坐标规律。

在左上到右下方向的同一条斜线上：

```text
row - col 相等
```

在右上到左下方向的同一条斜线上：

```text
row + col 相等
```

因此检查 `(row, col)` 是否安全时，
只需要和之前已经放好的皇后比较这三个条件：

- 是否同列。
- `prevRow - prevCol == row - col`。
- `prevRow + prevCol == row + col`。

### 实现细节

实现过程中还出现过一些小笔误：

- `false` 写成 `flase`。
- `notes` 写成 `note`。
- `getQ` 函数里弄丢了 `n`。
- 把 `'Q'` 和 `"."` 的字符、字符串类型混淆。
- 循环边界里 `k <= i` 和 `k < i` 差一点。

这些问题提醒自己：
算法思路清楚之后，
实现细节仍然需要非常精确。

## 回溯设计

`backtracking(row)` 的含义是：

为第 `row` 行寻找一个安全的皇后位置。

在这一层中，
用 `for col := 0; col < n; col++` 遍历所有列。

如果 `check(notes, row, col)` 通过，
说明当前位置安全。

然后执行回溯三部曲：

1. 记录选择：`notes[row] = col`，并把这一行棋盘加入 `path`。
2. 递归进入下一行：`backtracking(row + 1)`。
3. 撤销选择：从 `path` 中移除这一行。

当 `row == n` 时，
说明 `n` 行都已经放好皇后，
得到一个完整解。

## 最终代码

```go
func solveNQueens(n int) [][]string {
    res := [][]string{}
    path := []string{}
    notes := make([]int, n)

    backtracking(n, &notes, &res, &path, 0)
    return res
}

func backtracking(
    n int,
    notes *[]int,
    res *[][]string,
    path *[]string,
    row int,
) {
    if row == n {
        tmp := make([]string, n)
        copy(tmp, *path)
        *res = append(*res, tmp)
        return
    }

    for col := 0; col < n; col++ {
        if !check(notes, row, col) {
            continue
        }

        (*notes)[row] = col
        *path = append(*path, getQ(col, n))

        backtracking(n, notes, res, path, row+1)

        *path = (*path)[:len(*path)-1]
    }
}

func getQ(col int, n int) string {
    tmp := make([]rune, n)
    for k := 0; k < n; k++ {
        tmp[k] = '.'
    }
    tmp[col] = 'Q'
    return string(tmp)
}

func check(notes *[]int, row int, col int) bool {
    for prevRow := 0; prevRow < row; prevRow++ {
        prevCol := (*notes)[prevRow]

        sameCol := prevCol == col
        sameMainDiag := prevRow-prevCol == row-col
        sameAntiDiag := prevRow+prevCol == row+col

        if sameCol || sameMainDiag || sameAntiDiag {
            return false
        }
    }
    return true
}
```

## 本篇总结

N 皇后的关键是把二维棋盘问题转化成按行递归。

每一层只负责当前行，
在这一行里枚举所有列。

判断安全时，
只需要检查之前行已经放好的皇后：
同列、主对角线、副对角线三种冲突。

这个题把回溯的路径记录、合法性判断和撤销选择都集中在一起，
很适合作为回溯章节的综合练习。

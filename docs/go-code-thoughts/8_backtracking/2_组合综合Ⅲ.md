# 216 组合总和 III

[题目链接](https://leetcode.cn/problems/combination-sum-iii/)

## 题目描述

找出所有相加之和为 `n` 的 `k` 个数的组合，
并满足以下条件：

- 只使用数字 `1` 到 `9`。
- 每个数字最多使用一次。

返回所有可能的有效组合列表。

## 基础回溯思路

这道题可以看作 `77. 组合` 的升级版。

除了要选出 `k` 个数字，还要求这些数字的总和等于 `n`。
所以回溯过程中需要同时维护：

- 当前路径 `path`。
- 当前可选择的起点 `start`。
- 当前路径的总和 `sum`。

当 `len(path) == k` 且 `sum == n` 时，
才找到一个有效组合。

## 未剪枝代码

```go
func combinationSum3(k int, n int) [][]int {
    res := [][]int{}
    path := []int{}
    backtracking(k, n, 1, &res, &path, 0)
    return res
}

func backtracking(k int, n int, start int, res *[][]int, path *[]int, sum int) {
    if sum == n && len(*path) == k {
        temp := make([]int, k)
        copy(temp, *path)
        *res = append(*res, temp)
        return
    }

    for i := start; i <= 9; i++ {
        sum += i
        *path = append(*path, i)
        backtracking(k, n, i+1, res, path, sum)
        sum -= i
        *path = (*path)[:len(*path)-1]
    }
}
```

这个版本是在组合模板上增加 `sum` 的直接写法。

每次选择数字时，`path` 和 `sum` 都要同步更新。
回溯时，也要同步撤销这两个状态。

## 双重剪枝思路

这道题有两个限制条件：

- 数量必须等于 `k`。
- 总和必须等于 `n`。

因此可以从两个方向剪枝。

第一，按数量剪枝。
这和组合问题一样，
如果剩余数字数量已经不够凑满 `k` 个，
就不需要继续尝试。

`for` 循环上界可以从 `9` 改成：

```text
9 - (k - len(path)) + 1
```

第二，按总和剪枝。
如果当前 `sum + i > n`，
由于后面的数字只会更大，
继续枚举也不可能得到合法答案。
此时可以直接 `break` 结束本层循环。

也可以在递归入口处判断 `sum > n`，
提前返回。

## 剪枝代码

```go
func combinationSum3(k int, n int) [][]int {
    res := [][]int{}
    path := []int{}
    backtracking(k, n, 1, &res, &path, 0)
    return res
}

func backtracking(k int, n int, start int, res *[][]int, path *[]int, sum int) {
    if sum > n {
        return
    }

    if sum == n && len(*path) == k {
        temp := make([]int, k)
        copy(temp, *path)
        *res = append(*res, temp)
        return
    }

    for i := start; i <= 9-(k-len(*path))+1; i++ {
        if sum+i > n {
            break
        }

        sum += i
        *path = append(*path, i)
        backtracking(k, n, i+1, res, path, sum)
        sum -= i
        *path = (*path)[:len(*path)-1]
    }
}
```

## 本篇总结

回溯问题里的每一个约束条件，
都可能变成一个剪枝点。

这道题在组合模板的基础上增加了总和限制，
所以既可以按数量剪枝，
也可以按当前和是否超过目标值剪枝。

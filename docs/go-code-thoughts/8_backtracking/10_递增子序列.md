# 491 非递减子序列

[题目链接](https://leetcode.cn/problems/non-decreasing-subsequences/)

## 题目描述

给你一个整数数组 `nums`，
找出并返回所有该数组中不同的非递减子序列。

非递减子序列中至少有两个元素。

数组中可能含有重复元素。
如果两个整数相等，也可以视作非递减序列的一种情况。

## 最大的疑问

这道题最容易疑惑的是：

为什么结果里可以有 `[7, 7]`，
去重逻辑难道没有生效吗？

答案是要区分两种重复。

## 需要去掉的重复

需要去掉的是解集中的重复。

例如 `nums = [4, 7, 4]`，
如果不同位置的 `4` 在同一层都开启了相同分支，
就可能得到重复的 `[4, 7]`。

这种重复应该被去掉。

解决方式是在每一层递归中创建一个 `map`，
记录本层已经选择过哪些数。

同一层里，相同数值只允许作为分支起点出现一次。

## 允许存在的重复

允许存在的是子序列内部的重复元素。

例如 `nums = [4, 7, 7]`，
`[4, 7, 7]` 是合法结果，
因为 `7 >= 7`，它符合非递减定义。

本层 `map` 不会阻止这种情况。
因为选择第一个 `7` 后会进入下一层递归，
下一层会创建一个新的空 `map`。

所以第二个 `7` 在新的一层里仍然可以被选择。

## 为什么不能排序

子集 II 可以排序后用 `i > start` 去重。

但这道题要求的是子序列，
必须保持原数组中的相对顺序。
如果排序，原来的顺序信息就被破坏了。

因此不能用排序去重，
只能在每一层用 `map` 做同层去重。

## 回溯思路

这道题接近子集模型，
也是沿途收集。

但只有当 `len(path) >= 2` 时，
当前路径才是一个符合题意的非递减子序列。

在每次选择 `nums[i]` 前，需要做两个判断：

- 如果 `path` 不为空，`nums[i]` 必须大于等于 `path` 的最后一个数。
- 如果 `nums[i]` 在本层已经被用过，就跳过，避免重复分支。

## 最终代码

```go
func findSubsequences(nums []int) [][]int {
    res := [][]int{}
    path := []int{}
    backtracking(nums, &res, &path, 0)
    return res
}

func backtracking(nums []int, res *[][]int, path *[]int, startIndex int) {
    if len(*path) >= 2 {
        tmp := make([]int, len(*path))
        copy(tmp, *path)
        *res = append(*res, tmp)
    }

    if startIndex == len(nums) {
        return
    }

    usedMap := make(map[int]bool)

    for i := startIndex; i < len(nums); i++ {
        if len(*path) > 0 && nums[i] < (*path)[len(*path)-1] {
            continue
        }

        if _, ok := usedMap[nums[i]]; ok {
            continue
        }

        usedMap[nums[i]] = true

        *path = append(*path, nums[i])
        backtracking(nums, res, path, i+1)
        *path = (*path)[:len(*path)-1]
    }
}
```

## 本篇总结

非递减子序列不能排序，
因为子序列必须保留原数组相对顺序。

这道题的去重应该发生在每一层递归中，
用局部 `map` 记录本层已经选择过的数字。

同时要允许同一条路径里出现相等数字，
因为非递减序列允许 `nums[i] == nums[j]`。

# 90 子集 II

[题目链接](https://leetcode.cn/problems/subsets-ii/)

## 题目描述

给你一个整数数组 `nums`，
其中可能包含重复元素。

请返回该数组所有可能的子集，也就是幂集。

解集不能包含重复子集，
答案可以按任意顺序返回。

## 基础框架

这道题的基础仍然是 `78. 子集`。

因为要求所有子集，
所以依旧使用沿途收集策略：
每次进入递归函数时，
当前 `path` 就是一个合法子集，
应该先复制并加入结果集。

## 新问题：重复元素

如果直接套用子集 I 的代码，
对于输入 `[1, 2, 2]`，
会产生重复子集。

原因是两个值相同的 `2` 在同一层作为兄弟节点时，
会开启两条本质相同的分支。

所以这道题需要去重。

## 排序加同层去重

去重的前提是排序：

```go
sort.Ints(nums)
```

排序后，相同数字会相邻，
才能通过 `nums[i] == nums[i-1]` 识别重复值。

核心去重逻辑是：

```go
if i > start && nums[i] == nums[i-1] {
    continue
}
```

`i > start` 表示当前数字不是本层的第一个选择。

如果当前数字和前一个数字相同，
说明前一个兄弟分支已经处理过同样的选择，
当前分支会产生重复结果，应该跳过。

这个条件只对同一层生效。
它不会阻止更深层合法地使用相同数字。

例如 `[2, 2]` 中，
选择第一个 `2` 后进入下一层，
此时第二个 `2` 可以作为新层的第一个选择，
所以 `[2, 2]` 不会被误删。

## 最终代码

```go
import "sort"

func subsetsWithDup(nums []int) [][]int {
    res := [][]int{}
    path := []int{}

    sort.Ints(nums)

    backtracking(nums, &res, &path, 0)
    return res
}

func backtracking(nums []int, res *[][]int, path *[]int, start int) {
    tmp := make([]int, len(*path))
    copy(tmp, *path)
    *res = append(*res, tmp)

    for i := start; i < len(nums); i++ {
        if i > start && nums[i] == nums[i-1] {
            continue
        }

        *path = append(*path, nums[i])
        backtracking(nums, res, path, i+1)
        *path = (*path)[:len(*path)-1]
    }
}
```

## 本篇总结

子集 II 等于子集问题的沿途收集，
再加上重复元素去重。

因为数组中有重复值，
所以要先排序，再用 `i > start` 跳过同层重复分支。

去重只发生在同一层，
不能阻止同一条路径中合法使用重复元素。

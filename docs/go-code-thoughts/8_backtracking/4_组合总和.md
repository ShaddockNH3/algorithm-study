# 39 组合总和

[题目链接](https://leetcode.cn/problems/combination-sum/)

## 题目描述

给你一个无重复元素的整数数组 `candidates` 和一个目标整数 `target`，
找出 `candidates` 中可以使数字和为 `target` 的所有不同组合。

`candidates` 中的同一个数字可以无限制重复选取。

答案可以按任意顺序返回。

## 错误复盘

这道题整理过程中遇到过几个很有价值的问题。

### 指针和下标的优先级

一开始写过类似 `*candidates[i]` 的表达式。

在 Go 中，下标操作 `[i]` 的优先级会先于解引用 `*`。
因此这个表达式会被理解成先对 `candidates` 取下标，
再解引用。

如果 `candidates` 是切片指针，就不能这样写。
正确写法是：

```go
(*candidates)[i]
```

括号明确表示先解引用得到切片，
再从切片中取第 `i` 个元素。

### 循环边界

初版循环写成过：

```go
for i := start; i <= len(*candidates); i++ {
}
```

切片合法下标是 `0` 到 `len-1`。
如果写成 `i <= len(...)`，
最后一次循环会访问到越界下标。

正确边界应该是：

```go
for i := start; i < len(candidates); i++ {
}
```

### 剪枝前需要排序

如果数组没有排序，
看到某个数字让当前和超过 `target` 时，
不能直接 `break`。

例如数组是 `[7, 2, 3]`，
`7` 超过目标不代表后面的 `2` 和 `3` 也超过目标。

所以高效剪枝的前提是先排序。
排序后，如果 `candidates[i] > target`，
后续数字只会更大，就可以放心 `break`。

## 核心差异

这道题和普通组合问题最大的区别是：

同一个数字可以重复使用。

所以递归进入下一层时，
不能从 `i + 1` 开始，
而应该继续从 `i` 开始。

```go
backtracking(candidates, target-candidates[i], res, path, i)
```

这样表示当前数字在下一层仍然可以被再次选择。

## 最终代码

```go
import "sort"

func combinationSum(candidates []int, target int) [][]int {
    res := [][]int{}
    path := []int{}

    sort.Ints(candidates)

    backtracking(candidates, target, &res, path, 0)
    return res
}

func backtracking(
    candidates []int,
    target int,
    res *[][]int,
    path []int,
    start int,
) {
    if target == 0 {
        tmp := make([]int, len(path))
        copy(tmp, path)
        *res = append(*res, tmp)
        return
    }

    for i := start; i < len(candidates); i++ {
        if candidates[i] > target {
            break
        }

        path = append(path, candidates[i])
        backtracking(candidates, target-candidates[i], res, path, i)
        path = path[:len(path)-1]
    }
}
```

## 本篇总结

组合总和的核心变化是数字可以重复使用，
所以递归时下一层仍然从当前下标 `i` 开始。

剪枝时要先排序。
排序之后才能根据当前数字已经大于剩余目标值，
直接结束本层循环。

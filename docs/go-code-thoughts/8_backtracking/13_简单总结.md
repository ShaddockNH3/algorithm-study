# 回溯算法总结

原笔记写于 2025 年 8 月 2 日下午。
这一阶段主要把组合、分割、子集、排列和去重问题串了一遍。

回溯本质上是一种有组织的暴力枚举。

它通常可以抽象为一棵决策树。
在这棵树上，最核心的动作是：

1. 做出选择。
2. 递归进入下一层。
3. 撤销选择。

也就是常见的回溯三部曲。

## 组合和子集模型

组合和子集问题的共同点是顺序不重要。

`[1, 2]` 和 `[2, 1]` 被视为同一种结果。

### 基础组合

为了避免重复选择，
使用 `startIndex` 控制下一层从哪里开始。

选择 `i` 后，
下一层递归从 `i + 1` 开始。

### 组合总和 I

如果元素可以重复使用，
递归时下一层继续从 `i` 开始。

这表示当前元素可以在后续继续被选择。

### 子集

子集问题中，
决策树上的每一个节点都是答案。

所以使用沿途收集：
进入递归函数时就先收集当前 `path`。

### 组合和子集去重

当输入中有重复元素，
但结果不能重复时，
常用做法是：

1. 先排序。
2. 在同一层跳过重复元素。

典型条件是：

```go
if i > start && nums[i] == nums[i-1] {
    continue
}
```

这个条件处理的是树层去重。

## 排列模型

排列问题的核心是顺序重要。

`[1, 2]` 和 `[2, 1]` 是不同结果。

因此排列问题不能用 `startIndex` 限制向后选择。
每一层的 `for` 循环都要从 `0` 开始。

### 全排列 I

为了避免同一条路径中重复使用元素，
需要一个已用清单。

可以用 `used []bool` 或 `map` 记录当前路径里哪些元素已经被使用。

### 全排列 II

有重复元素的全排列需要更细的去重：

1. 先排序。
2. 使用 `used []bool` 按位置记录。
3. 先跳过已经使用的位置。
4. 再跳过重复元素造成的同层重复分支。

核心条件是：

```go
if i > 0 && nums[i] == nums[i-1] && !used[i-1] {
    continue
}
```

## 分割模型

分割问题的核心是把一个整体切成几块。

例如：

- 分割回文串。
- 复原 IP 地址。

这类题里，
`for` 循环的 `i` 通常表示切割点。

当前切出来的片段是：

```go
s[startIndex : i+1]
```

递归时从 `i + 1` 开始继续处理剩余部分。

分割问题的难点通常在合法性校验：
例如判断回文串，或判断 IP 段是否在 `0` 到 `255` 范围内。

## 快速判断

遇到组合或子集问题：
顺序不重要，优先考虑 `startIndex`。

遇到排列问题：
顺序重要，`for` 循环通常从 `0` 开始，并配合 `used`。

遇到分割问题：
把 `for` 循环里的 `i` 理解为切割点。

题目说元素可以重复使用：
递归传 `i`，而不是 `i + 1`。

题目说输入有重复但解集不能重复：
先考虑排序，再做树层去重。

题目不能排序：
例如递增子序列，需要用每层 `map` 记录本层已经使用过的数字。

## 本章总结

回溯题的关键不是死记模板，
而是判断当前题目属于哪种决策树。

先分清组合、子集、排列、分割，
再决定结果收集位置、下一层起点、去重方式和剪枝条件。

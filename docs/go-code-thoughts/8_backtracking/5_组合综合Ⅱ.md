# 40 组合总和 II

[题目链接](https://leetcode.cn/problems/combination-sum-ii/)

## 题目描述

给定一个候选人编号集合 `candidates` 和一个目标数 `target`，
找出 `candidates` 中所有可以使数字和为 `target` 的组合。

`candidates` 中的每个数字在每个组合中只能使用一次。

注意，解集不能包含重复组合。

## 题目难点

这道题融合了几个限制：

- 数字和要等于 `target`。
- 每个数字只能使用一次。
- `candidates` 本身可能有重复数字。
- 最终结果不能有重复组合。

关键问题是如何在不漏解的前提下去重。

## 排序是前提

去重之前要先排序。

排序可以让相同数字靠在一起，
后面才能通过比较 `candidates[i]` 和 `candidates[i-1]`
判断当前选择是否会导致重复分支。

排序也能支持剪枝：
如果当前数字已经大于剩余 `target`，
后续数字只会更大，可以直接 `break`。

## 去重条件

核心去重条件是：

```go
if i > start && candidates[i] == candidates[i-1] {
    continue
}
```

这里最重要的是 `i > start`。

它表示当前正在同一层的 `for` 循环里横向移动，
并且当前数字不是这一层的第一个选择。

如果当前数字和前一个数字相同，
说明前一个兄弟分支已经考虑过同样的选择，
当前分支会产生重复组合，所以跳过。

## 为什么不能简单跳过所有重复数字

`i == start` 时，当前数字是这一层递归的第一个选择。
即使它和前一个数字相同，也不能跳过。

例如 `[2, 2]` 这样的组合需要两个 `2`。
第二个 `2` 出现在更深的一层，
此时它可能正好是那一层的 `start`，
应该允许被选择。

所以 `i > start` 区分的是同层重复，
不是同一条路径里的重复。

## 最终代码

```go
import "sort"

func combinationSum2(candidates []int, target int) [][]int {
    res := [][]int{}
    path := []int{}

    sort.Ints(candidates)

    backtracking(candidates, target, &res, path, 0)
    return res
}

func backtracking(
    candidates []int,
    target int,
    res *[][]int,
    path []int,
    start int,
) {
    if target == 0 {
        tmp := make([]int, len(path))
        copy(tmp, path)
        *res = append(*res, tmp)
        return
    }

    for i := start; i < len(candidates); i++ {
        if candidates[i] > target {
            break
        }

        if i > start && candidates[i] == candidates[i-1] {
            continue
        }

        path = append(path, candidates[i])
        backtracking(candidates, target-candidates[i], res, path, i+1)
        path = path[:len(path)-1]
    }
}
```

## 本篇总结

组合总和 II 的关键是排序加同层去重。

每个数字只能使用一次，
所以递归进入下一层时从 `i + 1` 开始。

去重时只跳过同层中重复的兄弟分支，
不能跳过更深层路径中合法使用的重复数字。

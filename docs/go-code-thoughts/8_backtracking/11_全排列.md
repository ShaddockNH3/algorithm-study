# 46 全排列

[题目链接](https://leetcode.cn/problems/permutations/)

## 题目描述

给定一个不含重复数字的数组 `nums`，
返回其所有可能的全排列。

答案可以按任意顺序返回。

## 和组合问题的区别

排列问题和组合、子集问题最大的区别是：
顺序重要。

在组合问题里，`[1, 2]` 和 `[2, 1]` 被视为同一种结果。
所以组合问题会用 `startIndex` 控制下一层只能向后选择。

但在排列问题里，`[1, 2]` 和 `[2, 1]` 是两个不同答案。
因此每一层都必须从整个 `nums` 数组里重新选择。

所以排列问题的循环是：

```go
for i := 0; i < len(nums); i++ {
}
```

而不是从 `startIndex` 开始。

## 如何避免同一路径重复使用元素

每一层都从 `0` 开始选择，
就需要防止同一个排列里重复使用同一个数字。

解决方法是维护一个已用清单。

原代码使用 `map[int]bool`：

- `usedMap[nums[i]] = true` 表示当前路径已经使用过这个数字。
- 如果 `usedMap[nums[i]]` 为真，就跳过。
- 回溯时要 `delete(usedMap, nums[i])`，把这个数字还回去。

这个删除动作很关键。
否则后续其他分支就无法再次使用这个数字，
也就无法生成完整的全排列。

## 最终代码

```go
func permute(nums []int) [][]int {
    res := [][]int{}
    path := []int{}
    usedMap := make(map[int]bool)
    backtracking(nums, &res, &path, usedMap)
    return res
}

func backtracking(nums []int, res *[][]int, path *[]int, usedMap map[int]bool) {
    if len(*path) == len(nums) {
        tmp := make([]int, len(nums))
        copy(tmp, *path)
        *res = append(*res, tmp)
        return
    }

    for i := 0; i < len(nums); i++ {
        if usedMap[nums[i]] {
            continue
        }

        *path = append(*path, nums[i])
        usedMap[nums[i]] = true

        backtracking(nums, res, path, usedMap)

        *path = (*path)[:len(*path)-1]
        delete(usedMap, nums[i])
    }
}
```

## 本篇总结

排列问题中顺序重要，
所以不能用 `startIndex` 限制只能向后选择。

每一层都要从 `0` 开始遍历所有数字。

同时需要用已用清单记录当前路径中哪些元素已经被使用。
回溯撤销选择时，
路径和已用清单都要同步恢复。

# 77 组合

[题目链接](https://leetcode.cn/problems/combinations/)

## 题目描述

给定两个整数 `n` 和 `k`，
返回范围 `[1, n]` 中所有可能的 `k` 个数的组合。

可以按任意顺序返回答案。

## 基础回溯思路

组合问题最直接的思路就是回溯。

每一层通过 `for` 循环枚举当前可以选择的数字，
选择一个数字后进入下一层。
递归返回后撤销选择，再尝试下一个数字。

这里需要一个 `start` 参数。
它表示当前层应该从哪个数字开始枚举。
选择 `i` 之后，下一层从 `i + 1` 开始，
这样可以避免重复使用同一个数字，也可以避免排列顺序造成的重复。

当 `path` 的长度等于 `k` 时，
说明已经得到一组组合。
此时要拷贝一份 `path` 放入结果集。

## 未剪枝代码

```go
func combine(n int, k int) [][]int {
    res := [][]int{}
    path := []int{}
    backtracking(n, k, 1, &res, &path)
    return res
}

func backtracking(n int, k int, start int, res *[][]int, path *[]int) {
    if len(*path) == k {
        temp := make([]int, k)
        copy(temp, *path)
        *res = append(*res, temp)
        return
    }

    for i := start; i <= n; i++ {
        *path = append(*path, i)
        backtracking(n, k, i+1, res, path)
        *path = (*path)[:len(*path)-1]
    }
}
```

这个版本逻辑清晰，是标准回溯模板。

但是它会做一些无效搜索。
例如 `n = 10, k = 5` 时，
如果当前已经走到 `i = 8`，
后面只剩下 `8, 9, 10` 这些数，
数量已经不够凑成 5 个数，
继续搜索就没有意义。

## 剪枝思路

剪枝的核心是判断剩余数字还够不够。

当前还需要选择的数量是：

```text
k - len(path)
```

如果从 `i` 到 `n` 的数字数量不够，
就不需要再尝试这个 `i`。

从 `i` 到 `n` 一共有：

```text
n - i + 1
```

需要满足：

```text
n - i + 1 >= k - len(path)
```

整理后得到 `for` 循环的上界：

```text
i <= n - (k - len(path)) + 1
```

## 剪枝代码

```go
func combine(n int, k int) [][]int {
    res := [][]int{}
    path := []int{}
    backtracking(n, k, 1, &res, &path)
    return res
}

func backtracking(n int, k int, start int, res *[][]int, path *[]int) {
    if len(*path) == k {
        temp := make([]int, k)
        copy(temp, *path)
        *res = append(*res, temp)
        return
    }

    for i := start; i <= n-(k-len(*path))+1; i++ {
        *path = append(*path, i)
        backtracking(n, k, i+1, res, path)
        *path = (*path)[:len(*path)-1]
    }
}
```

## 本篇总结

组合问题需要用 `start` 控制下一层的起点，
避免重复选择和重复组合。

基础版本先保证搜索完整正确。

优化时再观察剩余元素数量是否足够，
通过缩小 `for` 循环上界来剪掉不可能产生答案的分支。

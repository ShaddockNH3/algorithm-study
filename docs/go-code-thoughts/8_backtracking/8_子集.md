# 78 子集

[题目链接](https://leetcode.cn/problems/subsets/)

## 题目描述

给你一个整数数组 `nums`，
数组中的元素互不相同。

返回该数组所有可能的子集，也就是幂集。

解集不能包含重复的子集，
答案可以按任意顺序返回。

## 曾经的误区

一开始容易套用组合问题的模板，
写出类似这样的逻辑：

```go
if len(*path) != 0 {
    // 收集结果
    return
}
```

这个想法的问题在于：
只要 `path` 不为空就 `return`，
会导致搜索过早结束。

例如已经得到 `[1]` 时，
如果立刻返回，
就不会继续向下搜索 `[1, 2]`、`[1, 2, 3]` 这些子集。

## 子集的收集策略

子集问题和组合问题最大的不同是：

决策树上的每一个节点都是答案。

以 `[1, 2, 3]` 为例：

- 根节点路径 `[]` 是空集。
- 第一层路径 `[1]` 是一个子集。
- 第二层路径 `[1, 2]` 也是一个子集。

所以答案收集不应该放在某个终止条件里，
而应该在每次进入递归函数时就先收集当前路径。

这就是沿途收集。

## 隐式终止条件

子集问题不一定需要显式写：

```go
if start >= len(nums) {
    return
}
```

当 `start` 越来越大，
`for i := start; i < len(nums); i++` 自然无法进入时，
当前递归就会结束并返回。

这就是隐式终止条件。

## 最终代码

```go
func subsets(nums []int) [][]int {
    res := [][]int{}
    path := []int{}

    backtracking(nums, &res, &path, 0)
    return res
}

func backtracking(nums []int, res *[][]int, path *[]int, start int) {
    tmp := make([]int, len(*path))
    copy(tmp, *path)
    *res = append(*res, tmp)

    for i := start; i < len(nums); i++ {
        *path = append(*path, nums[i])
        backtracking(nums, res, path, i+1)
        *path = (*path)[:len(*path)-1]
    }
}
```

## 本篇总结

子集问题不是只在叶子节点收集答案。

它的每一个节点都代表一个合法子集，
所以要在进入递归函数时立刻收集当前路径。

回溯框架仍然是选择、递归、撤销，
只是结果收集策略发生了变化。

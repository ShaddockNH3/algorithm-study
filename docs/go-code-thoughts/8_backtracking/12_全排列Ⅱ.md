# 47 全排列 II

[题目链接](https://leetcode.cn/problems/permutations-ii/)

## 题目描述

给定一个可包含重复数字的序列 `nums`，
按任意顺序返回所有不重复的全排列。

## 错误复盘

在找到最终写法之前，
有几个思路上的弯路很值得记录。

第一，最初想用 `map[int][]int` 记录每个重复数字的所有索引，
再通过 `Contains` 和 `removeElement` 管理状态。
这个方案太复杂，可维护性不好。

第二，曾经混淆过数值是否用过和位置是否用过。

例如 `mp[nums[i]]` 检查的是某个值是否使用过，
而 `used[i]` 检查的是某个位置是否使用过。

对于有重复元素的排列问题，
按位置追踪更清晰。
因为两个相同的值虽然数值一样，
但来自数组中的不同位置。

第三，最终剪枝条件一度写成把位置使用判断和重复元素判断混在一起。
后来明确了它们应该是两道按顺序执行的检查：

1. 先判断当前位置是否已经用过。
2. 再判断当前重复元素是否会造成重复分支。

## 排序是前提

先排序：

```go
sort.Ints(nums)
```

排序会让相同数字相邻。
只有相同数字相邻，
后续的 `nums[i] == nums[i-1]` 去重判断才有意义。

## 使用位置清单

`used []bool` 记录的是每个位置是否已经进入当前路径。

`used[i] = true` 表示 `nums[i]` 这个位置的元素已经被放入当前排列。

这样可以防止在一个排列中重复使用同一个位置的元素。

## 双重剪枝

第一道检查是：

```go
if used[i] {
    continue
}
```

如果当前位置已经被当前路径使用过，
就不能再次选择。

第二道检查是：

```go
if i > 0 && nums[i] == nums[i-1] && !used[i-1] {
    continue
}
```

这个条件用于处理重复元素。

它表示：

- 当前元素和前一个元素相同。
- 前一个相同元素此刻没有在当前路径中。

这说明以前一个相同元素开头的兄弟分支已经探索完。
如果再选择当前元素，
会生成重复排列，
所以应该跳过。

## 例子理解

以 `[1, 2, 2]` 为例。

当已经生成过以第一个 `2` 开头的分支后，
回溯到 `path = [1]`，
此时第一个 `2` 的 `used` 状态会变回 `false`。

如果接下来再选择第二个 `2` 作为同层分支，
得到的排列集合会和前面完全重复。

所以当 `nums[i] == nums[i-1] && !used[i-1]` 时，
当前重复元素要跳过。

## 最终代码

```go
import "sort"

func permuteUnique(nums []int) [][]int {
    sort.Ints(nums)
    res := [][]int{}
    path := []int{}
    used := make([]bool, len(nums))
    backtracking(nums, &res, &path, used)
    return res
}

func backtracking(nums []int, res *[][]int, path *[]int, used []bool) {
    if len(*path) == len(nums) {
        tmp := make([]int, len(nums))
        copy(tmp, *path)
        *res = append(*res, tmp)
        return
    }

    for i := 0; i < len(nums); i++ {
        if used[i] {
            continue
        }

        if i > 0 && nums[i] == nums[i-1] && !used[i-1] {
            continue
        }

        *path = append(*path, nums[i])
        used[i] = true

        backtracking(nums, res, path, used)

        *path = (*path)[:len(*path)-1]
        used[i] = false
    }
}
```

## 本篇总结

全排列 II 的关键是：

- 先排序，让重复元素相邻。
- 用 `used []bool` 按位置记录当前路径使用情况。
- 先跳过已经使用的位置。
- 再用 `!used[i-1]` 跳过会造成重复的同层分支。

这里不要只按数值记录是否用过。
对于重复元素排列，位置才是更稳定的状态。

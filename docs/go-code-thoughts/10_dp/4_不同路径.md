# 62 不同路径

[题目链接](https://leetcode.cn/problems/unique-paths/)

## 题目描述

一个机器人位于 `m x n` 网格的左上角。

机器人每次只能向下或向右移动一步。

问机器人到达右下角总共有多少条不同路径。

## dp 数组定义

定义：

`dp[i][j]` 表示机器人从左上角 `(0, 0)` 出发，
到达格子 `(i, j)` 的不同路径数量。

最终要求：

```go
dp[m-1][n-1]
```

## 状态转移

机器人只能从两个方向到达 `(i, j)`：

- 从上方 `(i-1, j)` 向下走一步。
- 从左方 `(i, j-1)` 向右走一步。

所以：

```text
dp[i][j] = dp[i-1][j] + dp[i][j-1]
```

## 初始化

起点：

```text
dp[0][0] = 1
```

第一行只能一路向右到达，
所以第一行所有位置都是 `1`。

第一列只能一路向下到达，
所以第一列所有位置也是 `1`。

## 原始写法

这版代码把起点、第一行、第一列和内部格子都放在一个双重循环中处理。

```go
func uniquePaths(m int, n int) int {
    dp := make([][]int, m)
    for i := range dp {
        dp[i] = make([]int, n)
    }

    for i := 0; i < m; i++ {
        for j := 0; j < n; j++ {
            if i == 0 && j == 0 {
                dp[i][j] = 1
            } else if i == 0 {
                dp[i][j] = dp[i][j-1]
            } else if j == 0 {
                dp[i][j] = dp[i-1][j]
            } else {
                dp[i][j] = dp[i][j-1] + dp[i-1][j]
            }
        }
    }

    return dp[m-1][n-1]
}
```

## 常见优化写法

也可以先初始化第一行和第一列，
再从 `(1, 1)` 开始填内部格子。

这样内部循环只需要处理状态转移。

```go
func uniquePathsClean(m int, n int) int {
    dp := make([][]int, m)
    for i := range dp {
        dp[i] = make([]int, n)
    }

    for i := 0; i < m; i++ {
        dp[i][0] = 1
    }
    for j := 0; j < n; j++ {
        dp[0][j] = 1
    }

    for i := 1; i < m; i++ {
        for j := 1; j < n; j++ {
            dp[i][j] = dp[i-1][j] + dp[i][j-1]
        }
    }

    return dp[m-1][n-1]
}
```

## 本篇总结

不同路径是二维 DP 入门题。

核心流程是：

1. 定义 `dp[i][j]` 为到达当前格子的路径数量。
2. 初始化第一行和第一列。
3. 内部格子由上方和左方转移而来。

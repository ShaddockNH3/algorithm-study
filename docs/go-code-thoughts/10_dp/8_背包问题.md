# 01 背包问题

01 背包是动态规划中的经典模型。

问题是：
有若干物品，
每个物品有重量和价值。
背包容量有限。
每件物品只能选一次或不选。

目标是在不超过容量的前提下，
让总价值最大。

## 二维 DP

最直观的状态是二维表。

定义：

`dp[i][j]` 表示从下标 `0` 到 `i` 的物品中任意选择，
放入容量为 `j` 的背包，
能得到的最大价值。

面对第 `i` 件物品时，
有两种选择。

不放第 `i` 件物品：

```text
dp[i-1][j]
```

放第 `i` 件物品：

```text
dp[i-1][j-weights[i]] + values[i]
```

前提是 `j >= weights[i]`。

所以状态转移是：

```text
dp[i][j] = max(
    dp[i-1][j],
    dp[i-1][j-weights[i]] + values[i],
)
```

在二维 DP 中，
第 `i` 行只依赖第 `i-1` 行。

因此内层容量 `j` 正序或倒序遍历都可以，
因为引用的数据都来自上一行。

## 一维 DP

二维 DP 可以压缩成一维。

定义：

`dp[j]` 表示容量为 `j` 的背包当前能得到的最大价值。

状态转移：

```text
dp[j] = max(dp[j], dp[j-weights[i]] + values[i])
```

一维写法中，
容量必须倒序遍历：

```go
for j := bagSize; j >= weights[i]; j-- {
}
```

## 为什么必须倒序

一维数组中，
`dp[j]` 和 `dp[j-weights[i]]` 都在同一个数组里。

如果正序遍历，
当计算 `dp[j]` 时，
`dp[j-weights[i]]` 可能已经在本轮被当前物品更新过。

这样就等于同一个物品被重复使用，
01 背包会变成完全背包。

倒序遍历可以保证：
计算 `dp[j]` 时，
`dp[j-weights[i]]` 仍然是上一轮物品留下的旧值。

这就模拟了二维 DP 中参考上一行的行为。

## 为什么不能只算最后一个容量

一个很重要的问题是：
如果最终只需要 `dp[bagSize]`，
为什么每一轮还要更新其他容量？

原因是动态规划依赖最优子结构。

`dp[bagSize]` 的最优解可能依赖 `dp[bagSize-k]`。
而 `dp[bagSize-k]` 又依赖更小容量的状态。

每个 `dp[j]` 都是容量为 `j` 时的子问题最优解。
内层循环的任务不是只算最终答案，
而是把整个 `dp` 数组从上一轮物品状态推进到当前轮物品状态。

如果跳过某些容量，
后续更大容量可能会依赖到错误或过期的子问题结果。

## 最终代码

```go
package main

import "fmt"

func solve() int {
    var itemCount, bagSize int
    fmt.Scan(&itemCount, &bagSize)

    weights := make([]int, itemCount)
    values := make([]int, itemCount)
    dp := make([]int, bagSize+1)

    for i := 0; i < itemCount; i++ {
        fmt.Scan(&weights[i])
    }
    for i := 0; i < itemCount; i++ {
        fmt.Scan(&values[i])
    }

    for i := 0; i < itemCount; i++ {
        for j := bagSize; j >= weights[i]; j-- {
            dp[j] = max(dp[j], dp[j-weights[i]]+values[i])
        }
    }

    return dp[bagSize]
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}

func main() {
    maxValue := solve()
    fmt.Println(maxValue)
}
```

## 本篇总结

01 背包的核心是每件物品只能使用一次。

二维 DP 中，
当前行依赖上一行。

一维 DP 中，
必须倒序遍历容量，
才能保证每件物品只被使用一次。

每个容量状态都是后续状态的基础，
所以不能只更新最终容量。

# 63 不同路径 II

[题目链接](https://leetcode.cn/problems/unique-paths-ii/)

## 题目描述

一个机器人位于 `m x n` 网格的左上角。

机器人每次只能向下或向右移动一步，
目标是到达右下角。

网格中可能包含障碍物。
障碍物和空位置分别用 `1` 和 `0` 表示。

如果路径中包含障碍物方格，
该路径无效。

返回机器人能够到达右下角的不同路径数量。

## dp 数组定义

定义：

`dp[i][j]` 表示机器人从 `(0, 0)` 出发，
在不经过障碍物的情况下，
到达格子 `(i, j)` 的不同路径数量。

最终要求：

```go
dp[m-1][n-1]
```

## 状态转移

如果当前格子是障碍物：

```text
dp[i][j] = 0
```

如果当前格子不是障碍物，
机器人仍然只能从上方或左方到达：

```text
dp[i][j] = dp[i-1][j] + dp[i][j-1]
```

## 初始化

如果起点 `(0, 0)` 是障碍物，
直接返回 `0`。

如果起点不是障碍物，
则：

```text
dp[0][0] = 1
```

第一行和第一列的路径数会受到障碍物影响。

一旦某个位置是障碍物，
它本身的 `dp` 值为 `0`，
后面依赖它的位置也会自然变成 `0`。

Go 中 `int` 切片默认值就是 `0`，
所以障碍物位置可以不额外赋值。

## 最终代码

```go
func uniquePathsWithObstacles(obstacleGrid [][]int) int {
    if obstacleGrid[0][0] == 1 {
        return 0
    }

    m, n := len(obstacleGrid), len(obstacleGrid[0])
    dp := make([][]int, m)
    for i := range dp {
        dp[i] = make([]int, n)
    }

    for i := 0; i < m; i++ {
        for j := 0; j < n; j++ {
            if obstacleGrid[i][j] == 0 {
                if i == 0 && j == 0 {
                    dp[i][j] = 1
                } else if i == 0 {
                    dp[i][j] = dp[i][j-1]
                } else if j == 0 {
                    dp[i][j] = dp[i-1][j]
                } else {
                    dp[i][j] = dp[i][j-1] + dp[i-1][j]
                }
            }
        }
    }

    return dp[m-1][n-1]
}
```

## 本篇总结

不同路径 II 是在不同路径的基础上增加障碍物。

障碍物的含义是：
路径不能到达这个格子，
所以该位置路径数为 `0`。

代码中只在空地上计算路径数，
障碍物位置依靠默认值 `0` 即可。

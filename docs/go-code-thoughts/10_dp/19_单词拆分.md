# 139 单词拆分

[题目链接](https://leetcode.cn/problems/word-break/)

## 题目描述

给你一个字符串 `s` 和一个字符串列表 `wordDict`。

判断是否可以利用字典中的单词拼接出 `s`。

字典中的单词可以重复使用。

## 问题模型

这道题可以看成字符串形式的完全背包存在性问题。

- 背包是目标字符串 `s`。
- 物品是 `wordDict` 中的单词。
- 单词可以重复使用。
- 目标是判断能否刚好拼出整个字符串。

定义：

`dp[i]` 表示字符串 `s[0:i]` 是否可以被字典单词拼出。

初始化：

```text
dp[0] = true
```

空字符串可以被拼出。

## 为什么先背包后物品

这道题的遍历顺序很关键。

正确做法是先遍历字符串长度，
再枚举分割点。

当计算 `dp[i]` 时，
所有 `j < i` 的 `dp[j]` 都已经计算完成。

只要存在某个分割点 `j`，
满足：

- `dp[j] == true`
- `s[j:i]` 在字典中

就可以令 `dp[i] = true`。

## 先物品后背包为什么会错

以：

```text
s = "applepenapple"
wordDict = ["apple", "pen"]
```

为例。

如果先遍历单词：

第一轮处理 `"apple"` 时，
可以点亮 `dp[5]`。

但此时 `dp[8]` 还没有被 `"pen"` 点亮，
所以 `"apple"` 无法继续利用 `dp[8]` 去点亮 `dp[13]`。

第二轮处理 `"pen"` 时，
它会利用 `dp[5]` 点亮 `dp[8]`。

但是 `"apple"` 已经处理结束，
不会再回来利用新的 `dp[8]`。

最终 `dp[13]` 会错误地保持 `false`。

这就是信息传递被截断。

## 最终代码

```go
func wordBreak(s string, wordDict []string) bool {
    wordSet := make(map[string]bool)
    for _, word := range wordDict {
        wordSet[word] = true
    }

    dp := make([]bool, len(s)+1)
    dp[0] = true

    for i := 1; i <= len(s); i++ {
        for j := 0; j < i; j++ {
            if dp[j] {
                word := s[j:i]
                if wordSet[word] {
                    dp[i] = true
                    break
                }
            }
        }
    }

    return dp[len(s)]
}
```

## 本篇总结

单词拆分是完全背包的存在性问题，
但遍历顺序不能简单套模板。

它需要先保证较短前缀的状态已经全部确定，
再用这些前缀去推导更长前缀。

所以这里使用先背包后物品，
也就是先遍历字符串长度，
再枚举分割点。

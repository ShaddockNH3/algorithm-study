# 198 打家劫舍

[题目链接](https://leetcode.cn/problems/house-robber/)

## 题目描述

你是一名小偷，
计划偷窃沿街的房屋。

每间房内都藏有一定金额。
相邻房屋装有联动防盗系统，
如果同一晚上偷窃相邻两间房屋就会触发报警。

给定数组 `nums`，
返回在不触发报警的情况下能偷到的最高金额。

## dp 数组定义

定义：

`dp[i]` 表示从前 `i` 间房子中，
能够偷到的最高金额。

这里前 `i` 间房子对应 `nums[0:i]`。

最终要求：

```go
dp[len(nums)]
```

## 状态转移

计算 `dp[i]` 时，
对应正在考虑第 `i` 间房子，
也就是 `nums[i-1]`。

有两种选择：

不偷第 `i` 间：

```text
dp[i-1]
```

偷第 `i` 间：

因为不能偷相邻房子，
只能加上前 `i-2` 间的最优结果：

```text
nums[i-1] + dp[i-2]
```

所以：

```text
dp[i] = max(dp[i-1], nums[i-1] + dp[i-2])
```

## 标准代码

```go
func rob(nums []int) int {
    if len(nums) == 0 {
        return 0
    }

    dp := make([]int, len(nums)+1)
    dp[0] = 0
    dp[1] = nums[0]

    for i := 1; i < len(nums); i++ {
        dp[i+1] = max(dp[i], nums[i]+dp[i-1])
    }

    return dp[len(nums)]
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

原始思路里已经抓住了转移关系。
需要注意的是空数组边界，
否则直接访问 `nums[0]` 会出错。

## 空间压缩

状态转移只依赖前两个状态：

- `dp[i-2]`
- `dp[i-1]`

所以可以用两个变量滚动。

```go
func robOptimized(nums []int) int {
    pre, curr := 0, 0

    for _, num := range nums {
        temp := max(curr, num+pre)
        pre = curr
        curr = temp
    }

    return curr
}
```

这里：

- `pre` 表示 `dp[i-2]`。
- `curr` 表示 `dp[i-1]`。

每次循环计算当前最优结果，
再向前滚动。

## 本篇总结

打家劫舍的核心是每间房偷或不偷。

偷当前房子时，
只能接上前两间之前的最优结果。

不偷当前房子时，
直接继承前一间的最优结果。

由于只依赖前两个状态，
可以自然做空间压缩。

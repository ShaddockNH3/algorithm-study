# 518 零钱兑换 II

[题目链接](https://leetcode.cn/problems/coin-change-ii/)

## 题目描述

给你一个整数数组 `coins`，
表示不同面额的硬币。

每种硬币数量无限。

给定一个总金额 `amount`，
返回可以凑成该金额的硬币组合数。

如果无法凑出该金额，返回 `0`。

## 问题模型

这是完全背包的计数问题。

- 物品是硬币。
- 每种硬币可以使用无限次。
- 背包容量是 `amount`。
- 求的是装满背包的组合数。

这里求的是组合数，
顺序不重要。

例如 `1 + 2` 和 `2 + 1` 是同一种组合。

## 最初的二维 DP 思路

定义：

`dp[i][j]` 表示使用前 `i` 种硬币，
凑成金额 `j` 的组合数。

初始化：

```text
dp[i][0] = 1
```

凑成金额 0 的方法只有一种：
不选任何硬币。

状态转移：

```text
dp[i][j] = dp[i-1][j] + dp[i][j-coin]
```

其中：

- `dp[i-1][j]` 表示不用第 `i` 种硬币。
- `dp[i][j-coin]` 表示使用第 `i` 种硬币。

因为是完全背包，
使用当前硬币后仍然可以继续使用当前硬币，
所以第二项是 `dp[i]` 本层，而不是 `dp[i-1]`。

## 一个重要误区

这道题求的是组合数，
不是最少硬币数。

因此不能写成：

```go
dp[i][j] = dp[i][j-coin] + 1
```

`dp[i][j-coin]` 本身已经表示方法数，
要做的是把方法数累加进来。

也不能直接覆盖掉 `dp[i-1][j]`，
因为不用当前硬币的方法也要保留。

## 二维代码

```go
func change2D(amount int, coins []int) int {
    coinCount := len(coins)
    dp := make([][]int, coinCount+1)
    for i := range dp {
        dp[i] = make([]int, amount+1)
    }

    for i := 0; i <= coinCount; i++ {
        dp[i][0] = 1
    }

    for i := 1; i <= coinCount; i++ {
        coin := coins[i-1]
        for j := 1; j <= amount; j++ {
            dp[i][j] = dp[i-1][j]
            if j >= coin {
                dp[i][j] += dp[i][j-coin]
            }
        }
    }

    return dp[coinCount][amount]
}
```

## 一维 DP

定义：

`dp[j]` 表示凑成金额 `j` 的组合数。

初始化：

```text
dp[0] = 1
```

状态转移：

```text
dp[j] += dp[j-coin]
```

因为求组合数，
要先遍历硬币，
再正序遍历金额。

这样硬币的使用顺序是固定的，
不会把同一组硬币的不同排列重复计算进去。

## 一维代码

```go
func change(amount int, coins []int) int {
    dp := make([]int, amount+1)
    dp[0] = 1

    for _, coin := range coins {
        for j := coin; j <= amount; j++ {
            dp[j] += dp[j-coin]
        }
    }

    return dp[amount]
}
```

## 本篇总结

零钱兑换 II 是完全背包的组合计数问题。

状态转移是方法数累加，
不是取最大值，
也不是 `+1`。

一维写法中，
先遍历硬币再遍历金额，
可以保证统计的是组合数而不是排列数。

# 494 目标和

[题目链接](https://leetcode.cn/problems/target-sum/)

## 题目描述

给你一个非负整数数组 `nums` 和一个整数 `target`。

向数组中的每个整数前添加 `+` 或 `-`，
然后串联起来构造成一个表达式。

返回可以使表达式结果等于 `target` 的不同表达式数量。

## 问题转化

把所有加 `+` 的数记为集合 `P`，
所有加 `-` 的数记为集合 `N`。

根据题意：

```text
sum(P) - sum(N) = target
```

而数组总和为：

```text
sum(P) + sum(N) = totalSum
```

两式相加：

```text
2 * sum(P) = target + totalSum
```

所以：

```text
sum(P) = (target + totalSum) / 2
```

原问题就转化为：
在 `nums` 中选择一个子集，
使其元素和恰好等于 `(target + totalSum) / 2`，
求这样的子集数量。

这就是 01 背包的计数问题。

## 剪枝条件

如果 `abs(target) > totalSum`，
无论怎么添加正负号都不可能达到目标。

如果 `totalSum + target` 是奇数，
就无法得到整数的正数集合和。

这两种情况都直接返回 `0`。

## 回溯写法

最直观的做法是回溯。

对每个数字都有两种选择：

- 添加 `+`。
- 添加 `-`。

递归函数中用 `index` 表示当前处理到哪个数，
用 `currentSum` 表示当前表达式累计和。

```go
func findTargetSumWaysDFS(nums []int, target int) int {
    count := 0
    backtrackHelper(nums, target, 0, 0, &count)
    return count
}

func backtrackHelper(
    nums []int,
    target int,
    index int,
    currentSum int,
    count *int,
) {
    if index == len(nums) {
        if currentSum == target {
            (*count)++
        }
        return
    }

    backtrackHelper(nums, target, index+1, currentSum+nums[index], count)
    backtrackHelper(nums, target, index+1, currentSum-nums[index], count)
}
```

Go 中 `int` 是值传递。
每次递归调用拿到的是自己的 `currentSum`，
所以返回时不需要显式撤销。

这个写法直观，
但时间复杂度是 `O(2^n)`。

## 二维 DP

转化成背包后，
可以定义：

`dp[i][j]` 表示用前 `i` 个数字，
凑出和为 `j` 的方法数。

初始化：

```text
dp[0][0] = 1
```

表示不用任何数字凑出和 0 有一种方法。

转移：

```text
dp[i][j] = dp[i-1][j] + dp[i-1][j-num]
```

其中第二项要求 `j >= num`。

```go
func findTargetSumWays2DDP(nums []int, target int) int {
    sum := 0
    for _, num := range nums {
        sum += num
    }

    if abs(target) > sum || (sum+target)%2 == 1 {
        return 0
    }

    bagSize := (sum + target) / 2
    dp := make([][]int, len(nums)+1)
    for i := range dp {
        dp[i] = make([]int, bagSize+1)
    }

    dp[0][0] = 1

    for i := 1; i <= len(nums); i++ {
        num := nums[i-1]
        for j := 0; j <= bagSize; j++ {
            dp[i][j] = dp[i-1][j]
            if j >= num {
                dp[i][j] += dp[i-1][j-num]
            }
        }
    }

    return dp[len(nums)][bagSize]
}
```

二维 DP 的状态清晰，
但空间复杂度是 `O(n * bagSize)`。

## 一维 DP

继续压缩空间。

定义：

`dp[j]` 表示凑成和为 `j` 的方法数。

转移：

```text
dp[j] += dp[j-num]
```

因为每个数字只能使用一次，
所以 `j` 必须倒序遍历。

```go
func findTargetSumWays(nums []int, target int) int {
    sum := 0
    for _, num := range nums {
        sum += num
    }

    if abs(target) > sum || (sum+target)%2 == 1 {
        return 0
    }

    bagSize := (sum + target) / 2
    dp := make([]int, bagSize+1)
    dp[0] = 1

    for _, num := range nums {
        for j := bagSize; j >= num; j-- {
            dp[j] += dp[j-num]
        }
    }

    return dp[bagSize]
}

func abs(x int) int {
    if x < 0 {
        return -x
    }
    return x
}
```

## 关于 0 的处理

一维 DP 能自然处理 `num == 0`。

当 `num` 为 0 时：

```text
dp[j] += dp[j]
```

这会让方法数翻倍。

这正好对应 `+0` 和 `-0` 两种表达式选择。

## 本篇总结

目标和的关键不是直接做加减，
而是把正号集合和负号集合转化为子集和问题。

转化后，
它就是 01 背包中的装满背包方法数问题。

回溯能直观理解决策树，
二维 DP 适合理解状态，
一维 DP 是最终更常用的空间优化写法。

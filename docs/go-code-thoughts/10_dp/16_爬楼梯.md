# 爬楼梯：从斐波那契到完全背包

[卡码网 57. 爬楼梯](https://kamacoder.com/problempage.php?pid=1067)

## 问题对比

经典爬楼梯问题中，
每次可以爬 `1` 或 `2` 个台阶。

这个版本中，
每次最多可以爬 `m` 个台阶，
也就是可以爬：

```text
1, 2, ..., m
```

## 从斐波那契到完全背包

经典版的递推关系是：

```text
dp[i] = dp[i-1] + dp[i-2]
```

因为到达第 `i` 阶，
只能从第 `i-1` 阶走 1 步，
或者从第 `i-2` 阶走 2 步。

当每次最多可以走 `m` 阶时，
到达第 `i` 阶可以来自：

- 第 `i-1` 阶，走 1 步。
- 第 `i-2` 阶，走 2 步。
- 一直到第 `i-m` 阶，走 `m` 步。

因此：

```text
dp[i] = dp[i-1] + dp[i-2] + ... + dp[i-m]
```

这就变成了完全背包。

## 背包建模

- 背包容量是目标阶数 `n`。
- 物品是每次可以走的步数 `1, 2, ..., m`。
- 每种步数可以使用多次。
- 目标是凑满 `n` 的方法数。

## 排列还是组合

爬楼梯问题中，
不同顺序代表不同方法。

例如：

```text
1 + 2
2 + 1
```

这两种爬法不同。

所以这是排列数问题。

求排列数时，
应该先遍历背包容量，
再遍历物品。

这样对每个阶数 `j`，
所有步数都有机会成为最后一步。

## dp 初始化

`dp[j]` 表示爬到第 `j` 阶的方法数。

初始化：

```text
dp[0] = 1
```

在背包问题中，
`dp[0]` 表示什么都不选也能凑出容量 0。

它是后续递推的起点。

## 最终代码

```go
package main

import "fmt"

func solve() {
    var n int
    var m int

    fmt.Scanln(&n, &m)

    dp := make([]int, n+1)
    dp[0] = 1

    for j := 1; j <= n; j++ {
        for step := 1; step <= m; step++ {
            if j >= step {
                dp[j] += dp[j-step]
            }
        }
    }

    fmt.Println(dp[n])
}

func main() {
    solve()
}
```

## 本篇总结

经典爬楼梯是这个问题在 `m = 2` 时的特例。

当步数扩展为 `1` 到 `m` 后，
它可以看成完全背包排列数问题。

因为爬楼梯的顺序不同算不同方法，
所以遍历顺序是先背包后物品。

# 337 打家劫舍 III

[题目链接](https://leetcode.cn/problems/house-robber-iii/)

## 题目描述

这次房屋排列成一棵二叉树。

如果两个直接相连的房屋在同一晚被偷，
就会触发报警。

直接相连指父节点和子节点相连。

返回在不触发报警的情况下能偷到的最高金额。

## 曾经的错误尝试

### 层序遍历求和

最开始想把每一层的节点金额求和，
把树转成一个线性数组，
再套打家劫舍 I。

这个思路不成立。

因为偷了某一层，
并不意味着下一层所有节点都不能偷。

规则只限制父子直接相连的节点，
不是限制整层。

### 后序遍历拍平

另一个想法是用后序遍历把树拍成序列，
再套线性打家劫舍。

这也不成立。

线性序列中的相邻，
不等于树中的父子相邻。

树中的父子节点在遍历序列里可能相隔很远，
遍历序列中相邻的两个节点也可能没有父子关系。

因此不能强行把树降维成线性问题。

## 树形 DP 思路

对每个节点，
需要向父节点返回两个状态：

- 不偷当前节点时，这棵子树能获得的最大金额。
- 偷当前节点时，这棵子树能获得的最大金额。

定义 `dfs(root)` 返回长度为 2 的数组：

```text
result[0] = 不偷 root 的最大金额
result[1] = 偷 root 的最大金额
```

递归使用后序遍历。
先拿到左右子树的结果，
再计算当前节点的结果。

## 状态转移

如果不偷当前节点，
左右孩子可以偷，也可以不偷。
所以取各自最大值：

```text
notRob = max(left[0], left[1]) + max(right[0], right[1])
```

如果偷当前节点，
左右孩子都不能偷：

```text
rob = root.Val + left[0] + right[0]
```

最终答案是：

```text
max(result[0], result[1])
```

## 记忆化说明

普通二叉树中，
每个节点只有一个父节点，
一次后序遍历不会重复计算同一棵子树。

所以这道题不一定需要记忆化。

原笔记中保留了 `memo` 写法，
它也可以工作，
只是对本题来说不是必需条件。

## 最终代码

```go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */

var memo map[*TreeNode][]int

func rob(root *TreeNode) int {
    memo = make(map[*TreeNode][]int)
    result := dfs(root)
    return max(result[0], result[1])
}

func dfs(root *TreeNode) []int {
    if root == nil {
        return []int{0, 0}
    }

    if result, ok := memo[root]; ok {
        return result
    }

    left := dfs(root.Left)
    right := dfs(root.Right)

    notRob := max(left[0], left[1]) + max(right[0], right[1])
    robCurrent := root.Val + left[0] + right[0]

    result := []int{notRob, robCurrent}
    memo[root] = result

    return result
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

## 本篇总结

打家劫舍 III 把线性 DP 扩展到了树形 DP。

不能把树简单拍平成数组，
因为父子关系才是限制条件。

每个节点返回偷与不偷两个状态，
父节点再根据这两个状态做选择。

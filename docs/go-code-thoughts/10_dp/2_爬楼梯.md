# 70 爬楼梯

[题目链接](https://leetcode.cn/problems/climbing-stairs/)

## 题目描述

假设正在爬楼梯。
需要 `n` 阶才能到达楼顶。

每次可以爬 `1` 或 `2` 个台阶。

问有多少种不同的方法可以爬到楼顶。

## 状态转移

要到达第 `n` 阶，
最后一步只有两种可能：

- 从第 `n-1` 阶爬 1 个台阶。
- 从第 `n-2` 阶爬 2 个台阶。

如果 `dp[i]` 表示到达第 `i` 阶的方法数，
那么：

```text
dp[i] = dp[i-1] + dp[i-2]
```

这和斐波那契数列非常接近。

## DP 表写法

一种严谨的定义是：
`dp[i]` 表示到达第 `i` 阶的方法数。

此时：

```text
dp[1] = 1
dp[2] = 2
```

在这个定义下，
`dp[0]` 不需要参与本题的基础写法。

```go
func climbStairs(n int) int {
    if n <= 2 {
        return n
    }

    dp := make([]int, n+1)
    dp[1], dp[2] = 1, 2

    for i := 3; i <= n; i++ {
        dp[i] = dp[i-1] + dp[i-2]
    }

    return dp[n]
}
```

## 关于 dp[0]

有些写法会定义 `dp[0] = 1`。

这种写法在数学上可以成立，
但需要额外解释：
站在第 0 阶不动，也算一种到达方式。

对于本题最直接的定义，
从 `dp[1]` 和 `dp[2]` 开始会更直观。

当问题扩展成背包模型时，
`dp[0] = 1` 的意义会变得更自然。

## 状态压缩

计算第 `i` 阶只依赖前两阶。

所以可以用两个变量滚动计算。

```go
func climbStairsCompressed(n int) int {
    if n == 1 {
        return 1
    }

    pre, next := 1, 2
    for i := 3; i <= n; i++ {
        pre, next = next, pre+next
    }

    return next
}
```

这里 `pre` 和 `next` 分别对应 `dp[i-2]` 和 `dp[i-1]`。

## 扩展：每次可以爬 1 到 m 阶

如果每次可以爬 `1, 2, ..., m` 个台阶，
问题就可以看成完全背包。

- 背包容量是楼梯总数 `n`。
- 物品是每次可走的步数 `1, 2, ..., m`。
- 目标是装满容量 `n` 的排列数。

此时 `dp[0] = 1` 是关键初始化。
它表示凑出 0 阶有一种方式，也就是不走。

```cpp
int climbStairsWithM(int n, int m) {
    vector<int> dp(n + 1, 0);
    dp[0] = 1;

    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= m; j++) {
            if (i - j >= 0) {
                dp[i] += dp[i - j];
            }
        }
    }

    return dp[n];
}
```

## 本篇总结

爬楼梯本质上是斐波那契模型。

重点是明确 `dp[i]` 的含义，
然后再决定初始化方式。

当题目扩展成每次可以走多种步数时，
它会自然转向完全背包问题，
这时 `dp[0] = 1` 就成为递推的基础。

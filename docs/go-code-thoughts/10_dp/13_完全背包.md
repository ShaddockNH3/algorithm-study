# 完全背包

完全背包和 01 背包非常像，
但有一个核心区别：

- 01 背包中，每件物品最多选一次。
- 完全背包中，每种物品可以选任意多次。

## 问题描述

有 `N` 种物品和一个容量为 `V` 的背包。

第 `i` 种物品重量为 `weight[i]`，
价值为 `value[i]`。

每种物品有无限件。

目标是在不超过背包容量的前提下，
让总价值最大。

## 二维 DP

定义：

`dp[i][j]` 表示从前 `i` 种物品中任意选择，
放入容量为 `j` 的背包，
能获得的最大价值。

面对第 `i` 种物品时，
有两种选择。

不放第 `i` 种物品：

```text
dp[i-1][j]
```

放第 `i` 种物品：

因为物品可以重复使用，
放入一件后仍然可以继续考虑第 `i` 种物品。

所以是：

```text
dp[i][j-weight[i]] + value[i]
```

状态转移：

```text
dp[i][j] = max(dp[i-1][j], dp[i][j-weight[i]] + value[i])
```

## 二维代码

```go
func completeKnapsack2D(weights []int, values []int, capacity int) int {
    itemCount := len(weights)
    dp := make([][]int, itemCount+1)
    for i := range dp {
        dp[i] = make([]int, capacity+1)
    }

    for i := 1; i <= itemCount; i++ {
        weight := weights[i-1]
        value := values[i-1]

        for j := 1; j <= capacity; j++ {
            dp[i][j] = dp[i-1][j]

            if j >= weight {
                dp[i][j] = max(dp[i][j], dp[i][j-weight]+value)
            }
        }
    }

    return dp[itemCount][capacity]
}
```

## 一维 DP

定义：

`dp[j]` 表示容量为 `j` 的背包能获得的最大价值。

状态转移：

```text
dp[j] = max(dp[j], dp[j-weight] + value)
```

完全背包的一维写法中，
容量要正序遍历：

```go
for j := weight; j <= capacity; j++ {
}
```

原因是计算 `dp[j]` 时，
需要允许 `dp[j-weight]` 已经在本轮被当前物品更新过。

这正好表示当前物品可以被重复使用。

## 一维代码

```go
func completeKnapsack1D(weights []int, values []int, capacity int) int {
    dp := make([]int, capacity+1)

    for i := 0; i < len(weights); i++ {
        weight := weights[i]
        value := values[i]

        for j := weight; j <= capacity; j++ {
            dp[j] = max(dp[j], dp[j-weight]+value)
        }
    }

    return dp[capacity]
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

## 和 01 背包的遍历区别

一维 01 背包中，
容量要倒序遍历。

这样能保证 `dp[j-weight]` 还是上一轮物品的状态，
当前物品不会被重复使用。

一维完全背包中，
容量要正序遍历。

这样能让 `dp[j-weight]` 使用本轮已经更新过的状态，
表示当前物品可以继续被选。

## 关于计数问题的顺序

如果完全背包求的是最大价值，
通常使用：

```text
外层遍历物品
内层正序遍历容量
```

如果求的是装满背包的方法数，
还要继续区分组合和排列。

求组合数时：

```text
外层遍历物品
内层遍历背包
```

求排列数时：

```text
外层遍历背包
内层遍历物品
```

这个区别会在零钱兑换 II 和组合总和 IV 中体现出来。

## 本篇总结

完全背包的核心是物品可以重复使用。

二维状态转移中，
放入当前物品后仍然停留在当前物品层：

```text
dp[i][j-weight[i]]
```

一维压缩后，
容量正序遍历就是这个思想的体现。

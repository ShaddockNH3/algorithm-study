# 322 零钱兑换

[题目链接](https://leetcode.cn/problems/coin-change/)

## 题目描述

给你一个整数数组 `coins`，
表示不同面额的硬币。

每种硬币数量无限。

给定一个总金额 `amount`，
返回可以凑成该金额所需的最少硬币数。

如果无法凑成该金额，返回 `-1`。

## 问题模型

这是完全背包的最小值问题。

- 背包容量是 `amount`。
- 物品是硬币。
- 每种硬币可以使用无限次。
- 目标是装满背包所需的最少物品数量。

## 最初的问题

一开始写过类似这样的逻辑：

```go
func coinChangeWrong(coins []int, amount int) int {
    dp := make([]int, amount+1)

    for _, coin := range coins {
        for j := coin; j <= amount; j++ {
            if j == coin {
                dp[coin] = 1
            }
            dp[j] = dp[j-coin] + 1
        }
    }

    return dp[amount]
}
```

这个写法有几个问题。

第一，`j == coin` 的特判是多余的。
如果 `dp[0] = 0`，
那么通用转移会自然得到：

```text
dp[coin] = dp[0] + 1 = 1
```

第二，直接赋值覆盖会让结果受硬币遍历顺序影响。

例如 `amount = 6, coins = [1, 3, 4]`，
`3 + 3` 只需要 2 枚硬币，
但后续可能被 `1 + 1 + 4` 的 3 枚硬币方案覆盖。

正确做法应该是在所有可能方案中取最小值。

第三，默认值 `0` 无法区分两种状态：

- 金额为 0，需要 0 枚硬币。
- 某个金额无法凑成。

所以需要一个哨兵值表示不可达。

## 正确思路

定义：

`dp[j]` 表示凑成金额 `j` 所需的最少硬币数。

初始化：

- `dp[0] = 0`
- 其他位置初始化为一个不可能达到的大值，例如 `amount + 1`

状态转移：

```text
dp[j] = min(dp[j], dp[j-coin] + 1)
```

如果最终 `dp[amount]` 仍然是哨兵值，
说明无法凑成，返回 `-1`。

## 最终代码

```go
func coinChange(coins []int, amount int) int {
    infinity := amount + 1
    dp := make([]int, amount+1)

    for i := 1; i <= amount; i++ {
        dp[i] = infinity
    }

    dp[0] = 0

    for _, coin := range coins {
        for j := coin; j <= amount; j++ {
            if dp[j-coin] != infinity {
                dp[j] = min(dp[j], dp[j-coin]+1)
            }
        }
    }

    if dp[amount] == infinity {
        return -1
    }

    return dp[amount]
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}
```

## 本篇总结

零钱兑换求的是最少硬币数，
所以状态转移要取 `min`。

不可达状态不能用默认值 `0` 表示，
需要用哨兵值区分。

`j == coin` 的特例可以被统一转移覆盖，
不需要单独写分支。

# 377 组合总和 IV

[题目链接](https://leetcode.cn/problems/combination-sum-iv/)

## 题目特点

题目名叫组合总和，
但这道题实际统计的是排列数。

顺序不同会被视为不同方案。

例如凑成 `3`：

```text
1 + 2
2 + 1
```

这两种会被算作不同结果。

所以在写 DP 前，
要先判断清楚它是排列问题。

## 背包模型

这道题是完全背包的装满背包方法数问题。

- 背包容量是 `target`。
- 物品是 `nums` 中的数字。
- 每个数字可以使用多次。
- 求凑成 `target` 的排列数。

定义：

`dp[j]` 表示凑成总和为 `j` 的排列数。

初始化：

```text
dp[0] = 1
```

凑成 0 的方法有一种：
什么都不选。

## 状态转移

计算 `dp[j]` 时，
可以枚举最后一个加入的数字 `num`。

如果 `j >= num`，
那么所有凑成 `j - num` 的排列，
都可以在末尾追加 `num`，
变成凑成 `j` 的排列。

所以：

```text
dp[j] += dp[j-num]
```

## 为什么先背包后物品

这道题求排列数，
所以外层要遍历背包容量，
内层遍历物品。

对于每个容量 `j`，
都让所有数字都有机会成为最后一个元素。

例如凑成 `3`：

- 在 `dp[3]` 中追加 `1`，来自 `dp[2]`。
- 在 `dp[3]` 中追加 `2`，来自 `dp[1]`。

这样 `1 + 2` 和 `2 + 1` 会通过不同的最后一步被统计出来。

如果先遍历物品，
物品顺序被固定，
统计出来的是组合数。

## 最终代码

```go
func combinationSum4(nums []int, target int) int {
    dp := make([]int, target+1)
    dp[0] = 1

    for j := 1; j <= target; j++ {
        for _, num := range nums {
            if j >= num {
                dp[j] += dp[j-num]
            }
        }
    }

    return dp[target]
}
```

## 本篇总结

组合总和 IV 的核心是识别出它实际求的是排列数。

完全背包计数题中：

- 求组合数时，通常先物品后背包。
- 求排列数时，通常先背包后物品。

本题要统计不同顺序，
所以使用先背包后物品的遍历顺序。

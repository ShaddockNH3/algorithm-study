# 509 斐波那契数

[题目链接](https://leetcode.cn/problems/fibonacci-number/)

## 题目描述

斐波那契数通常用 `F(n)` 表示。

数列从 `0` 和 `1` 开始，
后面的每一项都是前面两项的和：

```text
F(0) = 0
F(1) = 1
F(n) = F(n-1) + F(n-2), n > 1
```

给定 `n`，计算 `F(n)`。

## 递归思路

看到公式 `F(n) = F(n-1) + F(n-2)`，
最直接的想法是递归。

```go
func fibRecursive(n int) int {
    if n <= 1 {
        return n
    }
    return fibRecursive(n-1) + fibRecursive(n-2)
}
```

这个写法完全贴合数学公式，
但会有大量重复计算。

例如计算 `fib(5)` 时会计算 `fib(4)` 和 `fib(3)`，
而计算 `fib(4)` 时又会再次计算 `fib(3)`。

`n` 大时，重复计算会迅速膨胀。

## DP 表写法

用 `dp[i]` 表示 `F(i)`。

初始化：

```text
dp[0] = 0
dp[1] = 1
```

递推公式：

```text
dp[i] = dp[i-1] + dp[i-2]
```

代码：

```go
func fibDPTable(n int) int {
    if n <= 1 {
        return n
    }

    dp := make([]int, n+1)
    dp[0] = 0
    dp[1] = 1

    for i := 2; i <= n; i++ {
        dp[i] = dp[i-1] + dp[i-2]
    }

    return dp[n]
}
```

这个写法从底向上计算，
每个状态只计算一次。

## 状态压缩

计算 `dp[i]` 时只依赖 `dp[i-1]` 和 `dp[i-2]`。

因此不需要保存整个 `dp` 数组，
用两个变量即可。

```go
func fib(n int) int {
    if n == 0 || n == 1 {
        return n
    }

    pre, next := 0, 1
    for i := 2; i <= n; i++ {
        pre, next = next, pre+next
    }

    return next
}
```

这里：

- `pre` 表示 `F(i-2)`。
- `next` 表示 `F(i-1)`。

每次循环都向前滚动一步。

## 本篇总结

斐波那契是最基础的 DP 模型。

它体现了几个核心点：

- 大问题可以拆成小问题。
- 小问题之间有状态转移关系。
- 自底向上计算可以避免重复递归。
- 如果只依赖有限个前置状态，可以做状态压缩。

# 135 分发糖果

[题目链接](https://leetcode.cn/problems/candy/)

## 题目描述

`n` 个孩子站成一排。
给你一个整数数组 `ratings`，
表示每个孩子的评分。

需要按照以下要求分发糖果：

- 每个孩子至少分到 1 个糖果。
- 相邻两个孩子中，评分更高的孩子要获得更多糖果。

返回需要准备的最少糖果数。

## 最初的困惑

这道题难在一个孩子会同时受到左右邻居影响。

如果一个孩子评分比两边都高，
他既要比左边糖果多，
也要比右边糖果多。

一开始尝试过统计大于号、小于号关系，
或者对评分做某些预处理。
但这些思路很难处理连续上升和连续下降的传递关系。

## 两次遍历思路

突破点是把两个方向的约束拆开。

第一次从左到右遍历，
只处理一种规则：

如果当前孩子评分比左边孩子高，
当前孩子糖果数就应该比左边多 1。

第二次从右到左遍历，
只处理另一种规则：

如果当前孩子评分比右边孩子高，
当前孩子糖果数就应该比右边多 1。

第二次遍历时不能直接覆盖第一次的结果。
因为当前孩子可能同时需要满足左右两个方向的规则。
所以要取最大值。

## 最终代码

```go
func candy(ratings []int) int {
    n := len(ratings)
    if n == 0 {
        return 0
    }

    candies := make([]int, n)
    for i := 0; i < n; i++ {
        candies[i] = 1
    }

    for i := 1; i < n; i++ {
        if ratings[i] > ratings[i-1] {
            candies[i] = candies[i-1] + 1
        }
    }

    for i := n - 2; i >= 0; i-- {
        if ratings[i] > ratings[i+1] {
            candies[i] = max(candies[i], candies[i+1]+1)
        }
    }

    totalCandies := 0
    for _, c := range candies {
        totalCandies += c
    }

    return totalCandies
}

func max(a, b int) int {
    if a > b {
        return a
    }
    return b
}
```

## 补充写法

原笔记还附了力扣官方题解中的另一种写法：
先用 `left` 数组记录左规则，
再从右往左用变量 `right` 记录右规则，
每个位置累加 `max(left[i], right)`。

来源链接：
[力扣官方题解](https://leetcode.cn/problems/candy/solutions/533150/fen-fa-tang-guo-by-leetcode-solution-f01p/)

```go
func candyOptimized(ratings []int) (ans int) {
    n := len(ratings)
    left := make([]int, n)

    for i, r := range ratings {
        if i > 0 && r > ratings[i-1] {
            left[i] = left[i-1] + 1
        } else {
            left[i] = 1
        }
    }

    right := 0
    for i := n - 1; i >= 0; i-- {
        if i < n-1 && ratings[i] > ratings[i+1] {
            right++
        } else {
            right = 1
        }
        ans += max(left[i], right)
    }

    return
}
```

## 本篇总结

当一个问题同时受左右两个方向约束时，
可以拆成两次遍历。

先满足从左到右的规则，
再满足从右到左的规则。

第二次处理时取最大值，
保证不破坏第一次已经满足的约束。

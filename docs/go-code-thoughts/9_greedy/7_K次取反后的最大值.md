# 1005 K 次取反后最大化的数组和

[题目链接](https://leetcode.cn/problems/maximize-sum-of-array-after-k-negations/)

## 题目描述

给定一个整数数组 `nums`，
只能通过以下方式修改数组：

选择某个索引 `i`，
并将 `nums[i]` 替换为 `-nums[i]`。

这个过程总共重复 `k` 次。
可以多次选择同一个索引。

返回修改后数组可能的最大和。

## 贪心标准

每一次取反都应该让总和增加得最多，
或者在不得不减少时，让损失最小。

如果有负数，
翻转绝对值越大的负数，
总和增加越多。

如果已经没有负数，
但还剩奇数次取反，
就只能翻转一个数。
为了让损失最小，
应该翻转绝对值最小的数。

## 思路一：两次排序

最直观的做法是：

1. 按数值从小到大排序。
2. 从最小的负数开始，尽量把负数翻成正数。
3. 如果 `k` 还有剩余并且是奇数，再排序一次。
4. 翻转当前最小的数。
5. 求和。

## 两次排序代码

```go
import "sort"

func largestSumAfterKNegationsTwoSorts(nums []int, k int) int {
    sort.Ints(nums)

    for i := 0; i < len(nums) && k > 0; i++ {
        if nums[i] < 0 {
            nums[i] = -nums[i]
            k--
        } else {
            break
        }
    }

    sort.Ints(nums)

    if k%2 == 1 {
        nums[0] = -nums[0]
    }

    sum := 0
    for _, num := range nums {
        sum += num
    }
    return sum
}
```

这个方法正确，
但第二次排序只是为了找到最小值，
有些浪费。

## 思路二：一次排序加一次遍历

第二次排序可以换成一次遍历。

翻转负数后，
如果 `k` 还剩奇数次，
只需要找到当前数组中的最小值，
然后让总和减去 `2 * minVal`。

## 一次排序代码

```go
import (
    "math"
    "sort"
)

func largestSumAfterKNegationsOneSort(nums []int, k int) int {
    sort.Ints(nums)

    for i := 0; i < len(nums) && k > 0; i++ {
        if nums[i] < 0 {
            nums[i] = -nums[i]
            k--
        } else {
            break
        }
    }

    sum := 0
    minVal := math.MaxInt
    for _, num := range nums {
        if num < minVal {
            minVal = num
        }
        sum += num
    }

    if k%2 == 1 {
        sum -= 2 * minVal
    }

    return sum
}
```

## 思路三：按绝对值排序

更直接的做法是按绝对值从大到小排序。

这样有两个好处：

- 绝对值大的负数会排在前面，可以优先翻转。
- 排序后最后一个元素就是绝对值最小的数。

遍历数组时，
遇到负数且 `k > 0` 就翻转。

如果遍历结束后 `k` 还剩奇数次，
直接翻转最后一个元素。

## 绝对值排序代码

```go
import (
    "math"
    "sort"
)

func largestSumAfterKNegations(nums []int, k int) int {
    sort.Slice(nums, func(i, j int) bool {
        return math.Abs(float64(nums[i])) > math.Abs(float64(nums[j]))
    })

    for i := 0; i < len(nums); i++ {
        if nums[i] < 0 && k > 0 {
            nums[i] = -nums[i]
            k--
        }
    }

    if k%2 == 1 {
        nums[len(nums)-1] = -nums[len(nums)-1]
    }

    sum := 0
    for _, num := range nums {
        sum += num
    }
    return sum
}
```

## 本篇总结

这道题的贪心目标始终围绕绝对值：

- 先翻转绝对值大的负数，让收益最大。
- 如果还必须翻转一次，就翻转绝对值最小的数，让损失最小。

按绝对值从大到小排序，
可以同时满足这两个需求。

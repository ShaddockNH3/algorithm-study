# 435 无重叠区间

[题目链接](https://leetcode.cn/problems/non-overlapping-intervals/)

## 题目描述

给定一个区间集合 `intervals`，
其中 `intervals[i] = [starti, endi]`。

返回需要移除区间的最小数量，
使剩余区间互不重叠。

注意，
只在一点上接触的区间不算重叠。
例如 `[1, 2]` 和 `[2, 3]` 是不重叠的。

## 核心贪心策略

第一步先排序。

为了方便比较相邻区间，
可以按左边界从小到大排序。

当两个区间发生重叠时，
必须移除其中一个。

贪心选择是：

保留结束更早的区间，
移除结束更晚的区间。

因为结束更早，
意味着后面留出的空间更大，
更有可能容纳更多区间。

## 思路一：修改数组传递状态

这个思路是在原数组上记录当前保留下来的区间。

排序后，从第二个区间开始比较 `intervals[i]` 和 `intervals[i-1]`。

如果发生重叠：

- 移除数量 `count++`。
- 保留右边界更小的那个区间。
- 如果前一个区间结束更早，就用它覆盖当前区间。

这样下一轮循环中，
`intervals[i]` 就代表上一次保留下来的区间，
可以继续和 `intervals[i+1]` 比较。

## 修改数组代码

```go
import "sort"

func eraseOverlapIntervalsModifyArray(intervals [][]int) int {
    if len(intervals) < 2 {
        return 0
    }

    sort.Slice(intervals, func(i, j int) bool {
        return intervals[i][0] < intervals[j][0]
    })

    count := 0
    for i := 1; i < len(intervals); i++ {
        if intervals[i][0] < intervals[i-1][1] {
            count++
            if intervals[i][1] > intervals[i-1][1] {
                intervals[i] = intervals[i-1]
            }
        }
    }

    return count
}
```

这个写法空间开销小，
但遍历时修改原数组，
逻辑会比较绕。

## 思路二：变量记录右边界

更清晰的写法是用一个变量记录当前保留区间的右边界。

`endBoundary` 表示当前已保留区间的最右边界。

遍历新区间时：

- 如果 `intervals[i][0] >= endBoundary`，说明不重叠，可以保留。
- 如果 `intervals[i][0] < endBoundary`，说明重叠，必须移除一个。

发生重叠时，
更新 `endBoundary` 为两个右边界中更小的那个，
这就等价于保留结束更早的区间。

## 右边界代码

```go
import "sort"

func eraseOverlapIntervals(intervals [][]int) int {
    if len(intervals) < 2 {
        return 0
    }

    sort.Slice(intervals, func(i, j int) bool {
        return intervals[i][0] < intervals[j][0]
    })

    count := 0
    endBoundary := intervals[0][1]

    for i := 1; i < len(intervals); i++ {
        if intervals[i][0] < endBoundary {
            count++
            if intervals[i][1] < endBoundary {
                endBoundary = intervals[i][1]
            }
        } else {
            endBoundary = intervals[i][1]
        }
    }

    return count
}
```

## 本篇总结

这道题的核心是重叠时保留结束更早的区间。

可以通过修改数组传递状态，
也可以用变量记录当前保留区间的右边界。

后者更清晰，
因为状态集中在 `endBoundary` 中，
不需要改变原始区间数组。

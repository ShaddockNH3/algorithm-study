# 53 最大子数组和

[题目链接](https://leetcode.cn/problems/maximum-subarray/)

## 题目描述

给你一个整数数组 `nums`，
请你找出一个具有最大和的连续子数组，
返回其最大和。

子数组最少包含一个元素。

## 解法一：前缀和加滑动窗口

最初的思路是先计算前缀和，
再用左右指针维护当前考察的连续子数组。

前缀和数组 `sums` 中，
`sums[i]` 表示 `nums[0]` 到 `nums[i]` 的总和。

这样就可以用 `O(1)` 时间计算任意区间和：

```go
sums[j] - sums[i-1]
```

滑动窗口中：

- `left` 表示当前子数组起点。
- `right` 表示当前子数组终点。

如果当前窗口和为负，
它对后续任何延伸都会是拖累，
所以应该放弃这段区间，
让下一段从 `right + 1` 重新开始。

## 前缀和代码

```go
import "math"

func maxSubArrayPrefixSum(nums []int) int {
    if len(nums) == 0 {
        return 0
    }

    sums := getSums(nums)

    maxSize := math.MinInt
    left, right := 0, 0

    for right < len(nums) {
        if left > right {
            right = left
            if right >= len(nums) {
                break
            }
        }

        currentSum := getSub(sums, left, right)

        if currentSum > maxSize {
            maxSize = currentSum
        }

        if currentSum < 0 {
            left = right + 1
        }

        right++
    }
    return maxSize
}

func getSums(nums []int) []int {
    sums := make([]int, len(nums))
    sums[0] = nums[0]
    for i := 1; i < len(nums); i++ {
        sums[i] = sums[i-1] + nums[i]
    }
    return sums
}

func getSub(sums []int, i int, j int) int {
    if i == 0 {
        return sums[j]
    }
    return sums[j] - sums[i-1]
}
```

这个解法直观，
但需要额外的前缀和数组，
空间复杂度是 `O(n)`，
代码也相对复杂。

## 解法二：Kadane 算法

经典解法只需要两个变量：

- `maxSoFar`：记录目前见过的最大子数组和。
- `currentSum`：记录当前连续子数组的和。

遍历每个数字时，
先把它加入 `currentSum`，
再更新 `maxSoFar`。

如果 `currentSum` 变成负数，
说明当前这段连续子数组对后续只会产生负贡献，
可以把它丢掉，
从下一个位置重新开始。

## Kadane 代码

```go
import "math"

func maxSubArray(nums []int) int {
    if len(nums) == 0 {
        return 0
    }

    maxSoFar := math.MinInt
    currentSum := 0

    for _, num := range nums {
        currentSum += num

        if currentSum > maxSoFar {
            maxSoFar = currentSum
        }

        if currentSum < 0 {
            currentSum = 0
        }
    }

    return maxSoFar
}
```

## 本篇总结

前缀和加滑动窗口能体现直观思路：
如果当前区间和为负，就从下一个位置重新开始。

Kadane 算法把这个想法压缩到两个变量里，
时间复杂度 `O(n)`，
空间复杂度 `O(1)`。

核心判断是：
负数的当前和只会拖累后续子数组，
所以可以舍弃。

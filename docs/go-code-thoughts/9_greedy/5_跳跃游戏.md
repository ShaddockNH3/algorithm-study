# 55 跳跃游戏

[题目链接](https://leetcode.cn/problems/jump-game/)

## 题目描述

给你一个非负整数数组 `nums`，
最初位于数组的第一个下标。

数组中的每个元素代表在该位置可以跳跃的最大长度。

判断是否能够到达最后一个下标。

## 思路一：剩余步数模拟

最初的思路是维护一个变量 `currentFuel`，
表示走到当前位置时还剩多少可用步数。

从左向右遍历时：

1. 到达位置 `i` 后，比较当前剩余步数和 `nums[i]`。
2. 选择更大的值作为当前位置之后可用的跳跃能力。
3. 如果在到达终点前剩余步数为 `0`，说明被卡住。
4. 每向前走一步，就消耗 1 点剩余步数。

## 剩余步数代码

```go
func canJumpFuelMethod(nums []int) bool {
    if len(nums) == 1 {
        return true
    }

    currentFuel := 0
    for i := 0; i < len(nums)-1; i++ {
        if nums[i] > currentFuel {
            currentFuel = nums[i]
        }

        if currentFuel == 0 {
            return false
        }

        currentFuel--
    }

    return true
}
```

这个方法能工作，
但 `currentFuel` 的更新和消耗交织在一起，
理解时需要小心。

## 思路二：维护最远可达位置

更清晰的贪心视角是：
维护当前能到达的最远下标 `maxReach`。

遍历每个位置 `i` 时：

- 如果 `i > maxReach`，说明当前位置不可达，返回 `false`。
- 否则可以从 `i` 出发，尝试更新 `maxReach = max(maxReach, i+nums[i])`。
- 如果 `maxReach` 已经覆盖最后一个下标，直接返回 `true`。

这个视角不关心还剩多少步，
只关心可达范围能扩张到哪里。

## 最远距离代码

```go
func canJump(nums []int) bool {
    maxReach := 0

    for i, jumpPower := range nums {
        if i > maxReach {
            return false
        }

        newReach := i + jumpPower
        if newReach > maxReach {
            maxReach = newReach
        }

        if maxReach >= len(nums)-1 {
            return true
        }
    }

    return true
}
```

## 本篇总结

跳跃游戏的关键是覆盖范围。

只要遍历到的位置都在当前可达范围内，
就可以用这个位置继续扩张最远可达下标。

一旦某个位置超过 `maxReach`，
说明无法到达它，
更不可能通过它继续前进。

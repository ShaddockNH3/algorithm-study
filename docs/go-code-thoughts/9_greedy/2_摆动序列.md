# 376 摆动序列

[题目链接](https://leetcode.cn/problems/wiggle-subsequence/)

## 题目描述

如果连续数字之间的差严格地在正数和负数之间交替，
则数字序列称为摆动序列。

第一个差可以是正数或负数。
仅有一个元素，
或者有两个不相等元素的序列，
也视作摆动序列。

子序列可以通过从原始序列中删除一些元素获得，
剩下元素保持原始顺序。

给定整数数组 `nums`，
返回 `nums` 中作为摆动序列的最长子序列长度。

## 最初的直觉

一开始的想法是：

1. 找到第一个方向，上坡或下坡。
2. 用 `flag` 记录期待的下一个方向。
3. 遍历数组，如果遇到期待方向，就计数并反转 `flag`。

这个方向本身不坏，
但需要处理很多边界：

- 数组长度为 1。
- 数组长度为 2。
- 开头有相等数字。
- 中间出现连续相等的平地。

当 `if-else` 越补越多时，
说明需要重新抓核心状态。

## 贪心思路

这道题找的是子序列，
所以可以跳过某些数字。

为了让摆动子序列尽可能长，
应该尽量保留所有拐点。

对于连续上升的一段，
只需要保留最后的最高点。

对于连续下降的一段，
只需要保留最后的最低点。

因此可以把问题转成：
统计相邻差值符号发生有效变化的次数。

## 核心状态

只需要两个变量：

- `count`：当前最长摆动序列长度。
- `prevDiff`：上一个有效坡度。

初始时，
第一个元素可以作为序列起点，
所以 `count = 1`。

`prevDiff = 0` 表示还没有有效坡度，
也可以理解为站在平地上，
第一步向上或向下都可以。

遍历时计算当前差值 `currDiff`。

如果满足以下任意一种情况，
说明找到了新的拐点：

- `currDiff > 0 && prevDiff <= 0`
- `currDiff < 0 && prevDiff >= 0`

找到拐点时，`count++`，
并更新 `prevDiff = currDiff`。

注意只在找到拐点时更新 `prevDiff`。
这样可以正确处理连续上坡、连续下坡和平地。

## 最终代码

```go
func wiggleMaxLength(nums []int) int {
    if len(nums) <= 1 {
        return len(nums)
    }

    count := 1
    prevDiff := 0

    for i := 1; i < len(nums); i++ {
        currDiff := nums[i] - nums[i-1]

        if (currDiff > 0 && prevDiff <= 0) ||
            (currDiff < 0 && prevDiff >= 0) {
            count++
            prevDiff = currDiff
        }
    }

    return count
}
```

## 本篇总结

摆动序列不需要复杂地记录期待方向。

真正关键的是当前有效坡度是否和上一个有效坡度相反。

平地可以自然融入 `prevDiff <= 0` 和 `prevDiff >= 0` 的判断中。
连续上坡或连续下坡只保留最后的峰值或谷值。

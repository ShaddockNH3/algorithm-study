# 455 分发饼干

[题目链接](https://leetcode.cn/problems/assign-cookies/)

## 题目描述

每个孩子最多只能分到一块饼干。

孩子 `i` 有胃口值 `g[i]`，
饼干 `j` 有尺寸 `s[j]`。

如果 `s[j] >= g[i]`，
这块饼干就可以满足这个孩子。

目标是满足尽可能多的孩子，
返回最大满足数量。

## 最初的暴力思路

最直观的想法是：
对每个孩子，都遍历所有饼干，
找出能满足他的最小饼干。

找到后，把这块饼干标记为已使用，
避免重复分配。

## 暴力代码

```go
func findContentChildren(g []int, s []int) int {
    cnt := 0
    for _, child := range g {
        minNum := getMin(s, child)
        if minNum != math.MaxInt32 {
            cnt++
        }
    }
    return cnt
}

func getMin(s []int, child int) int {
    minNum := math.MaxInt32
    index := -1

    for i := 0; i < len(s); i++ {
        if s[i] >= child {
            if minNum >= s[i] {
                minNum = s[i]
                index = i
            }
        }
    }

    if index != -1 {
        s[index] = math.MaxInt32
    }
    return minNum
}
```

这个方法容易理解，
但每个孩子都要扫描所有饼干，
时间复杂度接近 `O(len(g) * len(s))`。

## 贪心思路

优化的关键是先排序。

把孩子胃口和饼干尺寸都从小到大排序后：

1. 优先满足胃口最小的孩子。
2. 用能满足他的最小饼干。

这样可以把大饼干留给胃口更大的孩子。

如果当前最小饼干不能满足当前胃口最小的孩子，
它也不可能满足后面胃口更大的孩子，
所以只能跳过这块饼干。

## 贪心代码

```go
import "sort"

func findContentChildren(g []int, s []int) int {
    sort.Ints(g)
    sort.Ints(s)

    child, cookie := 0, 0
    cnt := 0

    for child < len(g) && cookie < len(s) {
        if g[child] <= s[cookie] {
            child++
            cookie++
            cnt++
        } else {
            cookie++
        }
    }

    return cnt
}
```

## 本篇总结

这道题的贪心策略是：
用尽量小的饼干满足当前胃口最小的孩子。

排序让这个策略可以用双指针自然实现。

如果当前饼干太小，
它对当前孩子和后面孩子都无效，
可以直接舍弃。

# 45 跳跃游戏 II

[题目链接](https://leetcode.cn/problems/jump-game-ii/)

## 题目描述

给定一个长度为 `n` 的 0 索引整数数组 `nums`。
初始位置为 `nums[0]`。

每个元素 `nums[i]` 表示从索引 `i` 向后跳转的最大长度。

返回到达 `nums[n-1]` 的最小跳跃次数。

测试用例保证可以到达最后一个位置。

## 最初的想法

一开始容易想到：

站在当前位置 `i`，
扫描所有能跳到的位置，
找出其中下一次能跳得最远的落脚点，
然后跳过去。

这个思路是有贪心味道的，
但实现上容易变成嵌套循环，
还要非常小心更新下标。

后来视角转变为：
不需要关心具体跳到哪一个点，
只需要关心当前这一跳能覆盖到哪里，
以及在这个覆盖范围内能为下一跳扩张出多远的边界。

## 边界扩张思路

维护三个变量：

- `jumps`：已经使用的跳跃次数。
- `currentEnd`：当前跳跃次数能够覆盖到的最远边界。
- `farthest`：在当前边界内扫描时，下一跳最远能到的位置。

遍历数组时，
`for` 循环不是在模拟每一次真实跳跃，
而是在扫描当前可达范围内的每个位置。

对每个位置 `i`，
用 `i + nums[i]` 更新 `farthest`。

当 `i == currentEnd` 时，
说明当前这一层的所有位置都已经考察完。
此时必须增加一次跳跃，
并把新的边界设为 `farthest`。

这很像 BFS 的层序遍历：
每次 `jumps++`，
相当于进入下一层可达范围。

## 最终代码

```go
func jump(nums []int) int {
    if len(nums) <= 1 {
        return 0
    }

    jumps := 0
    currentEnd := 0
    farthest := 0

    for i := 0; i < len(nums)-1; i++ {
        if i+nums[i] > farthest {
            farthest = i + nums[i]
        }

        if i == currentEnd {
            jumps++
            currentEnd = farthest
        }
    }

    return jumps
}
```

## 本篇总结

跳跃游戏 II 的贪心点不是每一步立刻跳到某个具体位置，
而是在当前可达范围内扫描所有可能，
找出下一次跳跃能扩张出的最远边界。

只在走到当前边界时才增加跳跃次数。
这能保证每一次跳跃都覆盖尽可能远的范围。

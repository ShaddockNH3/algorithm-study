# 738 单调递增的数字

[题目链接](https://leetcode.cn/problems/monotone-increasing-digits/)

## 题目描述

当且仅当每个相邻位数上的数字 `x` 和 `y`
都满足 `x <= y` 时，
这个整数是单调递增的。

给定一个整数 `n`，
返回小于或等于 `n` 的最大数字，
并且这个数字单调递增。

## 核心贪心策略

从右往左找到破坏单调递增规则的位置。

一旦某一位比它右边的数字大，
就需要把这一位减一，
并把它右边的所有位都改成 `9`。

这样能保证结果不超过原数，
同时尽可能大。

## 思路一：从左到右，发现后回溯

原先的思路是从左到右遍历。

当发现 `digits[i-1] > digits[i]` 时，
说明这里出现了断点。

但真正需要减一的位置，
可能还要继续向左追溯。

例如 `1332`：
在第二个 `3` 和 `2` 之间发现问题，
但需要修改的是第一个 `3`，
得到 `1299`。

所以发现问题后，
要向左找到连续相等数字的起点，
把这个起点减一，
再把它之后的所有位改成 `9`。

## 回溯修正代码

```go
import "strconv"

func monotoneIncreasingDigitsOriginal(n int) int {
    digits := intToSlice(n)

    for i := 1; i < len(digits); i++ {
        if digits[i-1] > digits[i] {
            for i >= 2 && digits[i-1] == digits[i-2] {
                i--
            }

            digits[i-1]--
            for j := i; j < len(digits); j++ {
                digits[j] = 9
            }
            return sliceToInt(digits)
        }
    }

    return n
}

func intToSlice(n int) []int {
    s := strconv.Itoa(n)
    digits := make([]int, len(s))

    for i, ch := range s {
        digits[i] = int(ch - '0')
    }

    return digits
}

func sliceToInt(digits []int) int {
    res := 0
    for _, digit := range digits {
        res = res*10 + digit
    }
    return res
}
```

这个写法能体现发现问题后追溯源头的过程。
但它会修改外层循环变量，
控制流相对复杂。

## 思路二：从右到左一次定位

更常见的写法是从右往左遍历。

如果发现 `s[i] > s[i+1]`，
就把 `s[i]` 减一，
并记录从 `i + 1` 开始都要变成 `9`。

因为遍历方向是从右到左，
最后一次记录的位置，
就是最靠左、最根本的修改点。

遍历结束后，
再统一把记录位置之后的所有数字改成 `9`。

## 从右往左代码

```go
import "strconv"

func monotoneIncreasingDigits(n int) int {
    s := []byte(strconv.Itoa(n))
    nineStartIndex := len(s)

    for i := len(s) - 2; i >= 0; i-- {
        if s[i] > s[i+1] {
            s[i]--
            nineStartIndex = i + 1
        }
    }

    for i := nineStartIndex; i < len(s); i++ {
        s[i] = '9'
    }

    res, _ := strconv.Atoi(string(s))
    return res
}
```

## 两种思路对比

从左到右写法：
发现断点后再向左追溯，
比较符合直觉，
但代码控制流更绕。

从右到左写法：
一边倒序扫描一边记录最终要变成 `9` 的起点，
最后统一修改，
代码更清晰。

## 本篇总结

单调递增数字的贪心关键是：

遇到左边数字大于右边数字时，
必须让左边这一位变小，
并让右侧尽可能大。

右侧尽可能大的方式就是全部置为 `9`。

从右往左扫描可以自然找到最靠左的修改点，
因此通常是更推荐的实现。

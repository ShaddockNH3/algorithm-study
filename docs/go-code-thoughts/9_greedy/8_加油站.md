# 134 加油站

[题目链接](https://leetcode.cn/problems/gas-station/)

## 题目描述

在一条环路上有 `n` 个加油站，
第 `i` 个加油站有汽油 `gas[i]` 升。

从第 `i` 个加油站开往第 `i+1` 个加油站，
需要消耗汽油 `cost[i]` 升。

从某个加油站出发时油箱为空。

如果可以按顺序绕环路行驶一周，
返回出发加油站的编号；
否则返回 `-1`。

如果存在解，题目保证它唯一。

## 暴力思路

最直观的方法是：
把每一个加油站都当成起点试一次。

外层循环枚举起点，
内层循环模拟从这个起点出发行驶一整圈。

如果中途油量小于 0，
说明这个起点失败，
继续尝试下一个起点。

这个思路是正确的，
但时间复杂度是 `O(n^2)`，
数据量大时容易超时。

## 贪心洞察一：全局油量

如果所有加油站的总油量小于总消耗，
无论从哪里出发都不可能绕一圈。

也就是：

```text
sum(gas) < sum(cost)
```

时必然无解。

## 贪心洞察二：失败区间可以整体排除

如果从 `A` 站出发，
在到达 `B` 站之前油量变成负数，
那么从 `A` 到 `B` 之间的任何一个站出发，
都不可能到达 `B`。

原因是：
从 `A` 出发时，已经累积了这一段能获得的全部油量。
这种相对更有利的状态都失败了，
从中间某个站出发只会更差。

所以一旦当前起点失败，
可以把起点直接跳到失败位置的下一个站。

## 一次遍历思路

维护三个变量：

- `totalTank`：全程总油量盈亏，用于最终判断是否有解。
- `currentTank`：从当前候选起点出发后的当前油量。
- `startStation`：当前候选起点。

遍历每个站时，
计算：

```go
diff := gas[i] - cost[i]
```

这个差值同时加入 `totalTank` 和 `currentTank`。

如果 `currentTank < 0`，
说明从当前 `startStation` 出发到不了下一站，
根据失败区间整体排除的结论，
新的起点只能是 `i + 1`，
并把 `currentTank` 重置为 `0`。

遍历结束后，
如果 `totalTank < 0` 返回 `-1`。
否则最后留下的 `startStation` 就是答案。

## 最终代码

```go
func canCompleteCircuit(gas []int, cost []int) int {
    totalTank := 0
    currentTank := 0
    startStation := 0

    for i := 0; i < len(gas); i++ {
        diff := gas[i] - cost[i]
        totalTank += diff
        currentTank += diff

        if currentTank < 0 {
            startStation = i + 1
            currentTank = 0
        }
    }

    if totalTank < 0 {
        return -1
    }

    return startStation
}
```

## 本篇总结

加油站问题从暴力到贪心的关键是：
失败的起点不是一个点，
而是一整段都可以排除。

同时用 `totalTank` 判断全局是否有解，
用 `currentTank` 判断当前候选起点是否失败。

一旦当前起点失败，
直接把起点推进到失败位置的下一站。

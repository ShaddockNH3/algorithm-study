# 替换数字：Builder 与从后往前填充

替换数字是一道字符串构建题。
目标是保留字母，把每个数字字符替换成 `number`。

这篇整理两种思路：

- 使用 `strings.Builder` 从前往后构建。
- 先预留空间，再从后往前填充。

## 卡码网 替换数字

[题目链接](https://kamacoder.com/problempage.php?pid=1064)

### 题意

给定一个字符串 `s`，其中包含小写字母和数字字符。

将字符串中的字母字符保持不变，
并将每个数字字符替换为 `number`。

## 方法一：strings.Builder

### Builder 思路

在 Go 中，频繁拼接字符串时适合使用 `strings.Builder`。

处理步骤：

1. 从前往后遍历字符串。
2. 如果当前字符是数字，写入 `number`。
3. 如果当前字符是字母，写入字符本身。
4. 最后输出 `builder.String()`。

这种方式代码直观，适合日常字符串构建。

### Builder 代码

```go
package main

import (
    "fmt"
    "strings"
)

func solve() {
    var s string
    fmt.Scanln(&s)

    var builder strings.Builder

    for _, char := range s {
        if char >= '0' && char <= '9' {
            builder.WriteString("number")
        } else {
            builder.WriteRune(char)
        }
    }

    fmt.Println(builder.String())
}

func main() {
    solve()
}
```

### Builder 特点

- 代码清晰。
- 从前往后处理，符合直觉。
- 避免循环中直接使用 `+` 拼接字符串。

## 方法二：从后往前填充

### 填充思路

这个方法先计算最终字符串长度，再一次性分配空间。

处理步骤：

1. 遍历原字符串，统计数字数量 `cnt`。
2. 每个数字会从 1 个字符变成 6 个字符。
3. 最终长度是 `len(s) - cnt + cnt * 6`。
4. 创建对应长度的 `[]byte`。
5. 使用两个指针从后往前填充：
   - `strIndex` 指向原字符串末尾。
   - `ansIndex` 指向结果切片末尾。
6. 如果遇到数字，就逆序写入 `number`。
7. 如果遇到字母，就写入原字符。

从后往前填充的好处是：扩容后不会覆盖还没处理的原始字符。

### 填充代码

```go
package main

import "fmt"

func solve() {
    var str string
    fmt.Scanln(&str)

    cnt := 0
    for _, c := range str {
        if c >= '0' && c <= '9' {
            cnt++
        }
    }

    ansLength := len(str) - cnt + cnt*6
    ans := make([]byte, ansLength)
    ansIndex := ansLength - 1

    for strIndex := len(str) - 1; strIndex >= 0; strIndex-- {
        char := str[strIndex]
        if char >= '0' && char <= '9' {
            ans[ansIndex] = 'r'
            ansIndex--
            ans[ansIndex] = 'e'
            ansIndex--
            ans[ansIndex] = 'b'
            ansIndex--
            ans[ansIndex] = 'm'
            ansIndex--
            ans[ansIndex] = 'u'
            ansIndex--
            ans[ansIndex] = 'n'
            ansIndex--
        } else {
            ans[ansIndex] = char
            ansIndex--
        }
    }

    fmt.Println(string(ans))
}

func main() {
    solve()
}
```

### 填充特点

- 只需要一次主要内存分配。
- 有助于理解扩容后从后往前处理的技巧。
- 对下标控制要求更高。

## 复杂度

两种方法的复杂度相同：

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

## 本题总结

如果追求代码简洁，使用 `strings.Builder` 更直接。

如果想练习空间预留和从后往前填充，
可以使用第二种写法。

核心是先明确替换后的长度变化：
每个数字从 1 个字符变成 6 个字符。

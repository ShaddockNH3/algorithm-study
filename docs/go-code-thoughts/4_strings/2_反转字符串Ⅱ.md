# 反转字符串 II：按固定步长处理区间

反转字符串 II 的重点不是复杂算法，
而是把题目规则准确翻译成代码。

核心规律是：每隔 `2k` 个字符，反转前 `k` 个字符。

## 541 反转字符串 II

[题目链接](https://leetcode.cn/problems/reverse-string-ii/)

### 541 题意

给定一个字符串 `s` 和一个整数 `k`。

从字符串开头算起，每计数至 `2k` 个字符，
就反转这 `2k` 个字符中的前 `k` 个字符。

规则如下：

- 如果剩余字符少于 `k` 个，则将剩余字符全部反转。
- 如果剩余字符小于 `2k` 但大于或等于 `k` 个，
  则反转前 `k` 个字符，其余字符保持原样。

## 核心思路

题目已经给出了处理节奏：每次处理一个长度为 `2k` 的大区间。

因此循环可以写成：

```go
for i := 0; i < n; i += 2 * k
```

其中 `i` 表示当前大区间的起始位置。

在每个大区间中，需要反转的是：

```text
[i, i+k-1]
```

但最后一个区间可能不足 `k` 个字符。
所以右边界不能直接写成 `i+k-1`，而应写成：

```go
right := min(i+k-1, n-1)
```

这样可以统一处理所有边界情况。

## 541 代码

```go
func reverseString(s []byte, left int, right int) {
    for left < right {
        s[left], s[right] = s[right], s[left]
        left++
        right--
    }
}

func reverseStr(s string, k int) string {
    t := []byte(s)
    n := len(s)

    for i := 0; i < n; i += 2 * k {
        right := min(i+k-1, n-1)
        reverseString(t, i, right)
    }

    return string(t)
}

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}
```

## 边界分析

### 剩余字符不少于 k 个

以 `s = "abcdefg"`，`k = 2` 为例。

第一轮：

```text
i = 0
right = min(0+2-1, 6) = 1
反转 ab，得到 bacdefg
```

第二轮：

```text
i = 4
right = min(4+2-1, 6) = 5
反转 ef，得到 bacdfeg
```

### 剩余字符少于 k 个

以 `s = "abcdefg"`，`k = 4` 为例。

处理到 `i = 4` 时，剩余字符是 `efg`。

理论右边界是：

```text
4 + 4 - 1 = 7
```

但字符串最大下标是 `6`。

因此：

```text
right = min(7, 6) = 6
```

翻转区间 `[4, 6]`，也就是把所有剩余字符全部反转。

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

Go 字符串不可变，代码中需要先转成 `[]byte`，
因此额外空间为 `O(n)`。

## 本题总结

这道题的关键是把规则转成稳定的区间处理。

每轮循环处理一个 `2k` 区间，
但真正反转的只是这个区间的前 `k` 个字符。

使用 `min(i+k-1, n-1)` 计算右边界，
可以避免为最后不足 `k` 个字符的情况单独写分支。

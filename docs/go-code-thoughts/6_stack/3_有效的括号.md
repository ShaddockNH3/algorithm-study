# 有效的括号：栈的后进先出应用

有效括号是栈的经典应用。
括号匹配天然符合后进先出的规则：
最后出现的左括号，应该最先被匹配。

## 20 有效的括号

[题目链接](https://leetcode.cn/problems/valid-parentheses/)

### 20 题意

给定一个只包括以下字符的字符串 `s`：

- `(`
- `)`
- `{`
- `}`
- `[`
- `]`

判断字符串是否有效。

有效字符串需要满足：

1. 左括号必须用相同类型的右括号闭合。
2. 左括号必须以正确顺序闭合。
3. 每个右括号都有对应的同类型左括号。

## 栈思路

### 为什么用栈

括号匹配要求最近打开的左括号最先闭合。

例如：

```text
({[]})
```

最里面的 `[` 必须先被 `]` 匹配。
这正好符合栈的后进先出特性。

### 处理规则

遍历字符串时：

1. 如果遇到左括号，直接入栈。
2. 如果遇到右括号，检查栈顶是否是对应的左括号。
3. 如果栈为空，说明右括号没有对应左括号，返回 `false`。
4. 如果栈顶不匹配，返回 `false`。
5. 如果匹配，弹出栈顶。
6. 遍历结束后，栈必须为空。

如果字符串长度是奇数，可以直接返回 `false`。
因为有效括号一定成对出现。

## 20 代码

```go
func isValid(s string) bool {
    if len(s)%2 == 1 {
        return false
    }

    pairs := map[byte]byte{
        ')': '(',
        ']': '[',
        '}': '{',
    }

    stack := []byte{}

    for i := 0; i < len(s); i++ {
        char := s[i]

        if _, ok := pairs[char]; ok {
            if len(stack) == 0 || stack[len(stack)-1] != pairs[char] {
                return false
            }
            stack = stack[:len(stack)-1]
        } else {
            stack = append(stack, char)
        }
    }

    return len(stack) == 0
}
```

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

最坏情况下，所有字符都是左括号，需要全部入栈。

## 本题总结

遇到成对匹配、最近打开的结构需要最先关闭的问题，
可以优先考虑栈。

这道题中：

- 左括号入栈。
- 右括号检查栈顶。
- 匹配则出栈。
- 最后栈为空才有效。

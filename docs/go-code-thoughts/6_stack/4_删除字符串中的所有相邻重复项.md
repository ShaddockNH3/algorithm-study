# 删除字符串中的所有相邻重复项：用栈模拟消除

删除相邻重复项适合用栈处理。

原因是每次只需要判断当前字符是否和最后一个未被消除的字符相同。

## 1047 删除字符串中的所有相邻重复项

[题目链接](https://leetcode.cn/problems/remove-all-adjacent-duplicates-in-string/)

### 1047 题意

给定一个由小写字母组成的字符串 `s`。

重复项删除操作会选择两个相邻且相同的字母，并删除它们。

在 `s` 上反复执行删除操作，直到无法继续删除。
返回最终字符串，答案保证唯一。

示例：

```text
输入：abbaca
输出：ca
```

## 栈思路

遍历字符串时，用 `[]byte` 模拟栈。

处理规则：

1. 如果栈为空，当前字符入栈。
2. 如果当前字符和栈顶字符不同，当前字符入栈。
3. 如果当前字符和栈顶字符相同，弹出栈顶。

弹出栈顶相当于消除一对相邻重复字符。

## 1047 代码

```go
func removeDuplicates(s string) string {
    stack := []byte{}

    for i := 0; i < len(s); i++ {
        char := s[i]

        if len(stack) > 0 && stack[len(stack)-1] == char {
            stack = stack[:len(stack)-1]
        } else {
            stack = append(stack, char)
        }
    }

    return string(stack)
}
```

## 示例过程

以 `abbaca` 为例。

遇到 `a`：

```text
stack = [a]
```

遇到 `b`：

```text
stack = [a, b]
```

再次遇到 `b`，和栈顶相同，弹出：

```text
stack = [a]
```

遇到 `a`，和栈顶相同，弹出：

```text
stack = []
```

遇到 `c`：

```text
stack = [c]
```

遇到 `a`：

```text
stack = [c, a]
```

最终结果是 `ca`。

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

最坏情况下没有任何字符被消除，所有字符都会留在栈中。

## 本题总结

当问题只关心当前元素和最近保留下来的元素之间的关系时，
可以考虑栈。

这道题中，栈顶始终代表当前结果的最后一个字符。
当前字符要么和栈顶抵消，要么进入栈成为新的结尾。

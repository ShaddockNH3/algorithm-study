# 797 所有可能的路径

[题目链接](https://leetcode.cn/problems/all-paths-from-source-to-target/)

## 题目特点

题目给的是有向无环图，也就是 DAG。

- 有向：边只能按指定方向走。
- 无环：从一个节点出发不会沿着有向边绕回自己。

因为无环，
这道题不需要 `visited` 数组防止死循环。

目标是找出从 `0` 号节点到 `n-1` 号节点的所有路径。

## DFS 回溯思路

用 `path` 记录当前路径。

初始路径为：

```go
[]int{0}
```

从当前节点 `x` 出发，
遍历所有邻居 `node`：

1. 把 `node` 加入 `path`。
2. 递归搜索 `node`。
3. 递归返回后，把 `node` 从 `path` 中移除。

当 `x == n-1` 时，
说明找到一条完整路径，
需要复制 `path` 放入结果集。

## 为什么要 copy

Go 的切片底层共享数组。

如果直接把 `path` append 到结果集，
后续回溯修改 `path` 时，
结果集中的路径也可能被影响。

所以找到答案时要创建快照：

```go
tmp := make([]int, len(path))
copy(tmp, path)
```

## 最终代码

```go
func allPathsSourceTarget(graph [][]int) [][]int {
    result := [][]int{}
    path := []int{0}
    n := len(graph)

    var dfs func(x int)
    dfs = func(x int) {
        if x == n-1 {
            tmp := make([]int, len(path))
            copy(tmp, path)
            result = append(result, tmp)
            return
        }

        for _, node := range graph[x] {
            path = append(path, node)
            dfs(node)
            path = path[:len(path)-1]
        }
    }

    dfs(0)

    return result
}
```

## 本篇总结

DAG 中找所有路径适合 DFS 回溯。

因为图无环，
不需要额外 visited。

核心是维护当前路径，
到达终点时复制路径，
返回时撤销最后一步。

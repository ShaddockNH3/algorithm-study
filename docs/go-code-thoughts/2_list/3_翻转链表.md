# 反转链表：逐个改变 Next 指针

反转链表是链表操作中的基础题。
它的核心不是移动节点，而是改变节点之间的 `Next` 指针方向。

## 206 反转链表

[题目链接](https://leetcode.cn/problems/reverse-linked-list/)

### 206 题意

反转一个单链表。

示例：

```text
输入：1 -> 2 -> 3 -> 4 -> 5 -> nil
输出：5 -> 4 -> 3 -> 2 -> 1 -> nil
```

## 迭代解法

### 指针含义

迭代反转链表时，需要维护三个指针：

- `pre`：指向已经反转好的部分。
- `tmp`：指向当前要处理的节点。
- `nex`：临时保存 `tmp.Next`，避免链表断开后找不到后续节点。

每一轮循环都做三件事：

1. 用 `nex` 记录下一个节点。
2. 让 `tmp.Next` 指向 `pre`。
3. 同时向后移动 `pre` 和 `tmp`。

### 206 代码

```go
func reverseList(head *ListNode) *ListNode {
    if head == nil || head.Next == nil {
        return head
    }

    pre := head
    tmp := head.Next
    pre.Next = nil

    for tmp != nil {
        nex := tmp.Next
        tmp.Next = pre
        pre = tmp
        tmp = nex
    }

    return pre
}
```

### 过程示例

以 `1 -> 2 -> 3 -> nil` 为例。

初始状态：

```text
pre = 1
tmp = 2
```

先执行：

```go
pre.Next = nil
```

此时节点 `1` 成为新链表的尾节点。

第一次循环，`tmp` 指向 `2`：

1. `nex` 记录 `3`。
2. `tmp.Next = pre`，得到 `2 -> 1 -> nil`。
3. `pre` 移动到 `2`。
4. `tmp` 移动到 `3`。

第二次循环，`tmp` 指向 `3`：

1. `nex` 记录 `nil`。
2. `tmp.Next = pre`，得到 `3 -> 2 -> 1 -> nil`。
3. `pre` 移动到 `3`。
4. `tmp` 移动到 `nil`。

循环结束后，`pre` 指向新的头节点。

### 206 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

## 本题总结

反转链表时最容易出错的点是丢失后续节点。

因此每次修改 `tmp.Next` 之前，都要先用 `nex` 保存原来的后继节点。

整体规律是：

```text
保存后继 -> 反转指针 -> 移动 pre 和 tmp
```

掌握这个过程后，很多链表局部反转问题都可以在此基础上扩展。

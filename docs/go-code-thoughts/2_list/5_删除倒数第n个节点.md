# 删除链表的倒数第 N 个节点：快慢指针的距离差

这道题的学习抓手是：让快指针先偷跑一段距离。

只要快慢指针之间保持固定距离，
当快指针到达终点时，慢指针的位置就能反推出倒数第 N 个节点。

## 19 删除链表的倒数第 N 个结点

[题目链接](https://leetcode.cn/problems/remove-nth-node-from-end-of-list/)

### 19 题意

给定一个链表，删除链表的倒数第 `n` 个结点，
并返回链表的头结点。

进阶要求是使用一趟扫描实现。

示例：

```text
输入：head = [1,2,3,4,5], n = 2
输出：[1,2,3,5]
```

## 快慢指针思路

如果先遍历链表得到总长度 `L`，
再从头走 `L - n` 步找到要删除的节点，就需要两趟扫描。

一趟扫描的关键是双指针：

1. 创建快指针 `sec` 和慢指针 `fir`。
2. 先让快指针向前走一段距离。
3. 再让快慢指针同步向后移动。
4. 当快指针到达末尾时，慢指针正好停在目标附近。

如果目标是直接找到倒数第 `n` 个节点，
可以让快指针领先慢指针 `n` 步。

但删除节点时，更方便的是找到待删除节点的前驱节点。
因此从虚拟头节点出发时，可以让快指针先走 `n + 1` 步。

这样同步移动结束后，慢指针 `fir` 会停在待删除节点的前一个节点。

## 虚拟头节点

如果要删除的是原链表头节点，就没有真实的前驱节点。

例如：

```text
head = [1,2,3], n = 3
```

要删除的是节点 `1`。

使用虚拟头节点后，原头节点也有了一个前驱节点。
这样删除头节点和删除普通节点就能共用同一套逻辑。

## 19 代码

```go
func removeNthFromEnd(head *ListNode, n int) *ListNode {
    dummyHead := &ListNode{Val: -1, Next: head}
    fir, sec := dummyHead, dummyHead

    for i := 0; i <= n; i++ {
        sec = sec.Next
    }

    for sec != nil {
        fir = fir.Next
        sec = sec.Next
    }

    fir.Next = fir.Next.Next

    return dummyHead.Next
}
```

## 指针距离解释

快指针先走 `n + 1` 步后，
`sec` 和 `fir` 之间隔着 `n` 个节点。

接着两个指针一起移动。

当 `sec == nil` 时，说明快指针已经越过链表尾部。
此时慢指针 `fir` 正好位于倒数第 `n` 个节点的前驱位置。

删除操作就是：

```go
fir.Next = fir.Next.Next
```

## 复杂度

- 时间复杂度：`O(L)`
- 空间复杂度：`O(1)`

其中 `L` 是链表长度。

## 本题总结

删除倒数第 N 个节点的关键是维护快慢指针的距离差。

这道题同时体现了两个链表常用技巧：

1. 快慢指针用于一趟扫描定位倒数位置。
2. 虚拟头节点用于统一删除头节点和普通节点。

实际写代码时，要先决定慢指针最终停在哪里。
如果要停在待删除节点的前驱节点，就让快指针多走一步。

# 设计链表：结构封装与虚拟头节点

设计链表考察的是对链表底层结构的理解。
这道题不只是调用链表，而是要自己维护节点、长度和增删查逻辑。

## 707 设计链表

[题目链接](https://leetcode.cn/problems/design-linked-list/)

### 707 题意

可以选择使用单链表或双链表，设计并实现自己的链表。

需要实现 `MyLinkedList`：

- `MyLinkedList()` 初始化链表对象。
- `get(index)` 获取下标为 `index` 的节点值。
  如果下标无效，返回 `-1`。
- `addAtHead(val)` 在链表头部插入节点。
- `addAtTail(val)` 在链表尾部追加节点。
- `addAtIndex(index, val)` 在指定下标之前插入节点。
- `deleteAtIndex(index)` 删除指定下标的节点。

## 核心设计

### 节点结构

`ListNode` 是链表中的基础节点，只保存两类信息：

- 当前节点的值 `Val`。
- 指向下一个节点的指针 `Next`。

### 链表结构

`MyLinkedList` 负责管理整个链表状态：

- `size` 记录链表长度。
- `dummyHead` 是虚拟头节点，作为链表固定入口。

虚拟头节点不存储有效业务数据，也不计入 `size`。
它的作用是统一插入和删除逻辑。

### 虚拟头节点的作用

有了 `dummyHead`，所有操作都可以从同一个入口开始。

它带来的好处包括：

1. 空链表和非空链表的入口统一。
2. 在头部插入节点，相当于在 `dummyHead` 后插入。
3. 删除头节点，相当于修改 `dummyHead.Next`。
4. 减少对 `head == nil` 的特殊判断。

在 `AddAtIndex` 和 `DeleteAtIndex` 中，
代码都先找到目标位置的前驱节点，再修改前驱节点的 `Next`。

当 `index == 0` 时，这个前驱节点就是 `dummyHead`。

## 707 代码

```go
type ListNode struct {
    Val  int
    Next *ListNode
}

type MyLinkedList struct {
    size      int
    dummyHead *ListNode
}

func Constructor() MyLinkedList {
    return MyLinkedList{
        size:      0,
        dummyHead: &ListNode{},
    }
}

func (this *MyLinkedList) Get(index int) int {
    if index < 0 || index >= this.size {
        return -1
    }

    tmp := this.dummyHead.Next
    for i := 0; i < index; i++ {
        tmp = tmp.Next
    }

    return tmp.Val
}

func (this *MyLinkedList) AddAtHead(val int) {
    this.AddAtIndex(0, val)
}

func (this *MyLinkedList) AddAtTail(val int) {
    this.AddAtIndex(this.size, val)
}

func (this *MyLinkedList) AddAtIndex(index int, val int) {
    if index < 0 || index > this.size {
        return
    }

    tmp := this.dummyHead
    for i := 0; i < index; i++ {
        tmp = tmp.Next
    }

    newNode := &ListNode{Val: val, Next: tmp.Next}
    tmp.Next = newNode
    this.size++
}

func (this *MyLinkedList) DeleteAtIndex(index int) {
    if index < 0 || index >= this.size {
        return
    }

    tmp := this.dummyHead
    for i := 0; i < index; i++ {
        tmp = tmp.Next
    }

    tmp.Next = tmp.Next.Next
    this.size--
}
```

## 操作分析

### Get

`Get` 先判断下标是否合法。

如果合法，就从真实头节点 `dummyHead.Next` 开始移动 `index` 步。
最终停下的位置就是目标节点。

### AddAtHead

头插可以复用 `AddAtIndex(0, val)`。

这样可以避免为头插单独维护一套逻辑。

### AddAtTail

尾插可以复用 `AddAtIndex(this.size, val)`。

当插入位置等于当前长度时，新节点会被插入到末尾。

### AddAtIndex

插入时要找到插入位置的前驱节点。

找到前驱节点 `tmp` 后：

```go
newNode := &ListNode{Val: val, Next: tmp.Next}
tmp.Next = newNode
```

这两句完成了新节点和原链表的连接。

### DeleteAtIndex

删除时同样先找到待删除节点的前驱节点。

然后执行：

```go
tmp.Next = tmp.Next.Next
```

这会让前驱节点跳过待删除节点，直接指向后一个节点。

## 复杂度

- `Get` 时间复杂度：`O(n)`
- `AddAtHead` 时间复杂度：`O(1)`
- `AddAtTail` 时间复杂度：`O(n)`
- `AddAtIndex` 时间复杂度：`O(n)`
- `DeleteAtIndex` 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`，不计链表本身存储

## 本题总结

设计单链表时，最重要的是维护好三个东西：

1. 节点结构。
2. 链表长度。
3. 固定入口，也就是虚拟头节点。

虚拟头节点让头部插入、头部删除、中间插入和中间删除
都能使用同一套前驱节点逻辑。

# 两两交换链表中的节点：虚拟头节点与局部重连

两两交换节点考察的是链表指针重连。
这道题不能只交换节点值，而是要实际交换节点本身。

## 24 两两交换链表中的节点

[题目链接](https://leetcode.cn/problems/swap-nodes-in-pairs/)

### 24 题意

给定一个链表，两两交换其中相邻的节点，并返回交换后的链表。

限制条件：

- 不能只改变节点内部的值。
- 必须实际交换节点。

## 解法思路

### 虚拟头节点

因为第一对节点交换后，链表头节点会发生变化。

例如：

```text
1 -> 2 -> 3 -> 4
```

交换第一对后，新的头节点会变成 `2`。

使用虚拟头节点可以避免单独处理头节点变化。
`dummyHead` 作为固定入口，让每一组交换都可以统一处理。

### 游标指针

代码中的 `temp` 是游标指针。

它始终指向当前要交换的两个节点的前一个节点。

只有当 `temp.Next` 和 `temp.Next.Next` 都不为空时，
后面才至少有两个节点可以交换。

## 24 代码

```go
func swapPairs(head *ListNode) *ListNode {
    dummyHead := &ListNode{Next: head}
    temp := dummyHead

    for temp.Next != nil && temp.Next.Next != nil {
        node1 := temp.Next
        node2 := temp.Next.Next

        temp.Next = node2
        node1.Next = node2.Next
        node2.Next = node1

        temp = node1
    }

    return dummyHead.Next
}
```

## 四步交换过程

以局部链表为例：

```text
temp -> 1 -> 2 -> 3
```

此时：

- `node1` 指向 `1`。
- `node2` 指向 `2`。

### 第一步

执行：

```go
temp.Next = node2
```

让 `temp` 越过 `node1`，先指向 `node2`。

局部关系变成：

```text
temp -> 2
1 -> 2 -> 3
```

### 第二步

执行：

```go
node1.Next = node2.Next
```

让 `node1` 指向 `node2` 原来的下一个节点。

局部关系变成：

```text
1 -> 3
```

这一步保证交换后处于后面的 `node1` 还能连接后续链表。

### 第三步

执行：

```go
node2.Next = node1
```

让 `node2` 指向 `node1`。

此时局部交换完成：

```text
temp -> 2 -> 1 -> 3
```

### 第四步

执行：

```go
temp = node1
```

交换完成后，`node1` 已经变成这一组的尾节点。
它正好是下一组待交换节点的前驱节点。

例如：

```text
dummy -> 2 -> 1 -> 3 -> 4
              |
             temp
```

## 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

## 本题总结

两两交换节点的关键是站在待交换节点的前一个位置。

每一轮都要完成三条连接：

1. 前驱节点连接第二个节点。
2. 第一个节点连接第二个节点的后继。
3. 第二个节点连接第一个节点。

然后把游标移动到交换后这一组的尾部，为下一轮交换做准备。

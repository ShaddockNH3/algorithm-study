# 移除链表元素：虚拟头节点统一删除逻辑

移除链表元素是链表删除操作的基础题。
这道题最重要的技巧是使用虚拟头节点，
让删除头节点和删除普通节点的逻辑保持一致。

## 203 移除链表元素

[题目链接](https://leetcode.cn/problems/remove-linked-list-elements/)

### 203 题意

给定一个链表的头节点 `head` 和一个整数 `val`，
删除链表中所有满足 `Node.val == val` 的节点，
并返回新的头节点。

示例：

```text
输入：head = [1,2,6,3,4,5,6], val = 6
输出：[1,2,3,4,5]
```

![示例](https://assets.leetcode.com/uploads/2021/03/06/removelinked-list.jpg)

## 虚拟头节点

### 使用原因

如果直接在原始链表上删除节点，最麻烦的情况是要删除头节点。

例如：

```text
head = [7,7,7], val = 7
```

这时需要不断更新 `head`，否则无法正确返回新的头节点。

虚拟头节点可以统一这个边界情况。
它本身不保存有效数据，只让 `Next` 指向真实头节点。
这样原来的头节点就变成了虚拟头节点的后继节点。

删除原头节点时，也可以用和删除中间节点完全一样的方式处理。

### 指针含义

代码中使用 `tmp` 从 `dummyHead` 开始遍历。

`tmp` 始终表示当前待检查节点的前一个节点。

因此真正需要检查的是 `tmp.Next`：

- 如果 `tmp.Next.Val == val`，就删除 `tmp.Next`。
- 如果 `tmp.Next.Val != val`，`tmp` 才向前移动。

删除节点时 `tmp` 不能立即移动。
因为新的 `tmp.Next` 可能仍然需要被删除，
这正好可以处理连续多个目标值的情况。

### 203 代码

```go
func removeElements(head *ListNode, val int) *ListNode {
    dummyHead := &ListNode{Val: -1, Next: head}
    tmp := dummyHead

    for tmp.Next != nil {
        if tmp.Next.Val == val {
            tmp.Next = tmp.Next.Next
        } else {
            tmp = tmp.Next
        }
    }

    return dummyHead.Next
}
```

### 203 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

## 本题总结

链表删除问题中，虚拟头节点可以减少边界判断。

它的价值在于：

1. 统一删除头节点和删除中间节点的逻辑。
2. 让遍历指针始终站在待检查节点的前一个位置。
3. 删除后保持原地不动，继续检查新的后继节点。

后续链表题中，只要涉及头节点可能变化，
都可以优先考虑虚拟头节点。

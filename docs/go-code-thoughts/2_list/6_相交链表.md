# 相交链表：双指针、哈希集合与长度差

链表相交问题要求找到两个单链表相交的起始节点。
这里的相交指的是节点地址相同，而不是节点值相同。

这篇整理三种常见解法：

- 双指针法。
- 哈希集合法。
- 长度差法。

## 160 相交链表

[题目链接](https://leetcode.cn/problems/intersection-of-two-linked-lists/)

### 160 题意

给定两个单链表的头节点 `headA` 和 `headB`，
找出并返回两个单链表相交的起始节点。

如果两个链表没有交点，返回 `nil`。

## 方法一：双指针法

### 双指针思路

双指针法是空间复杂度为 `O(1)` 的解法。

设置两个指针：

- `pa` 从 `headA` 出发。
- `pb` 从 `headB` 出发。

当某个指针走到当前链表末尾后，就切换到另一条链表的头节点继续走。

如果两个链表相交，两个指针会在交点相遇。
如果不相交，两个指针最终会同时变成 `nil`。

### 双指针代码

```go
func getIntersectionNode(headA, headB *ListNode) *ListNode {
    if headA == nil || headB == nil {
        return nil
    }

    pa, pb := headA, headB

    for pa != pb {
        if pa == nil {
            pa = headB
        } else {
            pa = pa.Next
        }

        if pb == nil {
            pb = headA
        } else {
            pb = pb.Next
        }
    }

    return pa
}
```

### 双指针推导

设：

- 链表 A 独有部分长度为 `a`。
- 链表 B 独有部分长度为 `b`。
- 公共部分长度为 `c`。

那么：

- `pa` 走过的路径是 `a + c + b`。
- `pb` 走过的路径是 `b + c + a`。

二者长度相同：

```text
a + c + b == b + c + a
```

所以两个指针速度相同时，最终会在同一个位置相遇。

如果存在交点，相遇点就是第一个公共节点。
如果不存在交点，也就是 `c = 0`，两个指针会同时到达 `nil`。

### 双指针复杂度

- 时间复杂度：`O(m + n)`
- 空间复杂度：`O(1)`

## 方法二：哈希集合法

### 哈希集合思路

哈希集合法比较直观。

1. 遍历链表 A，把所有节点地址存入集合。
2. 遍历链表 B。
3. 如果 B 中某个节点地址出现在集合中，它就是交点。
4. 如果 B 遍历结束仍未找到，说明没有交点。

### 哈希集合代码

```go
func getIntersectionNode(headA, headB *ListNode) *ListNode {
    set := make(map[*ListNode]struct{})

    for tmp := headA; tmp != nil; tmp = tmp.Next {
        set[tmp] = struct{}{}
    }

    for tmp := headB; tmp != nil; tmp = tmp.Next {
        if _, ok := set[tmp]; ok {
            return tmp
        }
    }

    return nil
}
```

### 哈希集合复杂度

- 时间复杂度：`O(m + n)`
- 空间复杂度：`O(m)`

也可以先存较短链表，空间复杂度就取较短链表长度。

## 方法三：长度差法

### 长度差思路

长度差法先让两个指针距离终点的长度对齐。

1. 分别计算链表 A 和链表 B 的长度。
2. 计算长度差 `diff`。
3. 让较长链表的指针先走 `diff` 步。
4. 然后两个指针同步前进。
5. 第一次相遇的节点就是交点。

如果没有交点，两个指针会同时走到 `nil`。

### 长度差代码

```go
func getIntersectionNode(headA, headB *ListNode) *ListNode {
    lenA, lenB := 0, 0

    for tmp := headA; tmp != nil; tmp = tmp.Next {
        lenA++
    }
    for tmp := headB; tmp != nil; tmp = tmp.Next {
        lenB++
    }

    pa, pb := headA, headB
    if lenB > lenA {
        pa, pb = headB, headA
    }

    diff := lenA - lenB
    if lenB > lenA {
        diff = lenB - lenA
    }

    for i := 0; i < diff; i++ {
        pa = pa.Next
    }

    for pa != pb {
        pa = pa.Next
        pb = pb.Next
    }

    return pa
}
```

### 长度差复杂度

- 时间复杂度：`O(m + n)`
- 空间复杂度：`O(1)`

## 方法对比

- 双指针法：
  时间 `O(m + n)`，空间 `O(1)`，代码最简洁。
- 哈希集合法：
  时间 `O(m + n)`，空间 `O(m)`，思路最直观。
- 长度差法：
  时间 `O(m + n)`，空间 `O(1)`，推导清晰。

## 本题总结

链表相交判断的是节点地址是否相同。

双指针法的核心是让两个指针分别走过 A 和 B 两条链表的总长度。
无论两条链表长度是否相同，它们最终都会对齐到同一个位置。

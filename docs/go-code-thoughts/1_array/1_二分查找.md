# 二分查找：从基础查找到边界问题

二分查找不是某一个固定函数，而是一种算法思想。
它适合在有序空间中，通过不断缩小搜索区间来定位目标。

这篇笔记整理了几类常见场景：

- 查找一个确切的值。
- 寻找左侧边界。
- 在概念上的有序空间中二分。

## 二分查找的前提

二分查找最重要的前提是有序。

只要搜索空间满足某种单调性，就可以考虑二分。
这个空间可以是数组，也可以是一个逻辑上的数字范围。

二分查找之所以快，是因为每次判断都能排除一半候选范围。
例如在 1024 个位置中查找目标，顺序扫描最坏需要 1024 次，
而二分查找每次减半，查找次数增长得很慢。

## 704 二分查找

[题目链接](https://leetcode.cn/problems/binary-search/)

### 704 题意

在一个没有重复元素的有序数组中，找到 `target` 的位置。
如果找到，返回下标；如果找不到，返回 `-1`。

### 704 思路

这是最基础的二分模板。

本篇统一使用闭区间 `[left, right]`：

1. 初始化 `left := 0`，`right := len(nums)-1`。
2. 当 `left <= right` 时，说明搜索区间仍然存在。
3. 计算中点 `mid := left + (right-left)/2`。
4. 如果 `nums[mid] == target`，直接返回。
5. 如果 `nums[mid] > target`，目标在左侧，更新 `right`。
6. 如果 `nums[mid] < target`，目标在右侧，更新 `left`。

### 704 代码

```go
func search(nums []int, target int) int {
    left, right := 0, len(nums)-1

    for left <= right {
        mid := left + (right-left)/2

        if nums[mid] == target {
            return mid
        } else if nums[mid] > target {
            right = mid - 1
        } else {
            left = mid + 1
        }
    }

    return -1
}
```

### 704 小结

这个模板适合处理没有重复元素，并且只需要找到一个目标值的场景。

## 35 和 34 寻找边界

- [35 搜索插入位置](https://leetcode.cn/problems/search-insert-position/)
- [34 在排序数组中查找元素的第一个和最后一个位置](https://leetcode.cn/problems/find-first-and-last-position-of-element-in-sorted-array/)

### 边界问题目标

边界问题不只是判断能不能找到目标，而是要找到某个位置。

常见目标包括：

- 找到 `target` 应该插入的位置。
- 找到 `target` 第一次出现的位置。
- 找到 `target` 最后一次出现的位置。

这两个问题的核心都是寻找左侧边界：
第一个大于或等于 `target` 的元素位置。

### 左侧边界思路

基础查找中，`nums[mid] == target` 意味着已经找到答案。

但在寻找左侧边界时，`nums[mid] == target` 只说明 `mid`
是一个可能答案。为了判断左边是否还有更靠前的目标，
仍然要继续向左收缩。

因此判断条件可以合并成：

```go
if nums[mid] >= target {
    right = mid - 1
} else {
    left = mid + 1
}
```

循环结束后，`left` 就是第一个大于或等于 `target` 的位置。

### 左侧边界代码

```go
func findLeftBoundary(nums []int, target int) int {
    left, right := 0, len(nums)-1

    for left <= right {
        mid := left + (right-left)/2

        if nums[mid] >= target {
            right = mid - 1
        } else {
            left = mid + 1
        }
    }

    return left
}
```

### 35 应用

`findLeftBoundary(nums, target)` 的返回值就是 `target` 的插入位置。

如果数组中已经有 `target`，返回它第一次出现的位置。
如果数组中没有 `target`，返回它应该插入的位置。

### 34 应用

查找起始位置：

```go
leftBound := findLeftBoundary(nums, target)
```

查找结束位置：

```go
rightBound := findLeftBoundary(nums, target+1) - 1
```

找到 `leftBound` 后，还需要检查：

1. `leftBound` 是否越界。
2. `nums[leftBound]` 是否真的等于 `target`。

如果检查不通过，说明 `target` 不存在。

## 69 和 367 概念空间二分

- [69 x 的平方根](https://leetcode.cn/problems/sqrtx/)
- [367 有效的完全平方数](https://leetcode.cn/problems/valid-perfect-square/)

### 概念空间目标

二分查找不一定只能用在数组中。

只要搜索空间具备单调性，就可以二分。
例如在 `[0, x]` 这个数字范围内，寻找最大的整数 `k`，
使得：

```text
k * k <= x
```

这就是 69 x 的平方根的本质。

### 69 思路

这道题可以理解成寻找右侧边界：
最大的满足 `k*k <= x` 的整数 `k`。

为了避免 `mid * mid` 溢出，可以使用：

```go
mid <= x/mid
```

如果条件成立，说明 `mid` 是一个可能答案，
继续向右尝试更大的值。

如果条件不成立，说明 `mid` 太大，需要向左收缩。

### 69 代码

```go
func mySqrt(x int) int {
    if x == 0 || x == 1 {
        return x
    }

    left, right := 1, x
    ans := 0

    for left <= right {
        mid := left + (right-left)/2

        if mid <= x/mid {
            ans = mid
            left = mid + 1
        } else {
            right = mid - 1
        }
    }

    return ans
}
```

### 367 应用

判断 `num` 是否是完全平方数，可以先求整数平方根：

```go
k := mySqrt(num)
```

然后判断：

```go
return k*k == num
```

如果担心乘法溢出，也可以继续使用除法形式判断。

## 二分查找总结

二分查找常见问题可以先分成三类：

1. 查找一个确切的值。
2. 查找满足条件的边界。
3. 在逻辑上有序的空间中查找答案。

写二分时，要先明确搜索区间。
本篇采用闭区间 `[left, right]`，对应循环条件是 `left <= right`。

当二分查找变得模糊时，可以先问自己：
目标是找一个确切值，还是找一个满足条件的边界。
这个问题想清楚后，代码边界会清晰很多。

# 双指针专题：原地修改数组与字符串

这一组题都围绕双指针展开。

双指针通常是在数组或字符串上使用两个索引，
通过不同的移动策略完成原地修改、去重、交换或比较。

它的常见优势是：

- 时间复杂度通常可以做到 `O(n)`。
- 空间复杂度通常可以做到 `O(1)`。

## 快慢指针

快慢指针是最常见的双指针模式。

- 慢指针负责维护已经处理好的区域。
- 快指针负责向前遍历，寻找需要保留或处理的元素。

## 27 移除元素

[题目链接](https://leetcode.cn/problems/remove-element/)

### 27 题意

给定一个数组 `nums` 和一个值 `val`，
原地移除所有数值等于 `val` 的元素，并返回新长度。

元素顺序可以改变，题目只关心新长度范围内的元素是否正确。

### 27 思路

使用快慢指针。

1. `right` 遍历整个数组。
2. `left` 指向下一个应该放入有效元素的位置。
3. 如果 `nums[right] != val`，说明这是需要保留的元素。
4. 把它写到 `nums[left]`，然后移动 `left`。
5. 如果 `nums[right] == val`，直接跳过。

最终 `left` 就是新数组长度。

### 27 代码

```go
func removeElement(nums []int, val int) int {
    left, right := 0, 0
    numsLength := len(nums)

    for right < numsLength {
        if nums[right] != val {
            nums[left] = nums[right]
            left++
        }
        right++
    }

    return left
}
```

### 27 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

## 26 删除有序数组中的重复项

[题目链接](https://leetcode.cn/problems/remove-duplicates-from-sorted-array/)

### 26 题意

给定一个非严格递增排列的数组 `nums`，
原地删除重复出现的元素，使每个元素只出现一次。

返回删除后数组的新长度，并保持元素的相对顺序。

### 26 思路

因为数组已经有序，重复元素一定相邻。

1. `left` 指向当前已保留区域的最后一个元素。
2. `right` 从下一个位置开始向右探索。
3. 如果 `nums[right] != nums[left]`，说明遇到新元素。
4. 先移动 `left`，再把 `nums[right]` 写入 `nums[left]`。
5. 遍历结束后，新长度是 `left + 1`。

### 26 代码

```go
func removeDuplicates(nums []int) int {
    numsLength := len(nums)
    if numsLength <= 1 {
        return numsLength
    }

    left, right := 0, 1

    for ; right < numsLength; right++ {
        if nums[left] != nums[right] {
            left++
            nums[left] = nums[right]
        }
    }

    return left + 1
}
```

### 26 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

## 283 移动零

[题目链接](https://leetcode.cn/problems/move-zeroes/)

### 283 题意

给定一个数组 `nums`，将所有 `0` 移动到数组末尾，
同时保持非零元素的相对顺序。

要求在不复制数组的情况下原地操作。

### 283 思路

移动零也是快慢指针，但和前两题不同。

前两题可以直接覆盖元素，因为被覆盖的值本来就不需要保留。
这道题要把 `0` 移到末尾，因此更适合用交换。

1. `left` 指向下一个非零元素应该放置的位置。
2. `right` 负责寻找非零元素。
3. 当 `nums[right] != 0` 时，交换 `nums[left]` 和 `nums[right]`。
4. 交换后移动 `left`。
5. `right` 始终向右遍历。

这样可以把非零元素按原相对顺序移动到前面。

### 283 代码

```go
func moveZeroes(nums []int) {
    left, right := 0, 0

    for right < len(nums) {
        if nums[right] != 0 {
            nums[left], nums[right] = nums[right], nums[left]
            left++
        }
        right++
    }
}
```

### 283 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`

## 对撞指针

对撞指针通常让一个指针从左侧开始，另一个指针从右侧开始，
两个指针向中间移动。

它适合处理排序数组中两端信息都重要的问题。

## 977 有序数组的平方

[题目链接](https://leetcode.cn/problems/squares-of-a-sorted-array/)

### 977 题意

给定一个按非递减顺序排序的整数数组 `nums`，
返回每个数字的平方组成的新数组，要求结果也按非递减顺序排序。

进阶要求是设计时间复杂度为 `O(n)` 的算法。

### 977 思路

原数组有序，但可能包含负数。
平方之后，最大值可能来自数组最左端，也可能来自最右端。

例如 `[-10, -1, 5]` 平方后，最大值来自 `-10`。

因此可以使用对撞指针：

1. `left` 指向数组开头。
2. `right` 指向数组末尾。
3. `k` 指向结果数组末尾。
4. 每次比较 `nums[left]` 和 `nums[right]` 的绝对值。
5. 把平方较大的值写入 `ans[k]`。
6. 移动对应指针，并继续向前填充 `ans`。

### 977 代码

```go
func absInt(x int) int {
    if x >= 0 {
        return x
    }
    return -x
}

func sortedSquares(nums []int) []int {
    numsLength := len(nums)
    ans := make([]int, numsLength)
    left, right := 0, numsLength-1

    for k := numsLength - 1; k >= 0; k-- {
        if absInt(nums[right]) > absInt(nums[left]) {
            ans[k] = nums[right] * nums[right]
            right--
        } else {
            ans[k] = nums[left] * nums[left]
            left++
        }
    }

    return ans
}
```

### 977 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`，需要返回新数组

## 独立移动的双指针

有些双指针不是在同一个数组中相向或同向移动，
而是分别在两个字符串或两个序列中独立移动。

## 844 比较含退格的字符串

[题目链接](https://leetcode.cn/problems/backspace-string-compare/)

### 844 题意

给定两个字符串 `s` 和 `t`。

当它们分别被输入到空白文本编辑器后，如果两者相等，返回 `true`。
字符 `#` 表示退格。

进阶要求是用 `O(n)` 时间复杂度和 `O(1)` 空间复杂度解决。

### 844 思路

如果新建字符串模拟输入过程，空间复杂度会变成 `O(n)`。

为了做到 `O(1)` 空间，可以从后往前遍历：

1. 用 `i` 指向 `s` 的末尾。
2. 用 `j` 指向 `t` 的末尾。
3. 用 `skipS` 记录 `s` 中还需要跳过的字符数。
4. 用 `skipT` 记录 `t` 中还需要跳过的字符数。
5. 每次分别找到两个字符串中的下一个有效字符。
6. 如果有效字符不同，返回 `false`。
7. 如果一个字符串还有有效字符，另一个没有，也返回 `false`。

这个方法用两个指针和两个计数器模拟退格。

### 844 代码

```go
func backspaceCompare(s string, t string) bool {
    i := len(s) - 1
    j := len(t) - 1
    skipS := 0
    skipT := 0

    for i >= 0 || j >= 0 {
        for i >= 0 {
            if s[i] == '#' {
                skipS++
                i--
            } else if skipS > 0 {
                skipS--
                i--
            } else {
                break
            }
        }

        for j >= 0 {
            if t[j] == '#' {
                skipT++
                j--
            } else if skipT > 0 {
                skipT--
                j--
            } else {
                break
            }
        }

        if i >= 0 && j >= 0 {
            if s[i] != t[j] {
                return false
            }
        } else if i >= 0 || j >= 0 {
            return false
        }

        i--
        j--
    }

    return true
}
```

### 844 复杂度

- 时间复杂度：`O(m + n)`
- 空间复杂度：`O(1)`

## 双指针总结

这一组题展示了双指针的几种常见形态：

- 快慢指针：
  适合原地移除、去重等问题。
- 交换型快慢指针：
  适合移动零这类需要维持相对顺序的问题。
- 对撞指针：
  适合有序数组两端信息都重要的问题。
- 独立移动双指针：
  适合比较两个字符串或序列，并且移动规则较复杂的问题。

遇到数组原地修改题时，可以先想清楚两个区域：
已处理区域在哪里，待探索区域在哪里。确定这两个区域后，
双指针的移动规则通常就会清晰很多。

# 区间和：前缀和的基础应用

区间和问题是前缀和的基础题。
它展示了前缀和最核心的价值：
用一次预处理，换取多次区间查询的高效率。

## 卡码网 58 区间和

[题目链接](https://kamacoder.com/problempage.php?pid=1070)

### 58 题目描述

给定一个整数数组 `Array`，计算该数组在每个指定区间内元素的总和。

### 58 输入描述

第一行输入整数数组 `Array` 的长度 `n`。

接下来 `n` 行，每行一个整数，表示数组元素。
随后的输入为需要计算总和的区间，直到文件结束。

### 58 输出描述

输出每个指定区间内元素的总和。

## 阶段一：暴力循环

### 暴力思路

最直接的想法是：每来一个区间查询，就从区间起点加到终点。

对于查询 `[start, end]`：

1. 从 `arr[start]` 开始累加。
2. 一直加到 `arr[end]`。
3. 输出区间总和。

这个想法正确，也很容易实现。

### 暴力代码

```go
package main

import "fmt"

func calculateSumNaively(arr []int, start int, end int) int {
    sum := 0
    for i := start; i <= end; i++ {
        sum += arr[i]
    }
    return sum
}
```

### 暴力解法问题

暴力方法的瓶颈是重复计算。

如果数组有很多元素，查询次数也很多，
并且每次查询的区间都很大，那么同一个元素会被反复累加。

单次查询最坏时间复杂度是 `O(n)`。
如果有 `q` 次查询，总复杂度会变成 `O(n * q)`。

## 阶段二：前缀和优化

### 前缀和思路

前缀和的核心是先构建一个辅助数组 `prefixSums`。

这里使用长度为 `n + 1` 的前缀和数组：

```text
prefixSums[i] = arr[0] + arr[1] + ... + arr[i-1]
```

因此：

```text
prefixSums[0] = 0
prefixSums[1] = arr[0]
prefixSums[2] = arr[0] + arr[1]
```

要求区间 `[start, end]` 的和，可以用：

```text
prefixSums[end+1] - prefixSums[start]
```

也就是：

```text
arr[start] + ... + arr[end]
= (arr[0] + ... + arr[end]) - (arr[0] + ... + arr[start-1])
```

长度加一的写法可以自然处理 `start = 0` 的情况，
不需要额外写条件判断。

### 前缀和代码

```go
package main

import (
    "fmt"
    "io"
)

func main() {
    var n int
    fmt.Scan(&n)

    arr := make([]int, n)
    for i := 0; i < n; i++ {
        fmt.Scan(&arr[i])
    }

    prefixSums := make([]int, n+1)
    prefixSums[0] = 0

    for i := 0; i < n; i++ {
        prefixSums[i+1] = prefixSums[i] + arr[i]
    }

    var start, end int
    for {
        _, err := fmt.Scan(&start, &end)
        if err == io.EOF {
            break
        }

        result := prefixSums[end+1] - prefixSums[start]
        fmt.Println(result)
    }
}
```

### 前缀和复杂度

设数组长度为 `n`，查询次数为 `q`。

- 预处理时间复杂度：`O(n)`
- 单次查询时间复杂度：`O(1)`
- 总时间复杂度：`O(n + q)`
- 空间复杂度：`O(n)`

## 本题总结

前缀和适合解决静态数组的频繁区间查询问题。

它体现了预处理思想：

1. 先花 `O(n)` 时间构建辅助信息。
2. 后续每次查询都用 `O(1)` 时间回答。

当题目中出现多次区间求和，并且数组本身不会频繁修改时，
前缀和通常是最优先考虑的工具。

# 开发商购买土地：按行列求和的前缀和应用

这道题是前缀和在二维网格切分问题中的应用。
核心不是构建完整二维前缀和，而是把问题转化为行和、列和的一维前缀和。

## 卡码网 44 开发商购买土地

[题目链接](https://kamacoder.com/problempage.php?pid=1044)

### 44 题目描述

一个城市区域被划分成 `n * m` 个连续区块。
每个区块都有一个权值，表示土地价值。

现在有 A、B 两家公司购买土地，需要把所有区块分配给这两家公司。

限制条件如下：

- 只能按横向或纵向切分成两个子区域。
- 每个子区域都必须包含至少一个区块。
- 区块不可再分。

目标是找到一种切分方式，使两家公司获得土地总价值之差最小。

## 初始思路

最直接的思路是枚举所有可能的切割方案。

对每种切法：

1. 计算切分后两块区域的价值和。
2. 求两块区域价值差的绝对值。
3. 不断维护最小差值。

如果每次都重新遍历网格求和，会有大量重复计算。
因此可以先计算行和、列和，再对它们分别构建前缀和。

## 初始代码片段与问题

原始尝试已经抓住了前缀和方向，但实现细节还有几个问题。

```go
func solve(arr [][]int) int {
    nArr := make([]int, len(arr))
    mArr := make([]int, len(arr[0]))

    for i := 0; i < len(arr[0]); i++ {
        for j := 0; j < len(arr); j++ {
            nArr[i] += arr[j][i]
        }
    }

    for i := 0; i < len(arr); i++ {
        for j := 0; j < len(arr[0]); j++ {
            mArr[i] += arr[i][j]
        }
    }

    nSums := getSums(nArr)
    mSums := getSums(mArr)

    nAns, mAns := math.MaxInt, math.MaxInt

    for i := 0; i < len(arr); i++ {
        left := getSum(nSums, 0, i)
        right := getSum(nSums, i+1, len(arr)-1)
        nAns = min(nAns, abs(left-right))
    }

    for i := 0; i < len(arr[0]); i++ {
        left := getSum(mSums, 0, i)
        right := getSum(mSums, i+1, len(arr[0])-1)
        mAns = min(mAns, abs(left-right))
    }

    return min(nAns, mAns)
}
```

这段初始思路有三个需要修正的点：

1. 数组大小要对应清楚。
   行和数组长度应该是 `n`，列和数组长度应该是 `m`。
2. 横向切割应该使用行前缀和，纵向切割应该使用列前缀和。
3. 切割必须保证两边都有区域。
   因此横向切割只能到 `n-2`，纵向切割只能到 `m-2`。

## 修正后的思路

更稳定的做法是：

1. 计算每一行的总和，得到 `rowSums`。
2. 计算每一列的总和，得到 `colSums`。
3. 分别构建行前缀和、列前缀和。
4. 枚举横向切割点，用行前缀和计算上下两部分价值。
5. 枚举纵向切割点，用列前缀和计算左右两部分价值。
6. 更新最小差值。

横向切割时：

```text
topSum = rowPrefixSums[i]
bottomSum = totalRowSum - topSum
```

纵向切割时：

```text
leftSum = colPrefixSums[j]
rightSum = totalColSum - leftSum
```

## 修正后的代码

```go
package main

import "math"

func min(a, b int) int {
    if a < b {
        return a
    }
    return b
}

func abs(x int) int {
    if x < 0 {
        return -x
    }
    return x
}

func getPrefixSums(arr []int) []int {
    if len(arr) == 0 {
        return []int{}
    }

    sums := make([]int, len(arr))
    sums[0] = arr[0]

    for i := 1; i < len(arr); i++ {
        sums[i] = sums[i-1] + arr[i]
    }

    return sums
}

func solve(arr [][]int) int {
    n := len(arr)
    m := len(arr[0])

    rowSums := make([]int, n)
    colSums := make([]int, m)

    for i := 0; i < n; i++ {
        for j := 0; j < m; j++ {
            rowSums[i] += arr[i][j]
            colSums[j] += arr[i][j]
        }
    }

    rowPrefixSums := getPrefixSums(rowSums)
    colPrefixSums := getPrefixSums(colSums)

    minDiff := math.MaxInt

    if n > 1 {
        totalRowSum := rowPrefixSums[n-1]
        for i := 0; i < n-1; i++ {
            topSum := rowPrefixSums[i]
            bottomSum := totalRowSum - topSum
            diff := abs(topSum - bottomSum)
            minDiff = min(minDiff, diff)
        }
    }

    if m > 1 {
        totalColSum := colPrefixSums[m-1]
        for j := 0; j < m-1; j++ {
            leftSum := colPrefixSums[j]
            rightSum := totalColSum - leftSum
            diff := abs(leftSum - rightSum)
            minDiff = min(minDiff, diff)
        }
    }

    return minDiff
}
```

## 复杂度

- 时间复杂度：`O(n * m)`
- 空间复杂度：`O(n + m)`

需要遍历整个网格来统计行和、列和。
之后枚举切割点只需要 `O(n + m)`。

## 本题总结

这道题说明，一个正确的算法思想还需要严谨的实现细节支撑。

初始思路已经抓住了前缀和和枚举切割点。
真正需要修正的是：

- 行和、列和数组大小。
- 横向切割与行前缀和的对应关系。
- 纵向切割与列前缀和的对应关系。
- 切割边界不能让任意一边为空。

当前缀和用于二维问题时，不一定总要构建完整二维前缀和。
如果切割方式只和整行或整列有关，行和、列和的一维前缀和就足够。

450 删除二叉搜索树中的节点 ​

题目链接

题目描述 ​

给定一个二叉搜索树的根节点 root 和一个值 key， 删除二叉搜索树中的 key 对应的节点， 并保证二叉搜索树的性质不变。

返回二叉搜索树的根节点。 这个根节点有可能因为删除操作而发生变化。

递归函数的约定 ​

这道题的关键是先明确递归函数的含义：

deleteNode(root, key) 会在以 root 为根的树中删除 key， 并返回删除之后这棵子树的新根节点。

有了这个约定之后，上层节点就可以这样接收结果：

go
root.Left = deleteNode(root.Left, key)
root.Right = deleteNode(root.Right, key)

因为删除操作可能会让某棵子树的根节点变化。 例如删除的节点正好是这棵子树的根， 那就需要由它的孩子或继承节点来接替这个位置。

查找阶段 ​

利用 BST 的有序性寻找要删除的节点。

  • 如果 key < root.Val，说明目标只可能在左子树。
  • 如果 key > root.Val，说明目标只可能在右子树。
  • 如果 key == root.Val，说明当前节点就是要删除的节点。

查找左子树或右子树时， 都要把递归返回的新子树重新接回当前节点。

删除阶段 ​

找到目标节点后，需要根据它的孩子数量分类处理。

没有左子树 ​

如果 root.Left == nil，删除当前节点后， 能接替当前位置的只有右子树。

go
return root.Right

如果当前节点本来就是叶子节点， 那么 root.Right 也是 nil。 此时返回 nil，正好表示删除了这个叶子节点。

没有右子树 ​

如果 root.Right == nil，删除当前节点后， 能接替当前位置的只有左子树。

go
return root.Left

同时有左右子树 ​

如果当前节点左右子树都存在，不能直接把某一边返回。 否则另一边会丢失。

一种经典处理方式是找当前节点的后继节点： 右子树中最左边的节点。

这个节点满足：

  • 它比当前节点大。
  • 它是右子树中最小的节点。
  • 用它替换当前节点后，仍然能保持 BST 性质。

处理步骤是：

  1. 在右子树中一路向左，找到后继节点。
  2. 用后继节点的值覆盖当前节点的值。
  3. 再到右子树中删除这个后继节点。

第三步很重要。 因为值复制之后，树中会暂时出现两个相同值， 所以还要把后继节点原来的位置删掉。

最终代码 ​

go
/**
 * Definition for a binary tree node.
 * type TreeNode struct {
 *     Val int
 *     Left *TreeNode
 *     Right *TreeNode
 * }
 */
func deleteNode(root *TreeNode, key int) *TreeNode {
    if root == nil {
        return nil
    }

    if key < root.Val {
        root.Left = deleteNode(root.Left, key)
    } else if key > root.Val {
        root.Right = deleteNode(root.Right, key)
    } else {
        if root.Right == nil {
            return root.Left
        }
        if root.Left == nil {
            return root.Right
        }

        successor := root.Right
        for successor.Left != nil {
            successor = successor.Left
        }

        root.Val = successor.Val
        root.Right = deleteNode(root.Right, successor.Val)
    }

    return root
}

本篇总结 ​

删除 BST 节点最难的是删除后的连接关系。

递归函数必须返回当前子树删除后的新根节点， 这样父节点才能重新接好左子树或右子树。

当目标节点有两个孩子时， 可以用右子树最左节点作为后继节点替换当前节点， 再递归删除后继节点原来的位置。
